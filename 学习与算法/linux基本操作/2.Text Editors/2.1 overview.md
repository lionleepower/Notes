# Linux Text Editors：Overview 文本编辑器概览

本节开始介绍 Linux 中常见的文本编辑器，重点是：

- `Vim`
- `Emacs`

同时也提到了更适合新手的编辑器：

- `nano`
- `gedit`

这一节的核心思想是：

```text
Linux 中有很多文本编辑器，不同编辑器有不同工作流，选择适合自己的即可。
````

---

## 1. Linux 中为什么需要文本编辑器？

在 Linux 中，经常需要编辑文本文件，例如：

- 配置文件
    
- shell 脚本
    
- Python/C/C++ 源代码
    
- Markdown 笔记
    
- 日志说明文件
    
- 远程服务器上的文件
    

所以熟悉至少一个 Linux 文本编辑器非常重要。

尤其是在远程服务器、HPC 集群、WSL 或 SSH 环境中，图形界面不一定可用，这时命令行文本编辑器会非常有用。

---

## 2. 本节介绍的主要编辑器

本节主要简要介绍两个经典编辑器：

|编辑器|英文|特点|
|---|---|---|
|Vim|Vi Improved|模式化编辑，轻量、高效、常见|
|Emacs|Emacs|功能强大，强调快捷键和编辑器内部生态|

除了它们之外，还提到：

|编辑器|特点|
|---|---|
|`nano`|简单、适合新手、容易上手|
|`gedit`|图形界面文本编辑器，简单直观|

# Part 1：Vim 和 Emacs

## 3. Vim 是什么？

`Vim` 全称通常理解为：

```text
Vi Improved
```

也就是 Vi 的增强版本。

Vim 是 Linux 中非常常见的命令行文本编辑器。

它的主要特点是：

- 使用不同的 editing modes
    
- 启动速度快
    
- 在服务器环境中很常见
    
- 不依赖图形界面
    
- 适合快速编辑配置文件和代码
    
- 学习曲线较陡，但熟练后效率很高
    

---

## 4. Vim 的核心特点：编辑模式

Vim 和普通文本编辑器最大的区别是：

```text
Vim 有不同的模式。
```

常见模式包括：

|模式|英文|用途|
|---|---|---|
|Normal mode|普通模式|移动光标、删除、复制、粘贴、执行命令|
|Insert mode|插入模式|输入文本|
|Command-line mode|命令行模式|保存、退出、搜索、替换等|

例如：

```text
在 Vim 中，刚打开文件时通常不是直接输入文字，
而是处于 Normal mode。
```

如果想输入文字，需要进入 Insert mode。

---

## 5. Emacs 是什么？

`Emacs` 是另一个非常经典、功能强大的文本编辑器。

它的特点是：

- 功能非常丰富
    
- 依赖大量 key-chords
    
- 可以在编辑器内部完成很多复杂任务
    
- 可扩展性强
    
- 不只是文本编辑器，也可以像一个工作环境
    

---

## 6. Emacs 的核心特点：key-chords

课本中提到：

```text
Emacs relies heavily on key-chords.
```

`key-chords` 指的是多个按键组合，也就是快捷键组合。

例如类似：

```text
Ctrl + 某个键
Alt + 某个键
连续多个快捷键组合
```

Emacs 用户通常会大量使用快捷键来完成编辑、移动、保存、搜索等操作。

---

## 7. Vim 和 Emacs 的工作流区别

课本中特别提到了两者的使用习惯差异。

|对比点|Vim|Emacs|
|---|---|---|
|核心机制|distinct editing modes|key-chords|
|使用方式|经常进入和退出编辑器|倾向于长时间停留在编辑器中|
|复杂任务|通常结合 shell 完成|通常在 Emacs 内部完成|
|风格|编辑器 + Shell 配合|编辑器作为完整工作环境|

---

## 8. Vim 用户的典型习惯

Vim 用户通常倾向于：

```text
进入 Vim 编辑文件，
编辑完退出，
然后回到 Shell 中执行其他命令。
```

例如：

```bash
vim script.sh
```

编辑完成后退出 Vim。

然后在 Shell 中运行：

```bash
bash script.sh
```

或者：

```bash
./script.sh
```

也就是说，Vim 用户常常把：

```text
文本编辑交给 Vim，
复杂任务交给 Shell。
```

---

## 9. Emacs 用户的典型习惯

Emacs 用户通常倾向于：

```text
长时间停留在 Emacs 中，
并在 Emacs 内部完成很多任务。
```

例如：

- 编辑文件
    
- 管理项目
    
- 运行命令
    
- 使用插件
    
- 处理复杂编辑任务
    

所以 Emacs 更像是一个可扩展的工作环境。

# Part 2：其他文本编辑器

## 10. nano

`nano` 是一个简单易用的命令行文本编辑器。

它适合：

- Linux 初学者
    
- 简单修改配置文件
    
- 不想学习复杂快捷键的用户
    
- 临时编辑文本
    

启动方式：

```bash
nano filename
```

例如：

```bash
nano notes.txt
```

nano 的优点是：

```text
上手简单，底部会显示常用快捷键提示。
```

例如保存、退出等操作通常会显示在界面底部。

---

## 11. gedit

`gedit` 是一个图形界面的文本编辑器。

它的特点是：

- 简单
    
- 有 GUI
    
- 类似普通记事本
    
- 适合桌面 Linux 环境
    

启动方式：

```bash
gedit filename
```

例如：

```bash
gedit notes.txt
```

不过在远程服务器或没有图形界面的环境中，`gedit` 不一定可用。

---

## 12. 其他编辑器

除了本节提到的编辑器，Linux 中还有很多其他选择，例如：

- VS Code
    
- Neovim
    
- Sublime Text
    
- Kate
    
- Geany
    
- Micro
    

对于现代开发来说，如果是在本地电脑上，VS Code 很常见。

但在纯命令行服务器环境中，`vim`、`nano`、`emacs` 仍然很重要。

# Part 3：什么是 Vi？

## 13. Vi 是什么？

`Vi` 是一个早期的 Unix 文本编辑器。

它的命令是：

```bash
vi
```

Vim 是 Vi 的增强版本。

两者关系可以简单理解为：

```text
Vi 是前身，Vim 是改进版。
```

其中 Vim 的名字可以理解为：

```text
Vi Improved
```

---

## 14. 为什么现在更关注 Vim？

很多现代 Linux 系统中，输入：

```bash
vi
```

实际上可能打开的是：

```bash
vim
```

也就是说，系统可能把 `vi` 设置成了 `vim` 的别名或链接。

所以本教程主要关注 Vim，而不是单独讲 Vi。

---

## 15. 查看 `vi` 实际对应什么命令

可以使用：

```bash
which vi
```

示例输出：

```bash
$ which vi
alias vi='vim'
    /bin/vim
```

这表示：

```text
当你输入 vi 时，系统实际会执行 vim。
```

---

## 16. `alias vi='vim'` 是什么意思？

输出中的：

```bash
alias vi='vim'
```

表示：

```text
vi 是 vim 的别名。
```

也就是说，当你输入：

```bash
vi
```

Shell 会把它解释为：

```bash
vim
```

所以这两条命令实际效果可能一样：

```bash
vi file.txt
```

```bash
vim file.txt
```

---

## 17. `/bin/vim` 是什么意思？

输出中的：

```bash
/bin/vim
```

表示 `vim` 程序在系统中的位置。

也就是说，实际执行的程序路径是：

```bash
/bin/vim
```

这里的 `/bin` 是 Linux 中常见的可执行程序目录。

# Part 4：如何选择编辑器？

## 18. 初学者推荐

如果只是想快速修改文件，可以优先使用：

```bash
nano
```

例如：

```bash
nano config.txt
```

它比较直观，不需要先理解复杂模式。

---

## 19. 服务器和 HPC 环境推荐

在远程服务器、SSH、HPC 集群中，建议至少掌握一点：

```bash
vim
```

原因是：

- 很多系统默认有 Vim 或 Vi
    
- 不需要图形界面
    
- 适合快速修改配置文件
    
- 适合编辑 shell 脚本和提交脚本
    

例如在 HPC 中，经常需要编辑：

```text
job.slurm
run.sh
config.txt
```

这时 Vim 很常用。

---

## 20. 长期工作流选择

课本建议：

```text
可以都尝试一下，然后选择最适合自己工作流的编辑器。
```

大多数用户最后会形成偏好，并长期使用其中一个。

常见选择方式：

|用户偏好|可能适合|
|---|---|
|想简单直接|nano|
|想高效命令行编辑|Vim|
|想要强大、可扩展工作环境|Emacs|
|想要图形界面|gedit / VS Code|

# 易混点总结

## 1. Vim 不是普通输入型编辑器

刚进入 Vim 时，不能直接像记事本一样输入文字。

因为 Vim 有不同模式。

需要先进入 Insert mode 才能输入文本。

---

## 2. Emacs 依赖大量快捷键

Emacs 不是通过模式切换为主，而是大量使用快捷键组合。

所以它的学习重点和 Vim 不一样。

---

## 3. Vi 和 Vim 不完全一样

Vi 是较早的编辑器。

Vim 是 Vi 的增强版本。

但在很多现代系统中：

```bash
vi
```

可能实际上就是：

```bash
vim
```

---

## 4. 命令行编辑器和图形编辑器不同

`vim`、`emacs`、`nano` 可以在终端中使用。

`gedit` 通常需要图形界面。

在 SSH 远程服务器中，`gedit` 不一定能用。

# 本节重点理解

最重要的内容是：

```text
Vim 使用不同编辑模式。
Emacs 依赖大量快捷键组合。
```
