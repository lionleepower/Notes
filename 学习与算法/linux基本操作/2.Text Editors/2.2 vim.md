# Linux Text Editors：Vim

本节主要介绍 Linux 中非常经典的文本编辑器：

```text
Vim
````

Vim 是一个基于模式的命令行文本编辑器。它非常适合在 Linux、远程服务器、SSH、HPC 集群环境中编辑文件。

本节重点包括：

- 如何打开、创建、保存和退出文件
    
- Vim 的不同模式
    
- Normal mode 普通模式
    
- Insert mode 插入模式
    
- Visual mode 可视模式
    
- Vim 中的复制、粘贴、删除、搜索和替换
    
- Vim 与 Shell 的交互
    
- `.vimrc` 配置文件
    
- `vimtutor` 入门教程
    

---

## 1. Vim 的基本使用

### 1.1 打开已有文件

在 Shell 中输入：

```bash
vim filename
```

例如：

```bash
vim notes.txt
```

表示用 Vim 打开 `notes.txt`。

---
### 1.2 创建新文件

创建新文件的方式和打开文件一样：

```bash
vim newfile.txt
```

如果 `newfile.txt` 不存在，Vim 会打开一个新的空文件。

但是注意：

```text
新文件不会自动保存，必须手动保存。
```

---

## 2. 保存和退出

Vim 的保存和退出命令一般在 Normal mode 中输入。

这些命令通常以 `:` 开头。

---

### 2.1 保存文件：`:w`

```vim
:w
```

含义：

```text
write，保存当前文件。
```

---

### 2.2 退出文件：`:q`

```vim
:q
```

含义：

```text
quit，退出 Vim。
```

如果文件被修改过但没有保存，Vim 会提示错误，不会直接退出。

---

### 2.3 不保存强制退出：`:q!`

```vim
:q!
```

含义：

```text
强制退出，不保存修改。
```

其中：

|符号|含义|
|---|---|
|`q`|quit|
|`!`|force，强制执行|

适合在改乱文件后想直接放弃修改。

---

### 2.4 保存并退出：`:wq`

```vim
:wq
```

含义：

```text
write and quit，保存并退出。
```

也可以理解为：

```text
先 :w 保存，再 :q 退出。
```

---

## 3. 取消正在输入的命令

如果你在 Vim 中开始输入某个命令，但还没输完，想取消，可以按：

```text
Esc Esc
```

也就是连续按两次 `Esc`。

这可以帮助你回到比较安全的 Normal mode。

# Part 1：Vim 的模式

## 4. Vim 是模式化编辑器

Vim 和普通文本编辑器最大的区别是：

```text
Vim 有不同的模式。
```

常见模式包括：

|模式|英文|作用|
|---|---|---|
|Normal mode|普通模式 / 命令模式|移动、复制、删除、粘贴、输入命令|
|Insert mode|插入模式|像普通编辑器一样输入文字|
|Visual mode|可视模式|选中文本后复制、删除等|
|Command-line mode|命令行模式|输入 `:w`、`:q`、搜索替换等命令|

---

## 5. Normal Mode 普通模式

Vim 打开文件后，默认进入：

```text
Normal mode
```

也叫：

```text
Command mode
```

在 Normal mode 中，按键通常不是直接输入文字，而是执行命令。

例如：

|按键|作用|
|---|---|
|`h`|左移|
|`j`|下移|
|`k`|上移|
|`l`|右移|
|`x`|删除一个字符|
|`dd`|删除当前行|
|`yy`|复制当前行|
|`p`|粘贴|

---

## 6. 如何判断自己在 Normal mode？

当你处于 Normal mode 时，Shell 底部一般不会显示：

```text
-- INSERT --
```

也就是说，底部通常没有明显的插入模式提示。

# Part 2：Navigation 导航移动

## 7. 基本移动方式

在 Normal mode 中，可以用方向键移动，也可以使用字母键移动。

|移动方向|按键|
|---|---|
|左|`h`|
|下|`j`|
|上|`k`|
|右|`l`|

---

## 8. 为什么 Vim 使用 `h j k l` 移动？

因为使用这些按键时，手不用离开键盘主区域。

这让熟练用户可以更快地移动光标并执行编辑命令。

简单记忆：

```text
h：left
j：down
k：up
l：right
```

其中 `j` 可以想象成向下的钩子，所以是下移。

---

## 9. 移动到行首：`0`

```vim
0
```

含义：

```text
移动到当前行的开头。
```

---

## 10. 移动到行尾：`$`

```vim
$
```

含义：

```text
移动到当前行的末尾。
```

---

## 11. 移动到下一个单词开头：`w`

```vim
w
```

含义：

```text
移动到下一个 word 的开头。
```

也可以加数字：

```vim
5w
```

表示：

```text
向前移动 5 个单词。
```

---

## 12. 移动到当前或下一个单词结尾：`e`

```vim
e
```

含义：

```text
移动到 word 的结尾。
```

也可以加数字：

```vim
5e
```

表示：

```text
向前移动到第 5 个单词的结尾。
```

|命令|作用|
|---|---|
|`h`|左移|

|`l`|右移|
|`0`|移动到行首|

# Part 3：Insert Mode 插入模式

## 14. 什么是 Insert Mode？

`Insert mode` 是真正输入文字的模式。

刚打开 Vim 时，一般不在 Insert mode，而是在 Normal mode。

如果想编辑文本，需要先进入 Insert mode。

---

## 15. 使用 `i` 进入插入模式

在 Normal mode 中按：

```vim
i
```

表示：

```text
在当前光标位置进入 Insert mode。
```

进入后就可以正常输入文字。

底部会显示：

```text
-- INSERT --
```

---

## 16. 使用 `a` 进入插入模式

在 Normal mode 中按：

```vim
a
```

`a` 表示 append。

它的作用是：

```text
在当前光标后一个字符的位置进入 Insert mode。
```

区别如下：

|命令|含义|
|---|---|
|`i`|在光标当前位置前插入|
|`a`|在光标后面追加|

---

## 17. 退出 Insert Mode

在 Insert mode 中按：

```text
Esc
```

即可回到 Normal mode。

这是 Vim 中最重要的按键之一。

---

## 18. Insert Mode 中如何移动？

在 Insert mode 中，通常使用普通方向键移动：

```text
↑ ↓ ← →
```

不过更典型的 Vim 工作流是：

```text
Esc 回到 Normal mode，再使用 Vim 的导航命令移动。
```

---

# Part 4：Editing Features 编辑功能

## 19. 撤销：`u`

在 Normal mode 中按：

```vim
u
```

表示：

```text
undo，撤销上一个命令。
```

即使是在 Insert mode 中做的最后一次编辑，也可以回到 Normal mode 后按 `u` 撤销。

---

## 20. 重做：`Ctrl-R`

如果撤销后想恢复，可以按：

```text
Ctrl-R
```

含义：

```text
redo，重做刚刚撤销的操作。
```

---

## 21. 复制：yank

在 Vim 中，复制通常叫：

```text
yank
```

常见命令如下：

|命令|含义|
|---|---|
|`yl`|复制光标下的一个字符|
|`yy`|复制当前行|
|`y#y`|复制指定数量的行|
|`#yy`|复制指定数量的行|

其中 `#` 替换成数字。

例如：

```vim
y25y
```

或：

```vim
25yy
```

表示：

```text
复制 25 行。
```

---

## 22. 粘贴：put

在 Vim 中，粘贴也叫：

```text
put
```

常见命令：

| 命令      | 含义             |
| ------- | -------------- |
| `p`(小写) | 粘贴到光标后，或粘贴到下一行 |
| `P`（大写） | 粘贴到光标前，或粘贴到上一行 |

具体区别：

```text
如果复制的是字符或单词：
p 粘贴到光标后
P 粘贴到光标前
```

```text
如果复制的是整行：
p 粘贴到下一行
P 粘贴到上一行
```

---

## 23. 删除或剪切

Vim 中的删除通常也可以理解为剪切，因为删除后的内容可以被粘贴。

常见命令如下：

|命令|含义|
|---|---|
|`x`|删除光标下的一个字符|
|`dd`|删除当前行|
|`d#d`|删除指定数量的行|
|`#dd`|删除指定数量的行|

例如：

```vim
d25d
```

或：

```vim
25dd
```

表示：

```text
删除 25 行。
```

删除后的内容可以用：

```vim
p
```

粘贴到别的位置。

---

## 24. 复制、粘贴、删除总结

|操作|命令|含义|
|---|---|---|
|撤销|`u`|undo|
|重做|`Ctrl-R`|redo|
|复制当前行|`yy`|yank line|
|复制 25 行|`25yy`|yank 25 lines|
|粘贴到后面/下一行|`p`|put after|
|粘贴到前面/上一行|`P`|put before|
|删除一个字符|`x`|delete character|
|删除当前行|`dd`|delete line|
|删除 25 行|`25dd`|delete 25 lines|

# Part 5：搜索和替换

## 25. 搜索单词

在 Normal mode 中输入：

```vim
/word
```

然后按 Enter。

含义：

```text
从当前光标位置向后搜索 word。
```

例如：

```vim
/foo
```

表示搜索 `foo`。

---

## 26. 搜索短语

搜索不一定只能搜单词，也可以搜索短语。

例如：

```vim
/hello world
```

表示搜索短语 `hello world`。

---

## 27. 跳转到下一个匹配：`n`

当搜索已经激活后，可以按：

```vim
n
```

表示：

```text
跳转到下一个匹配项。
```

---

## 28. 跳转到上一个匹配：`N`

按：

```vim
N
```

表示：

```text
跳转到上一个匹配项。
```

注意：

```text
n 和 N 是反方向的。
```

---

## 29. 全局搜索替换

课本给出的例子是：

```vim
:%s/foo/bar/g
```

含义：

```text
在整个文件中，把所有 foo 替换成 bar。
```

---

## 30. 解释 `:%s/foo/bar/g`

这个命令可以拆开理解：

|部分|含义|
|---|---|
|`:`|进入命令行模式|
|`%`|整个文件|
|`s`|substitute，替换|
|`foo`|要查找的内容|
|`bar`|替换成的内容|
|`g`|global，替换每一行中的所有匹配项|

所以：

```vim
:%s/foo/bar/g
```

表示：

```text
对整个文件执行 substitute，把所有 foo 替换为 bar。
```

# Part 6：Split Screen 分屏

## 31. 横向分屏：`:sp`

可以在 Vim 中横向分屏打开另一个文件：

```vim
:sp filename
```

例如：

```vim
:sp notes2.txt
```

含义：

```text
在当前文件上方打开 notes2.txt，并横向分割屏幕。
```

`sp` 可以理解为 split。

---

## 32. 纵向分屏：`:vsp`

可以纵向分屏打开另一个文件：

```vim
:vsp filename
```

例如：

```vim
:vsp notes2.txt
```

含义：

```text
在当前文件左侧打开 notes2.txt，并纵向分割屏幕。
```

`vsp` 可以理解为 vertical split。

---

## 33. 在分屏之间移动

在分屏窗口之间移动，需要先按：

```text
Ctrl-W
```

然后再按方向键或 Vim 导航键。

例如：

```text
Ctrl-W h
```

表示移动到左边的窗口。

也可以：

```text
Ctrl-W ←
```

移动到左边窗口。

常见组合：

|命令|作用|
|---|---|
|`Ctrl-W h`|移动到左侧分屏|
|`Ctrl-W l`|移动到右侧分屏|
|`Ctrl-W j`|移动到下方分屏|
|`Ctrl-W k`|移动到上方分屏|

# Part 7：命令组合

## 34. Vim 命令可以组合

Vim 的强大之处在于：

```text
编辑命令可以和移动命令组合。
```

例如：

```vim
5de
```

表示：

```text
删除接下来 5 个单词。
```

再例如：

```vim
y$
```

表示：

```text
从当前光标位置复制到本行末尾。
```

---

## 35. 为什么组合命令很强大？

因为它让你不用反复进入 Insert mode，也不用用鼠标选中文本。

例如：

|命令|含义|
|---|---|
|`d$`|删除到行尾|
|`y$`|复制到行尾|
|`5w`|向前移动 5 个单词|
|`5de`|删除接下来 5 个单词|
|`dd`|删除一整行|
|`yy`|复制一整行|

这就是 Vim 的核心效率来源之一。

# Part 8：Visual Mode 可视模式

## 36. 什么是 Visual Mode？

`Visual mode` 用于选择一段文本，然后对它执行操作。

例如：

- 选中文本后复制
    
- 选中文本后删除
    
- 选中文本后替换
    

从 Normal mode 进入 Visual mode：

```vim
v
```

---

## 37. Visual Mode 中如何移动？

进入 Visual mode 后，可以使用：

- Normal mode 的导航键：`h j k l`
    
- 普通方向键：`← ↓ ↑ →`
    

移动时，被经过的文本会被高亮选中。

---

## 38. Visual Mode 复制文本流程

如果想复制一段文字，可以这样做：

1. 在 Normal mode 中移动到要复制内容的第一个字符
    
2. 按 `v` 进入 Visual mode
    
3. 移动到要复制内容的最后一个字符
    
4. 按 `y` 复制选中文本
    
5. 移动到要粘贴的位置
    
6. 按 `p` 粘贴
    

对应命令流程：

```text
移动光标 → v → 移动选择范围 → y → 移动到目标位置 → p
```

---

## 39. Visual Mode 中 `p` 和 `P`

如果复制的是 Visual mode 中选中的文本：

```vim
p
```

通常会粘贴到光标后。

```vim
P
```

通常会粘贴到光标前。

课本特别提醒：

```text
即使你复制了多行，Visual mode 下的粘贴行为也可能是粘贴到光标之后，而不是像整行复制那样自动到下一行。
```

---

# Part 9：Vim and the Shell

## 40. 在 Vim 中运行 Shell 命令

在 Normal mode 中，可以使用：

```vim
:!command
```

执行一个 Shell 命令，而不用关闭 Vim。

例如：

```vim
:!ls
```

含义：

```text
在 Vim 中临时执行 ls，显示当前目录内容。
```

---

## 41. `:!ls` 的执行效果

当执行：

```vim
:!ls
```

Vim 会暂时把界面切到 Shell 输出，让你看到 `ls` 的结果。

命令执行完成后，会看到类似：

```text
Press ENTER or type command to continue
```

按 Enter 后返回 Vim 文件界面。

---

## 42. 临时回到 Shell：`Ctrl-Z`

也可以在 Normal mode 中按：

```text
Ctrl-Z
```

把 Vim 暂停并放到后台。

这样你会回到 Shell，可以继续输入命令。

---

## 43. 使用 `fg` 回到 Vim

当 Vim 被 `Ctrl-Z` 暂停后，可以在 Shell 中输入：

```bash
fg
```

回到刚才的 Vim 会话。

这和前面学过的 Job Control 是一致的：

```text
Ctrl-Z 暂停前台进程
fg 把后台任务带回前台
```

---

## 44. 多个 Vim 文件后台切换

如果你用 `Ctrl-Z` 后台了多个 Vim 会话，那么：

```bash
fg
```

会默认恢复最近被后台的那个任务。

也就是：

```text
most recent file accessed first
```

如果需要查看当前 Shell 中的后台任务，可以用：

```bash
jobs
```

然后使用：

```bash
fg %1
```

或：

```bash
fg %2
```

恢复指定任务。

# Part 10：Customization 自定义 Vim

## 45. `.vimrc` 是什么？

Vim 使用配置文件：

```bash
~/.vimrc
```

进行个性化设置。

这个文件里可以写一些 Vim 命令，让 Vim 每次启动时自动执行。

可以理解为：

```text
.vimrc 是 Vim 的个人启动配置文件。
```

---

## 46. 系统级 vimrc 和用户级 vimrc

Vim 配置可能有两个层次：

|配置类型|作用范围|
|---|---|
|global vimrc|系统范围，对所有用户生效|
|`~/.vimrc`|当前用户个人配置|

通常我们自己修改的是：

```bash
~/.vimrc
```

其中 `~` 表示当前用户的 home 目录。

---

## 47. 示例 `.vimrc`

课本给出的示例：

```vim
syntax on
set tabstop=4
set expandtab
set number
set hls
```

这些配置会在每次打开 Vim 时自动生效。

---

## 48. `.vimrc` 配置解释

|配置|含义|
|---|---|
|`syntax on`|开启语法高亮|
|`set tabstop=4`|设置 tab 显示宽度为 4 个字符|
|`set expandtab`|把 tab 转换为空格|
|`set number`|显示行号|
|`set hls`|搜索时高亮匹配内容|

---

## 49. 为什么推荐这些配置？

### `syntax on`

适合写代码，例如：

- C
    
- Python
    
- Shell script
    
- Markdown
    
- LaTeX
    

它会让不同语法结构显示为不同颜色，更容易阅读。

---

### `set tabstop=4`

让 Tab 显示为 4 个字符宽。

这对代码缩进比较重要。

---

### `set expandtab`

将 Tab 转换为空格。

这可以避免不同环境下 Tab 宽度显示不一致。

---

### `set number`

显示行号。

这对调试代码、查看报错位置很有用。

例如编译器报错：

```text
error at line 37
```

如果 Vim 显示行号，就能快速找到第 37 行。

---

### `set hls`

`hls` 可以理解为：

```text
highlight search
```

搜索某个词时，所有匹配项会被高亮显示。

---

# Part 11：Hands-On Tutorial

## 50. 使用 `vimtutor` 学习 Vim

Vim 自带一个交互式教程：

```bash
vimtutor
```

运行后，它会打开一个教学文件，带你练习 Vim 的主要功能。

---

## 51. 为什么推荐 `vimtutor`？

因为 Vim 只看命令表很容易忘。

更有效的方法是：

```text
直接在 Vim 中边看边练。
```

`vimtutor` 会帮助你练习：

- 移动光标
    
- 进入和退出 Insert mode
    
- 删除文本
    
- 复制和粘贴
    
- 搜索和替换
    
- 保存和退出
    

如果以后会偶尔使用 Vim，`vimtutor` 非常值得做一遍。

# 易混点总结

## 1. Vim 打开后不能直接输入文字

因为 Vim 默认进入：

```text
Normal mode
```

如果要输入文字，需要先按：

```vim
i
```

或者：

```vim
a
```

进入 Insert mode。

---

## 2. 保存和退出前要先回到 Normal mode

如果正在 Insert mode 中，直接输入 `:wq` 只是输入文本。

正确流程是：

```text
Esc → :wq → Enter
```

---

## 3. `q` 和 `q!` 不一样

|命令|含义|
|---|---|
|`:q`|退出，如果有未保存修改会失败|
|`:q!`|强制退出，放弃未保存修改|

---

## 4. `p` 和 `P` 的区别

|命令|字符/单词|整行|
|---|---|---|
|`p`|光标后|下一行|
|`P`|光标前|上一行|

---

## 5. 删除在 Vim 中经常等于剪切

例如：

```vim
dd
```

删除当前行后，可以用：

```vim
p
```

把这一行粘贴到别处。

---

## 6. `Ctrl-Z` 不是退出 Vim

在 Vim 中按：

```text
Ctrl-Z
```

是暂停 Vim，把它放到后台。

要回到 Vim，用：

```bash
fg
```

如果想真正退出 Vim，应该使用：

```vim
:q
```

或：

```vim
:wq
```

---

# 本节重点理解

Vim 的核心不是“像普通记事本一样输入文字”，而是：

```text
通过模式和命令快速编辑文本。
```

最基础的生存命令是：

```text
i：进入插入模式
Esc：回到普通模式
:w：保存
:q：退出
:q!：不保存退出
:wq：保存并退出
```

最常用的编辑命令是：

```text
h j k l：移动
x：删除字符
dd：删除行
yy：复制行
p：粘贴
u：撤销
/word：搜索
:%s/foo/bar/g：全文替换
```

对于初学者，最重要的是先形成这个流程：

```text
打开文件 → i 输入内容 → Esc 回普通模式 → :wq 保存退出
```

如果之后会经常在 Linux、SSH、WSL 或 HPC 集群中工作，建议运行：

```bash
vimtutor
```

完整练习一遍 Vim 的基础操作。
