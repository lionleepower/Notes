# Linux Filesystem 文件系统

## 1. 快速概览

Linux 使用一种 **hierarchical filesystem 层级式文件系统**。

也就是说，Linux 中的文件和目录不是随意摆放的，而是组织成一棵树状结构。

这棵树的最顶端是：

```bash
/
````

它叫做：

```text
root directory 根目录
```

所有文件和目录都位于 `/` 之下。

> Linux 文件系统像一棵倒过来的树，`/` 是树根，其他目录都是从它下面分出来的分支。

## 2. Filesystem Hierarchy Standard, FHS

Linux 文件系统的目录结构通常遵循：

```text
FHS = Filesystem Hierarchy Standard
```

FHS 的作用是规定 Linux 系统中主要目录的大致用途。

因此，不同 Linux 发行版虽然细节可能不同，但核心目录结构通常比较相似。

例如，大多数 Linux 系统都会有：

```bash
/bin
/etc
/home
/usr
/var
/tmp
```

这些目录。

---

## 3. Root Directory 根目录

Linux 文件系统的最顶层目录是：

```bash
/
```

注意：

```bash
/
```

不是普通目录名，而是整个 Linux 文件系统的起点。

所有目录都在 `/` 下面，例如：

```bash
/home
/bin
/etc
/usr
/tmp
```

可以理解为：

```text
/
├── bin
├── boot
├── dev
├── etc
├── home
├── usr
├── var
└── tmp
```

---

## 4. Subdirectories 子目录

根目录 `/` 下有很多默认创建的子目录，叫做：

```text
subdirectories
```

这些目录大多在安装 Linux 发行版时自动生成。

它们大致可以分成两类：

### 4.1 系统使用的目录

这些目录主要由系统使用，一般用户不应该随意修改。

例如：

```bash
/bin
/boot
/dev
/etc
/lib
/proc
/sys
/sbin
```

### 4.2 用户可以使用的目录

普通用户主要在自己的 home directory 中工作：

```bash
/home/username
```

例如：

```bash
/home/alice
/home/leyan
```

在自己的 home directory 下，用户通常可以自由创建文件和子目录，不需要管理员权限。

# 5. Linux 根目录下的主要目录

下面是 FHS 中常见的根目录子目录总结。

## 5.1 `/bin`

```bash
/bin
```

全称：

```text
binary
```

作用：

> 存放基本命令的可执行文件。

例如，一些常见 Linux 命令的程序文件可能位于这里。

可以理解为：

```text
/bin = basic command binaries
```

---

## 5.2 `/boot`

```bash
/boot
```

作用：

> 存放启动 Linux 系统所需的文件。

例如：

- boot loader 相关文件
    
- kernel image
    
- 系统启动相关配置
    

普通用户一般不会修改这个目录。

---

## 5.3 `/dev`

```bash
/dev
```

全称：

```text
devices
```

作用：

> 存放设备文件，用于和系统连接的设备进行交互。

Linux 中很多硬件设备会以文件形式出现在 `/dev` 中。

例如：

```bash
/dev/sda
/dev/null
/dev/tty
```

简单理解：

> 在 Linux 中，很多设备也被抽象成文件。

---

## 5.4 `/etc`

```bash
/etc
```

作用：

> 存放系统配置文件。

例如：

- 用户账户相关配置
    
- 服务配置
    
- 网络配置
    
- 软件配置
    

很多 Linux 系统级配置文件都在这里。

常见例子：

```bash
/etc/passwd
/etc/hosts
/etc/ssh/
```

注意：

> `/etc` 很重要，普通用户不应该随意修改其中内容。

---
## 5.5 `/home`

```bash
/home
```

作用：

> 存放普通用户的 home directories。

例如：

```bash
/home/alice
/home/bob
/home/leyan
```

如果用户名是 `leyan`，那么用户的主目录通常是：

```bash
/home/leyan
```

这是普通用户最常工作的地方。

---

## 5.6 `/lib`

```bash
/lib
```

全称：

```text
libraries
```

作用：

> 存放 `/bin` 和 `/sbin` 中程序运行所需的系统共享库。

可以理解为：

```text
/lib = shared libraries needed by essential system binaries
```

程序本身可能在 `/bin`，但运行时需要调用一些库文件，这些库文件可能在 `/lib`。

---

## 5.7 `/media`

```bash
/media
```

作用：

> 临时挂载可移动媒体设备。

例如：

- USB drive
    
- CD-ROM
    
- external disk
    

当插入 U 盘或外接设备时，系统可能会把它挂载到 `/media` 下。

---

## 5.8 `/mnt`

```bash
/mnt
```

作用：

> 临时挂载文件系统。

`/mnt` 和 `/media` 都和 mount 挂载有关。

区别可以简单理解为：

|目录|常见用途|
|---|---|
|`/media`|自动挂载可移动设备|
|`/mnt`|手动临时挂载文件系统|

---

## 5.9 `/opt`

```bash
/opt
```

全称：

```text
optional
```

作用：

> 存放可选的应用软件包。

例如，一些第三方软件可能会安装到：

```bash
/opt/software_name
```

---

## 5.10 `/proc`

```bash
/proc
```

作用：

> 虚拟文件系统 virtual filesystem，用于提供进程和系统信息。

`/proc` 不是普通意义上的硬盘目录，而是系统动态生成的信息接口。

例如：

```bash
/proc/cpuinfo
/proc/meminfo
```

可以查看：

- CPU 信息
    
- 内存信息
    
- 进程信息
    
- kernel 状态
    

---

## 5.11 `/root`

```bash
/root
```

作用：

> root 用户的 home directory。

注意：

```bash
/root
```

和：

```bash
/
```

不是一个概念。

|路径|含义|
|---|---|
|`/`|整个文件系统的根目录|
|`/root`|root 用户的主目录|

普通用户的 home directory 通常在 `/home/username`，但 root 用户的 home directory 是 `/root`。

---

## 5.12 `/run`

```bash
/run
```

作用：

> 存放系统自上次启动以来的运行时变量数据。

例如：

- 当前运行服务的信息
    
- 进程 ID 文件
    
- socket 文件
    
- 临时状态文件
    

这个目录中的内容通常和当前系统运行状态有关。

---

## 5.13 `/sbin`

```bash
/sbin
```

全称：

```text
system binaries
```

作用：

> 存放系统管理相关的可执行文件。

例如：

- 系统维护命令
    
- 管理员命令
    
- 启动和修复系统相关命令
    

和 `/bin` 的区别可以简单理解为：

|目录|主要内容|
|---|---|
|`/bin`|普通基础命令|
|`/sbin`|系统管理命令|

---

## 5.14 `/srv`

```bash
/srv
```

全称：

```text
service
```

作用：

> 存放系统提供的服务相关数据。

例如：

- web server 数据
    
- FTP server 数据
    
- 其他服务数据
    

如果系统作为服务器提供服务，相关数据可能放在 `/srv` 下。

---

## 5.15 `/sys`

```bash
/sys
```

作用：

> 虚拟目录，用于提供系统信息。

和 `/proc` 类似，`/sys` 也不是普通数据目录，而是 kernel 暴露系统信息的接口。

它通常和：

- 设备
    
- 驱动
    
- kernel
    
- 系统硬件状态
    

有关。

---

## 5.16 `/tmp`

```bash
/tmp
```

全称：

```text
temporary
```

作用：

> 存放临时文件。

很多程序会把临时数据放在 `/tmp` 中。

注意：

- `/tmp` 中的文件可能会被系统自动清理
    
- 不应该把重要文件长期放在 `/tmp`
    
- 适合短期中间文件 intermediate files
    

---

## 5.17 `/usr`

```bash
/usr
```

作用：

> 存放所有用户共享的只读用户数据 read-only user data。

这里的 `usr` 历史上和 user 相关，但现在更常被理解为存放大量系统和用户级程序资源的目录。

`/usr` 下有很多重要子目录。

---

### 5.17.1 `/usr/bin`

```bash
/usr/bin
```

作用：

> 存放程序二进制文件 program binaries。

很多常用程序都位于这里。

---

### 5.17.2 `/usr/include`

```bash
/usr/include
```

作用：

> 存放 include files，尤其是 C/C++ 头文件。

例如编译程序时可能会用到这里的头文件。

---

### 5.17.3 `/usr/lib`

```bash
/usr/lib
```

作用：

> 存放 `/usr/bin` 和 `/usr/sbin` 中程序所需的库文件。

---

### 5.17.4 `/usr/local`

```bash
/usr/local
```

作用：

> 存放本地主机 local host 上安装或维护的数据和软件。

例如，管理员手动安装的软件可能位于：

```bash
/usr/local/bin
/usr/local/lib
```

---

### 5.17.5 `/usr/sbin`

```bash
/usr/sbin
```

作用：

> 存放非必要的系统管理二进制文件 non-essential system binaries。

---

### 5.17.6 `/usr/share`

```bash
/usr/share
```

作用：

> 存放共享数据 shared data。

例如：

- 文档 documentation
    
- 图标 icons
    
- 字典数据
    
- 应用程序共享资源
    

---

### 5.17.7 `/usr/src`

```bash
/usr/src
```

作用：

> 存放 kernel source code 和 headers。

也就是内核源码和相关头文件可能位于这里。

---

## 5.18 `/var`

```bash
/var
```

全称：

```text
variable
```

作用：

> 存放经常变化的数据 variable data。

例如：

- log files
    
- cache
    
- mail
    
- spool files
    
- runtime-generated data
    

常见例子：

```bash
/var/log
/var/cache
```

可以理解为：

```text
/var = data that changes over time
```

# 6. 用户日常最常使用哪里？

在 Linux 系统中，大部分普通用户的工作通常都在：

```bash
/home/username
```

也就是自己的 home directory。

例如：

```bash
/home/leyan
```

你通常会在这里：

- 写代码
    
- 保存笔记
    
- 存放数据
    
- 创建项目目录
    
- 下载文件
    
- 运行普通用户程序
    

而系统程序和配置文件通常分布在其他目录，例如：

```bash
/bin
/etc
/usr
/var
```

# 7. 重要区别：`/` 和 `/root`

这两个很容易混淆。

|路径|含义|
|---|---|
|`/`|root directory，整个文件系统的起点|
|`/root`|root user 的 home directory|

所以：

```bash
cd /
```

表示进入整个系统的根目录。

```bash
cd /root
```

表示进入 root 用户的主目录，但普通用户通常没有权限访问。

---

# 8. 重要区别：`/bin`、`/sbin` 和 `/usr/bin`

这几个目录都可能存放可执行文件，但用途略有区别。

|目录|主要内容|
|---|---|
|`/bin`|基础命令 binaries|
|`/sbin`|系统管理命令 system binaries|
|`/usr/bin`|大量普通程序 binaries|
|`/usr/sbin`|非必要系统管理程序|

简单理解：

```text
/bin      = essential user commands
/sbin     = essential system/admin commands
/usr/bin  = most user programs
/usr/sbin = non-essential system/admin commands
```

---

# 9. 重要区别：`/tmp` 和 `/var`

这两个都可能存放变化的数据，但用途不同。

|目录|作用|
|---|---|
|`/tmp`|临时文件，可能被清理|
|`/var`|系统运行中持续变化的数据|

例如：

```bash
/tmp/test_output.txt
```

适合短期临时文件。

```bash
/var/log/syslog
```

适合系统日志这类持续变化的数据。

# 13. 学习这一节时不需要死记硬背

这张目录表不需要全部背下来。

更重要的是理解 Linux 的整体布局：

1. `/` 是所有目录的起点。
    
2. `/home/username` 是普通用户主要工作的地方。
    
3. `/etc` 放系统配置。
    
4. `/bin`、`/usr/bin`、`/sbin` 放程序和命令。
    
5. `/tmp` 放临时文件。
    
6. `/var` 放经常变化的数据。
    
7. `/proc` 和 `/sys` 是虚拟文件系统，用于查看系统状态。
    
8. `/root` 不是根目录，而是 root 用户的主目录。

# 14. 简单例子：查看这些目录

可以使用：

```bash
ls /
```

查看根目录下有哪些内容。

也可以进入自己的 home directory：

```bash
cd ~
pwd
```

查看当前路径：

```bash
pwd
```

如果当前用户是 `alice`，可能会显示：

```bash
/home/alice
```

# 15. 和前面内容的联系

这一节和前面学过的内容关系很密切：

- `cd /`：进入 root directory
    
- `cd ~`：进入 home directory
    
- `ls /`：查看根目录下的子目录
    
- `pwd`：查看当前所在路径
    
- `ssh username@host`：远程登录后通常进入远程系统中的 home directory
    
- `scp file username@host:~/Project`：把文件复制到远程用户 home directory 下的 `Project` 目录
    

所以理解 filesystem 后，很多路径相关命令会更清楚。

```

这一节的核心其实不是背目录，而是建立“Linux 所有东西都挂在 `/` 下面”的路径意识。后面学权限、远程传输、环境变量、软件安装、HPC 作业目录时，这个结构会一直用到。
```
