# Linux Accounts 用户账户

## 1. 快速概览

在 Linux 系统中，用户需要拥有一个 **账户 account** 才能登录系统。账户的作用是让系统能够识别用户、验证用户身份，并决定用户可以访问哪些资源。

一个 Linux 用户账户通常包括：

- 用户名 username
- 密码 password
- 用户 ID UID, User ID
- 组 ID GID, Group ID
- 主目录 home directory
- 默认 shell default shell

简单来说：

> username 用来给人看，UID 用来给系统识别；password 用来验证身份；group 用来管理共享权限；home directory 是用户自己的工作空间。

---

## 2. 用户账户 Account

### 2.1 什么是用户账户？

在 Linux 中，用户账户是用户登录和使用系统的身份信息集合。

一个账户通常包含两类信息：
### 身份信息 Identity Information

包括：

- username
- password
- UID
- GID

这些信息用于让系统识别“你是谁”。

### 资源信息 Resource Information

包括：

- 可访问的磁盘空间
- home directory
- 默认 shell preference

这些信息决定用户可以在哪里工作，以及登录后默认使用什么 shell。

---

## 3. 用户名 Username

每个 Linux 账户都有一个唯一的用户名。

### 3.1 用户名的特点

Linux 用户名通常具有以下特点：

- 唯一 unique
- 通常由字母和数字组成 alphanumeric characters
- 一般至少 3 个字符
- 区分大小写 case-sensitive

例如：

```bash
Apple01
apple01
````

这两个在 Linux 中是不同的用户名。

---

### 3.2 UID：User ID

虽然用户平时看到的是 username，但 Linux 系统内部主要使用 **UID** 来管理用户。

UID 是一个唯一的整数。

例如：

```text
username: leyang
UID: 1001
```

Linux 更倾向于使用 UID，而不是 username，因为：

> 处理数字比处理字符串更高效。

所以在系统内部，Linux 实际上更关心 UID，而不是用户名本身。

一般用户不需要记住自己的 UID，但理解它的存在很重要。

---

## 4. 用户组 Group

Linux 中不仅有用户 account，也有用户组 group。

### 4.1 什么是 Group？

Group 是一组用户的集合，通常用于：

- 共享文件
    
- 共享进程
    
- 管理权限 permissions
    

例如，一个项目组里的多个用户可以属于同一个 group，这样他们就可以共享某些文件。

---

### 4.2 GID：Group ID

每个 group 都有一个对应的数字 ID，叫做：

```text
GID = Group ID
```

每个用户账户都会被分配到一个 **primary group 主组**。

一个用户可以属于多个 group，但只能有一个 primary group。

---

### 4.3 Primary Group 和 Multiple Groups

一个用户：

- 可以属于多个 groups
    
- 只能有一个 primary group
    

例如：

```text
User: alice

Primary group:
- students

Other groups:
- projectA
- lab
- docker
```

这里 `students` 是 alice 的主组，`projectA`、`lab`、`docker` 是她额外加入的组。

---

### 4.4 Group 的用途

Group 主要用于权限管理。

例如：

```text
某个文件只允许 projectA 组的成员读取和修改。
```

那么只要用户属于 `projectA` 这个 group，就可以获得对应权限。

这在多人共用服务器、HPC 集群、实验室服务器中非常常见。

---

## 5. 密码 Password

每个用户名都需要一个密码。

### 5.1 Username 和 Password 的区别

可以这样理解：

|概念|作用|
|---|---|
|username|标识用户 identity|
|password|验证用户 authenticator|

也就是说：

> username 告诉系统“我是谁”，password 证明“我确实是这个人”。

---

## 6. 密码要求 Password Requirements

不同系统对密码的要求可能不同。

一种常见要求是：

- 至少 8 个字符
    
- 至少包含以下 4 类字符中的 3 类：
    

```text
lower-case letters    小写字母
upper-case letters    大写字母
numerical digits      数字
punctuation           标点符号
```

例如，一个相对符合要求的密码可能包含：

```text
大写字母 + 小写字母 + 数字 + 标点符号
```

---

## 7. 密码如何被系统验证？

当用户输入密码时，系统不会直接保存或对比明文密码。

Linux 通常会：

1. 用户输入密码
    
2. 系统对输入的密码进行加密或哈希处理
    
3. 系统将处理后的结果与已存储的加密字符串进行比较
    
4. 如果匹配，则认证成功
    
5. 如果不匹配，则认证失败
    

重点是：

> 操作系统本身也不应该知道你的明文密码。

这是一种更安全的设计。

---

## 8. 为什么不直接保存明文密码？

如果系统直接保存明文密码，那么一旦数据库或系统文件泄露，攻击者就能直接看到所有用户的密码。

而使用加密或哈希后的字符串，即使攻击者拿到了存储内容，也不能直接知道原始密码。

这种机制也广泛用于：

- Linux 服务器
    
- 网站账户系统
    
- 在线服务
    
- 远程登录系统
    

---

## 9. SSH 与远程登录

当你拥有远程 Linux 系统的：

- username
    
- password
    

你就可以通过 **SSH Secure Shell** 登录远程系统。

SSH 是一种安全的远程连接方式。

常见命令形式：

```bash
ssh username@remote_host
```

例如：

```bash
ssh alice@server.example.com
```

如果服务器要求密码登录，系统会提示输入密码。

# 11. 例子：一个 Linux 账户可能长什么样？

假设有一个用户账户：

```text
Username: alice
UID: 1001
Primary Group: students
GID: 100
Home Directory: /home/alice
Default Shell: /bin/bash
```

含义是：

- `alice` 是用户名
    
- `1001` 是系统内部识别 alice 的 UID
    
- `students` 是 alice 的主组
    
- `100` 是 students 组的 GID
    
- `/home/alice` 是 alice 的个人目录
    
- `/bin/bash` 是 alice 登录后默认使用的 shell

# 12. 进一步理解：username 和 UID 的关系

可以把 username 和 UID 理解成：

```text
username = 给人看的名字
UID = 系统真正识别的编号
```

类似于：

```text
学生姓名 = Leyan Li
学生 ID = 123456
```

学校系统可能显示你的名字，但真正管理学籍时更依赖学生 ID。

Linux 也是类似逻辑。

---

# 13. 进一步理解：Group 为什么重要？

如果 Linux 只有单个用户权限，会很难管理多人协作。

例如，有一个项目目录：

```bash
/project/data
```

你希望项目组所有成员都可以访问它，但其他人不能访问。

这时候可以创建一个 group：

```text
projectA
```

然后把相关用户加入这个 group。

这样权限可以设置为：

```text
projectA 组成员可以读写
其他用户不能访问
```

这比给每个用户单独设置权限更方便。

---

# 14. 本节需要记住的重点

1. Linux 用户必须通过 account 登录系统。
    
2. username 是用户的人类可读名称，但 Linux 内部主要使用 UID。
    
3. group 用于组织用户和管理权限。
    
4. 每个用户有一个 primary group，但也可以属于多个 group。
    
5. password 用于身份验证，系统通常不会保存明文密码。
    
6. SSH 可以使用 username 和 password 远程登录 Linux 系统。
