# Remote Connections：使用 SSH 远程连接

## 1. 快速概览

本节主要介绍两个远程操作工具：

- `ssh`：安全远程登录 Linux 系统
- `scp`：安全地在本地和远程系统之间复制文件

它们都基于 **SSH Secure Shell** 协议，用于在不安全网络中安全访问远程资源。

简单来说：

> `ssh` 用来登录远程机器，`scp` 用来复制文件到远程机器或从远程机器复制回来。

---

## 2. SSH 是什么？

**SSH Secure Shell** 是一种安全连接远程资源的方式。

它的作用是：

- 从本地机器连接到远程机器
- 在不安全网络上提供安全通信
- 支持密码 password 或私钥 private key 认证
- 常用于登录 HPC 系统、服务器、远程 Linux 主机

基本形式：

```bash
ssh username@remote_host
````

例如：

```bash
ssh jolo@foo.edu
```

含义是：

- `jolo` 是远程系统上的用户名 username
    
- `foo.edu` 是远程机器的地址 hostname
    
- 如果认证成功，就会登录到远程机器
    

---

## 3. SSH 基本命令

### 3.1 使用用户名和主机名登录

```bash
ssh jolo@foo.edu
```

这条命令会尝试连接远程机器 `foo.edu`，并以用户 `jolo` 的身份登录。

如果登录 HPC 系统，通常需要：

- password 密码
    
- private key 私钥
    
- 或两者结合
    

---

## 4. 使用私钥登录 SSH

除了密码登录，SSH 也经常使用 **key pair 密钥对** 登录。

如果私钥文件名是 `my_key`，命令如下：

```bash
ssh -i my_key jolo@foo.edu
```

其中：

|部分|含义|
|---|---|
|`ssh`|启动 SSH 连接|
|`-i my_key`|指定使用的私钥文件|
|`jolo`|远程用户名|
|`foo.edu`|远程主机名|

这里的 `-i` 表示 identity file，也就是指定身份认证用的私钥文件。

---

## 5. X11 Forwarding 图形界面转发

SSH 还有一个常见功能叫：

```text
X11 forwarding
```

它用于远程运行需要图形界面的程序。

常见选项：

```bash
ssh -X jolo@foo.edu
```

或：

```bash
ssh -Y jolo@foo.edu
```

`-X` 和 `-Y` 都用于开启 X11 forwarding。

适用场景：

- 远程打开 GUI 程序
    
- 运行图形化文本编辑器
    
- 使用需要窗口显示的软件
    
- 在远程服务器上启动图形应用
    

例如，如果你想在远程机器上打开 Emacs GUI、某些可视化程序或图形化工具，就可能需要 X11 forwarding。

更多选项可以参考：

- [ssh man page](https://chatgpt.com/c/6a060502-89e8-8385-92b7-446d02630a5f#)

# Securely Copy with SCP：使用 SCP 安全复制文件

## 8. SCP 是什么？

**SCP Secure Copy Protocol** 是基于 SSH 协议的文件复制工具。

它用于在网络中安全复制文件。

常见用途：

- 从本地复制文件到远程服务器
    
- 从远程服务器复制文件到本地
    
- 从一个远程服务器复制到另一个远程服务器
    
- 递归复制整个目录
    

简单理解：

> `ssh` 是远程登录，`scp` 是远程复制文件。

---

## 9. 复制文件到远程系统

假设本地当前目录下有一个文件：

```text
code.c
```

想把它复制到远程机器 `foo.edu` 上，用户 `jolo` 的 home directory 下的 `Project` 目录中：

```bash
scp code.c jolo@foo.edu:~/Project
```

含义如下：

|部分|含义|
|---|---|
|`scp`|使用 Secure Copy|
|`code.c`|本地要复制的文件|
|`jolo@foo.edu`|远程用户名和主机|
|`:~/Project`|远程目标目录|

这里的 `~` 表示远程用户 `jolo` 的 home directory。

所以：

```bash
jolo@foo.edu:~/Project
```

表示：

```text
远程机器 foo.edu 上，用户 jolo 的 home directory 里的 Project 目录
```

---

## 10. 从远程系统复制文件到本地

假设远程系统中有一个文件：

```text
~/Project/output.txt
```

你想把它复制到本地的 `Results` 目录，并在复制时改名为：

```text
Run12_data.txt
```

命令如下：

```bash
scp jolo@foo.edu:~/Project/output.txt ./Results/Run12_data.txt
```

含义如下：

|部分|含义|
|---|---|
|`jolo@foo.edu:~/Project/output.txt`|远程文件位置|
|`./Results/Run12_data.txt`|本地目标路径和新文件名|

这里的 `./Results/Run12_data.txt` 表示：

```text
当前目录下的 Results 文件夹中的 Run12_data.txt 文件
```

所以这条命令不仅复制文件，还顺便重命名了文件。

---

## 11. SCP 路径格式总结

SCP 的核心格式是：

```bash
scp source destination
```

其中 `source` 和 `destination` 可以是本地路径，也可以是远程路径。

远程路径格式：

```bash
username@hostname:path
```

例如：

```bash
jolo@foo.edu:~/Project/output.txt
```

可以拆解为：

|部分|含义|
|---|---|
|`jolo`|远程用户名|
|`foo.edu`|远程主机|
|`:`|分隔远程主机和远程路径|
|`~/Project/output.txt`|远程文件路径|

注意：

> 远程主机名和路径之间必须有冒号 `:`。

---

## 12. 递归复制目录：`scp -r`

如果要复制整个目录，需要使用 `-r` 选项。

例如，把本地目录 `Project` 复制到远程 home directory：

```bash
scp -r Project jolo@foo.edu:~
```

或者从远程复制整个目录到本地：

```bash
scp -r jolo@foo.edu:~/Project ./Project_backup
```

其中：

```bash
-r
```

表示 recursive，即递归复制整个目录及其内部文件。

更多 SCP 选项可以参考：

- [scp man page](https://chatgpt.com/c/6a060502-89e8-8385-92b7-446d02630a5f#)

# 15. SSH 和 SCP 的区别

|工具|全称|主要用途|
|---|---|---|
|`ssh`|Secure Shell|安全远程登录|
|`scp`|Secure Copy Protocol|安全复制文件|

简单区分：

```text
ssh = 登录远程机器
scp = 复制远程文件
```

例如：

```bash
ssh jolo@foo.edu
```

是登录远程机器。

```bash
scp code.c jolo@foo.edu:~/Project
```

是把文件复制到远程机器。

# 17. 和 HPC 的关系

在 HPC 系统中，SSH 和 SCP 非常常见。

通常你会用：

```bash
ssh username@login-node
```

登录到 HPC 的 login node。

然后用：

```bash
scp file username@login-node:~/some_directory
```

把本地代码、脚本或数据上传到远程系统。

也可以用：

```bash
scp username@login-node:~/results/output.txt ./local_results/
```

把远程运行结果下载回本地。

---

# 18. 大文件传输提醒

如果要复制大文件或大目录，直接使用 `scp` 可能比较慢。

课本后面会介绍：

- File Compression 文件压缩
    
- Directory Compression 目录压缩
    

通常可以先压缩，再传输。

例如：

```bash
tar -czf project.tar.gz Project
scp project.tar.gz jolo@foo.edu:~
```

这样可以减少传输数据量，也更方便传输整个目录。
