# Optional Topics：Linux 补充主题

## 1. 快速概览

本节属于 Linux 的补充内容，主要介绍一些后续使用中非常常见但相对进阶的主题：

- File and Directory Compression 文件与目录压缩
- Symbolic Links 符号链接
- Root and Sudo 管理员权限
- Package Managers 包管理器
- Mounting Storage Volumes or Devices 挂载存储设备

这些内容不一定都需要立刻完全掌握，但它们在实际使用 Linux、HPC、服务器、WSL 或远程系统时会经常遇到。

# 2. File and Directory Compression 文件与目录压缩

## 2.1 什么是压缩和归档？

在 Linux 中，文件和目录压缩通常包含两个概念：

| English Keyword | 中文含义 | 说明 |
|---|---|---|
| Archive | 归档 | 把多个文件或目录打包成一个文件 |
| Compression | 压缩 | 减小文件体积 |

常见流程是：

```text
多个文件/目录 -> tar 打包成一个 archive -> gzip 进一步压缩
```

例如：

```text
file1 file2 file3
        ↓
my_archive.tar
        ↓
my_archive.tar.gz
```

---

## 2.2 `tar` 是什么？

`tar` 的名字来自：

```text
tape archive
```

它最初用于把文件打包到磁带中，现在常用于 Linux 中的文件归档。

`tar` 可以把多个文件或目录打包成一个 `.tar` 文件。

注意：

> `.tar` 本身主要是打包 archive，不一定压缩；`.tar.gz` 或 `.tgz` 才通常表示经过 gzip 压缩。

---

## 2.3 `tar` 常见选项

| Option | 含义 | 作用 |
|---|---|---|
| `-c` | create | 创建归档文件 |
| `-x` | extract | 解包归档文件 |
| `-v` | verbose | 显示详细过程 |
| `-f` | file | 指定归档文件名 |
| `-z` | gzip | 使用 gzip 压缩或解压 |

常见组合：

```bash
tar -cvf
```

表示创建 `.tar` 归档，并显示详细过程。

```bash
tar -xf
```

表示解包 `.tar` 归档，不显示详细过程。

```bash
tar -czvf
```

表示创建 `.tar.gz` 压缩归档。

```bash
tar -xzf
```

表示解压 `.tar.gz` 压缩归档。

---

## 2.4 创建 tar archive

假设有三个文件：

```text
file1
file2
file3
```

要把它们打包成一个 tar 文件：

```bash
tar -cvf my_archive.tar file1 file2 file3
```

输出可能是：

```bash
file1
file2
file3
```

命令拆解：

| 部分 | 含义 |
|---|---|
| `tar` | 使用 tar 工具 |
| `-c` | create，创建归档 |
| `-v` | verbose，显示过程 |
| `-f my_archive.tar` | 指定输出文件名 |
| `file1 file2 file3` | 要打包的文件 |

结果会生成：

```bash
my_archive.tar
```

---

## 2.5 解包 tar archive

如果要解包：

```bash
tar -xf my_archive.tar
```

命令拆解：

| 部分 | 含义 |
|---|---|
| `-x` | extract，解包 |
| `-f my_archive.tar` | 指定要解包的归档文件 |

如果想看到解包过程，可以加 `-v`：

```bash
tar -xvf my_archive.tar
```

---

## 2.6 目录归档

`tar` 默认会递归加入目录中的内容，除非特别指定其他行为。

例如：

```bash
tar -cvf project.tar Project/
```

会把整个 `Project/` 目录打包成：

```bash
project.tar
```

# 3. gzip 和 `.tar.gz`

## 3.1 gzip 是什么？

`gzip` 是 Linux 中常用的压缩工具。

它通常会生成 `.gz` 文件。

例如：

```bash
gzip file
```

会把 `file` 压缩成：

```bash
file.gz
```

如果要压缩目录，需要配合 `-r`：

```bash
gzip -r directory
```

不过实际中，更常见的方式是：

> 先用 `tar` 打包目录，再用 gzip 压缩。

---

## 3.2 创建 `.tar.gz` 文件

可以用 `tar` 的 `-z` 选项直接调用 gzip 压缩。

例如：

```bash
tar -czvf my_archive.tar.gz file1 file2 file3
```

含义是：

- `-c`：创建归档
- `-z`：使用 gzip 压缩
- `-v`：显示过程
- `-f`：指定文件名

生成结果：

```bash
my_archive.tar.gz
```

---

## 3.3 解压 `.tar.gz` 文件

如果有一个压缩归档：

```bash
my_archive.tar.gz
```

可以使用：

```bash
tar -xzf my_archive.tar.gz
```

如果想显示解压过程：

```bash
tar -xzvf my_archive.tar.gz
```

---

## 3.4 `.tgz` 是什么？

`.tgz` 是 `.tar.gz` 的另一种常见扩展名。

也就是说：

```text
archive.tar.gz
```

和：

```text
archive.tgz
```

通常表示类似类型的文件。

都可以用类似命令解压：

```bash
tar -xzf archive.tgz
```

---

## 3.5 常见压缩命令总结

| 任务 | 命令 |
|---|---|
| 创建 `.tar` 文件 | `tar -cvf archive.tar files...` |
| 解包 `.tar` 文件 | `tar -xf archive.tar` |
| 创建 `.tar.gz` 文件 | `tar -czvf archive.tar.gz files...` |
| 解压 `.tar.gz` 文件 | `tar -xzf archive.tar.gz` |
| 创建目录归档 | `tar -czvf project.tar.gz Project/` |
| 解压 `.tgz` 文件 | `tar -xzf archive.tgz` |

# 4. Symbolic Links 符号链接

## 4.1 什么是 Symbolic Link？

**Symbolic link** 是一种特殊文件，它指向文件系统中的另一个文件或目录。

可以理解为：

```text
symbolic link = shortcut / pointer
```

也就是类似 Windows 中的快捷方式。

符号链接本身不保存目标文件内容，而是保存目标文件的位置。

---

## 4.2 Symbolic Link 的用途

Symbolic links 常用于：

- 在多个位置提供同一个文件的入口
- 简化很长的路径
- 方便程序执行
- 管理系统库版本
- 为常用目录创建快捷访问路径

例如，某个文件路径很长：

```bash
data/file/thats/far/away
```

可以创建一个短名字：

```bash
righthere
```

以后通过 `righthere` 就可以访问远处的目标。

---

## 4.3 创建 Symbolic Link

使用：

```bash
ln -s target link_name
```

例如：

```bash
ln -s data/file/thats/far/away righthere
```

含义是：

| 部分 | 含义 |
|---|---|
| `ln` | link command |
| `-s` | 创建 symbolic link |
| `data/file/thats/far/away` | 目标文件 target |
| `righthere` | 创建出的链接名 |

创建后：

```bash
righthere
```

会指向：

```bash
data/file/thats/far/away
```

---

## 4.4 查看 Symbolic Link

可以使用：

```bash
ls -l
```

查看符号链接。

可能显示类似：

```bash
righthere -> data/file/thats/far/away
```

其中 `->` 表示这个文件是一个 symbolic link，指向右边的目标路径。

---

## 4.5 Symbolic Link 的直观理解

假设你有一个很深的目录：

```bash
/home/leyan/course/linux/week5/materials/data/results/output.txt
```

每次访问都很麻烦。

可以创建链接：

```bash
ln -s /home/leyan/course/linux/week5/materials/data/results/output.txt output_shortcut
```

以后可以直接用：

```bash
cat output_shortcut
```

访问原文件内容。

更多 linking files 内容可以查看：

- [ln man page](#)

# 5. Root and Sudo 管理员权限

## 5.1 root user 是什么？

在 Linux 中，`root` 用户是系统管理员账户，拥有最高级别的权限。

它也叫：

```text
superuser
```

root 用户可以执行普通用户无法执行的操作，例如：

- 安装软件包
- 修改系统配置
- 管理其他用户
- 修改系统目录
- 启动或停止系统服务
- 改变文件所有权和权限

---

## 5.2 root 的 UID

root 用户的 UID 是：

```text
0
```

Linux 系统会把任何 UID 为 `0` 的用户当作 root。

也就是说：

```text
UID 0 = root privileges
```

这说明 Linux 内部真正判断 root 权限时，核心依据是 UID，而不仅仅是用户名字符串。

---

## 5.3 root 账户的使用原则

如果你有 root 权限，最佳实践是：

> 只在确实需要管理员权限时使用 root，不要长期用 root 做普通工作。

例如，适合用 root 或 sudo 的场景：

- 安装系统软件
- 修改 `/etc` 下的配置
- 管理用户
- 挂载系统设备
- 修改系统服务

不建议用 root 做的事情：

- 日常写代码
- 编辑普通文档
- 下载普通文件
- 运行不可信脚本

原因是：

> root 权限过高，一条错误命令可能破坏整个系统。

---

## 5.4 `/` 和 `/root` 的区别

这两个非常容易混淆。

| 路径 | 含义 |
|---|---|
| `/` | root directory，整个文件系统的根目录 |
| `/root` | root user 的 home directory |

注意：

```bash
/
```

是整个 Linux 文件系统的起点。

而：

```bash
/root
```

是 root 用户自己的 home directory。

普通用户的 home directory 通常是：

```bash
/home/username
```

root 用户的 home directory 是：

```bash
/root
```

---

## 5.5 sudo 是什么？

`sudo` 允许用户以另一个用户的权限运行命令。

默认情况下，`sudo` 通常是以 root 权限运行命令。

可以理解为：

```text
sudo = temporarily run command with elevated privileges
```

常见形式：

```bash
sudo command
```

例如：

```bash
sudo apt update
```

表示以 root 权限运行：

```bash
apt update
```

使用 `sudo` 时，通常需要输入当前用户自己的密码，而不是 root 密码。

---

## 5.6 sudo 的作用

`sudo` 的好处是：

- 不需要直接登录 root 账户
- 可以临时获得管理员权限
- 更安全
- 便于审计谁执行了管理员命令

例如安装软件：

```bash
sudo apt install package_name
```

修改系统文件：

```bash
sudo nano /etc/hosts
```

---

## 5.7 谁可以使用 sudo？

不是所有用户都能使用 `sudo`。

用户必须拥有 sudo privileges。

有些 Linux 发行版中，第一个创建的用户默认有 sudo 权限。

但很多系统需要管理员手动授予 sudo 权限。

常见方式包括：

- 编辑 `/etc/sudoers`
- 使用 `usermod` 命令
- 把用户加入特定管理组，例如 `sudo` 或 `wheel`

注意：

> `/etc/sudoers` 配置错误可能导致系统管理权限出问题，因此通常使用专门工具 `visudo` 编辑。

# 6. Package Managers 包管理器

## 6.1 什么是 Package？

在 Linux 中，软件通常以 package 的形式管理。

Package 可以理解为：

```text
package = software archive + related metadata/data
```

它通常包含：

- 程序文件
- 配置模板
- 依赖信息
- 版本信息
- 安装脚本
- 文档或相关数据

---

## 6.2 什么是 Package Manager？

**Package manager 包管理器** 是用来管理软件包的工具。

它可以用于：

- 安装软件 install
- 卸载软件 uninstall/remove
- 更新软件 update/upgrade
- 查询软件 search
- 管理依赖 dependencies

通常需要 root 或 sudo 权限才能对系统级软件进行安装和卸载。

---

## 6.3 常见 Package Managers

Linux 不同发行版使用不同的包管理器。

常见包管理器包括：

| Package Manager | 常见系统 | 说明 |
|---|---|---|
| `apt` | Debian / Ubuntu | Debian 推荐的命令行工具 |
| `apt-get` | Debian / Ubuntu | 传统 APT 命令行工具 |
| `aptitude` | Debian / Ubuntu | APT 相关工具 |
| `Synaptic` | Debian / Ubuntu | APT 的图形界面 |
| `dpkg` | Debian / Ubuntu | 底层 `.deb` 包管理工具 |
| `yum` | RHEL / CentOS 旧版本 | RPM 系包管理工具 |
| `pacman` | Arch Linux | Arch 的包管理器 |

APT 相关内容可以参考：

- [APT detailed explanation](#)

---

## 6.4 apt 的常见命令

在 Ubuntu 或 Debian 系统中，常见命令有：

```bash
sudo apt update
```

更新软件源索引。

```bash
sudo apt upgrade
```

升级已安装软件。

```bash
sudo apt install package_name
```

安装软件。

```bash
sudo apt remove package_name
```

卸载软件。

```bash
apt search keyword
```

搜索软件包。

---

## 6.5 Package Manager 和 HPC 的关系

在普通 Linux 电脑或服务器上，你可能使用：

```bash
sudo apt install gcc
```

安装软件。

但在 managed resource，例如 HPC 系统上，普通用户通常不能随便使用 sudo 安装系统软件。

HPC 上常见的软件管理方式是：

```text
Module Utility
```

也就是通过 module 系统加载软件环境。

例如：

```bash
module avail
module load gcc
module load python
```

简单理解：

| 环境 | 常见软件管理方式 |
|---|---|
| 普通 Ubuntu 系统 | `apt` |
| Arch Linux | `pacman` |
| RHEL/CentOS | `yum` / `dnf` |
| HPC 集群 | `module` |

# 7. Mounting Storage Volumes or Devices 挂载存储设备

## 7.1 什么是 Mount？

Linux 文件系统是一棵从 `/` 开始的目录树。

如果要访问另一个设备或存储卷的文件系统，需要把它连接到这棵目录树上的某个位置。

这个过程叫：

```text
mount 挂载
```

被挂载的位置叫：

```text
mount point 挂载点
```

例如，可以把一个外部硬盘挂载到：

```bash
/mnt/disk
```

之后访问：

```bash
/mnt/disk
```

就相当于访问该外部硬盘里的内容。

---

## 7.2 `mount` 命令

`mount` 命令用于把另一个设备的文件系统挂载到目录树中。

基本格式：

```bash
mount -t [type] [device] [path]
```

其中：

| 部分 | 含义 |
|---|---|
| `-t [type]` | 文件系统类型 |
| `[device]` | 设备名称 |
| `[path]` | 挂载点路径 |

例如概念上可以写成：

```bash
mount -t ext4 /dev/sdb1 /mnt/mydisk
```

含义是：

> 把 `/dev/sdb1` 这个设备上的 ext4 文件系统挂载到 `/mnt/mydisk`。

---

## 7.3 查看当前挂载设备

如果直接运行：

```bash
mount
```

不加参数，可以查看当前系统中已经挂载的设备和文件系统。

这在检查存储、外接设备、网络共享时很有用。

---

## 7.4 `umount` 卸载设备

卸载挂载的文件系统使用：

```bash
umount
```

注意命令是：

```bash
umount
```

不是：

```bash
unmount
```

例如：

```bash
umount /mnt/mydisk
```

表示卸载挂载在 `/mnt/mydisk` 的文件系统。

---

## 7.5 fstab 自动挂载

除了手动使用 `mount`，Linux 还可以通过：

```bash
/etc/fstab
```

自动挂载设备。

`fstab` 可以配置系统启动时自动挂载哪些设备或文件系统。

这适合：

- 固定硬盘分区
- 常用网络共享
- 固定存储卷

但修改 `/etc/fstab` 需要谨慎，因为错误配置可能影响系统启动或挂载行为。

---

## 7.6 Network Shares 网络共享

除了本地硬盘、U 盘等设备，Linux 也可以挂载网络共享。

例如：

- NFS
- SMB/CIFS
- 远程存储服务

只要提供合适的 credentials 凭据，就可以连接网络共享。

# 10. 补充理解：tar、gzip、scp 的关系

前面学习过 `scp`，用于在本地和远程系统之间复制文件。

如果要复制很多文件，直接：

```bash
scp -r Project username@host:~
```

虽然可以，但有时不够方便。

更常见的做法是先打包压缩：

```bash
tar -czvf Project.tar.gz Project/
```

然后传输：

```bash
scp Project.tar.gz username@host:~
```

在远程机器上解压：

```bash
tar -xzf Project.tar.gz
```

这样有几个好处：

- 文件数量变少，传输更方便
- 压缩后体积可能更小
- 更适合传输整个项目目录
- 在 HPC 或远程服务器上很常见

---

# 11. 补充理解：为什么不要随便用 root？

root 权限非常强。

例如，普通用户可能无法删除系统目录，但 root 可以。

这意味着 root 可以完成系统管理任务，但也意味着错误命令会很危险。

例如：

```bash
sudo rm -r important_directory
```

如果路径写错，可能删除大量重要文件。

所以实际使用时要遵循：

> 能不用 sudo 就不用 sudo；必须用时再用；用之前检查命令和路径。

# 12. 补充理解：Symbolic Link 的实际用途

假设你有一个课程项目目录：

```bash
/home/leyan/courses/linux/week08/project/data/results
```

路径太长，可以创建一个短链接：

```bash
ln -s /home/leyan/courses/linux/week08/project/data/results results
```

之后可以直接：

```bash
cd results
```

这比每次输入完整路径更方便。

如果原始目录还在，链接就能正常工作。

如果原始目录被删除或移动，链接会失效，这种失效的链接通常叫：

```text
broken symbolic link
```

# 13. 学习这一节时的重点

这一节不需要全部背下来，但需要形成几个实用意识：

1. 多文件传输前，可以先用 `tar` 和 `gzip` 打包压缩。
2. `.tar` 是归档，`.tar.gz` 或 `.tgz` 是压缩归档。
3. `ln -s` 可以创建类似快捷方式的 symbolic link。
4. root 是最高权限用户，UID 是 0。
5. `sudo` 是临时以管理员权限执行命令，不等于长期登录 root。
6. 普通 Linux 系统常用 package manager 安装软件。
7. HPC 系统通常用 module 管理软件环境，而不是让用户用 sudo 安装。
8. mount 是把设备或远程文件系统接入 Linux 的目录树。
9. `umount` 用于卸载，不是 `unmount`。
