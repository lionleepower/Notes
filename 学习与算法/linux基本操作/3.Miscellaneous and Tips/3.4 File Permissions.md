# File Permissions 文件权限

## 1. 快速概览

Linux 是一个 **multi-user environment 多用户环境**。很多用户可以同时运行程序、共享数据、访问同一台系统。

因此，Linux 需要使用 **file permissions 文件权限** 来控制：

- 谁可以读取文件 read
- 谁可以修改文件 write
- 谁可以执行文件 execute

文件权限的主要目的：

> 保护用户文件和系统文件，避免未授权访问、修改或执行。

---

## 2. 三种基本权限

Linux 文件和目录主要有三种权限：

| Permission | Symbol | 中文含义 | 作用 |
|---|---|---|---|
| Read | `r` | 读取权限 | 可以查看文件内容 |
| Write | `w` | 写入权限 | 可以修改文件内容 |
| Execute | `x` | 执行权限 | 可以运行文件，或进入目录 |

简单记忆：

```text
r = read
w = write
x = execute
```

---

## 3. 三类权限对象

Linux 文件权限不仅区分“能做什么”，也区分“谁能做”。

每个文件或目录都有三组权限：

| Category | Symbol | 中文含义 | 说明 |
|---|---|---|---|
| User | `u` | 用户 / 所有者 | 文件 owner 的权限 |
| Group | `g` | 用户组 | 文件所属 group 的权限 |
| Others / World | `o` | 其他人 | 系统中除 owner 和 group 外的所有人 |

有时 `a` 表示 all，也就是：

```text
a = user + group + others
```

---

## 4. 权限显示格式

文件权限通常显示为三组三个字符：

```text
rwx rwx rwx
```

对应关系是：

```text
User  Group Others
rwx   rwx   rwx
```

例如：

```text
rw- r-- r--
```

表示：

| 对象 | 权限 | 含义 |
|---|---|---|
| User | `rw-` | owner 可以读、写，但不能执行 |
| Group | `r--` | group 只能读 |
| Others | `r--` | 其他人只能读 |

所以：

```text
rw-r--r--
```

可以理解为：

> 文件所有者可读写，用户组和其他人只能读取。

---

## 5. 使用 `ls -l` 查看文件权限

可以用：

```bash
ls -l
```

查看文件或目录的详细信息。

例如：

```bash
ls -l $HOME
```

输出示例：

```bash
-rw-r--r-- 1 jdoe jdoe            796631 2009-11-20 14:25 image_data.dat
-rwxrwxr-- 1 jdoe community_group 355    2010-02-18 15:50 my_script.sh
```

---

## 6. `ls -l` 输出格式解释

以这一行为例：

```bash
-rw-r--r-- 1 jdoe jdoe 796631 2009-11-20 14:25 image_data.dat
```

可以拆解为：

| 部分 | 含义 |
|---|---|
| `-rw-r--r--` | 文件类型和权限 |
| `1` | links 链接数量 |
| `jdoe` | 文件 owner |
| `jdoe` | 文件所属 group |
| `796631` | 文件大小 |
| `2009-11-20 14:25` | 最后修改时间 |
| `image_data.dat` | 文件名 |

---

## 7. 第一个字符：文件还是目录？

权限字符串的第一个字符表示文件类型。

例如：

```text
-rw-r--r--
```

第一个字符是 `-`，表示这是一个普通文件。

```text
drwxr-xr-x
```

第一个字符是 `d`，表示这是一个目录 directory。

常见情况：

| 第一个字符 | 含义 |
|---|---|
| `-` | 普通文件 |
| `d` | 目录 directory |

所以权限字符串可以分成：

```text
- rw- r-- r--
```

其中：

```text
第 1 位：文件类型
第 2-4 位：User 权限
第 5-7 位：Group 权限
第 8-10 位：Others 权限
```

---

## 8. 示例 1：普通数据文件权限

```bash
-rw-r--r-- 1 jdoe jdoe 796631 2009-11-20 14:25 image_data.dat
```

权限部分是：

```text
-rw-r--r--
```

拆解如下：

```text
-   rw-   r--   r--
    User  Group Others
```

含义：

| 对象 | 权限 | 说明 |
|---|---|---|
| User `jdoe` | `rw-` | 可以 read 和 write，不能 execute |
| Group `jdoe` | `r--` | 只能 read |
| Others | `r--` | 只能 read |

所以 `image_data.dat` 的权限是：

> owner 可以读写，group 和 others 只能读取。

---

## 9. 示例 2：脚本文件权限

```bash
-rwxrwxr-- 1 jdoe community_group 355 2010-02-18 15:50 my_script.sh
```

权限部分是：

```text
-rwxrwxr--
```

拆解如下：

```text
-   rwx   rwx   r--
    User  Group Others
```

含义：

| 对象 | 权限 | 说明 |
|---|---|---|
| User `jdoe` | `rwx` | 可以读、写、执行 |
| Group `community_group` | `rwx` | 可以读、写、执行 |
| Others | `r--` | 只能读 |

所以：

> `jdoe` 和 `community_group` 组内成员可以读、写、执行这个脚本；其他人只能读取。

---

# 10. 修改权限：`chmod`

Linux 使用：

```bash
chmod
```

修改文件或目录权限。

`chmod` 全称可以理解为：

```text
change mode
```

基本作用：

> 改变文件或目录的 permission bits。

常见形式有两种：

1. symbolic mode 符号模式
2. numeric mode / octal mode 数字模式 / 八进制模式

如果要递归修改整个目录及其内部文件，可以使用：

```bash
chmod -R
```

其中：

```text
-R = recursive
```

---

# 11. Symbolic Mode 符号模式

## 11.1 基本语法

```bash
chmod [references][operator][modes] file
```

可以拆成三部分：

```text
references = 修改谁的权限
operator   = 如何修改
modes      = 修改什么权限
```

---

## 11.2 references：修改谁？

| Symbol | Meaning | 中文含义 |
|---|---|---|
| `u` | user | 文件所有者 |
| `g` | group | 文件所属组 |
| `o` | others | 其他人 |
| `a` | all | 所有人，包括 u、g、o |

---

## 11.3 operator：如何修改？

| Symbol | Meaning | 中文含义 |
|---|---|---|
| `+` | add | 添加权限 |
| `-` | remove | 移除权限 |
| `=` | set exactly | 精确设置权限 |

例如：

```bash
chmod u+x my_script.sh
```

表示给 owner 添加 execute 权限。

```bash
chmod g-w data.txt
```

表示移除 group 的 write 权限。

```bash
chmod o= file.txt
```

表示把 others 的权限设置为空。

---

## 11.4 modes：修改什么权限？

| Symbol | Meaning | 中文含义 |
|---|---|---|
| `r` | read | 读取 |
| `w` | write | 写入 |
| `x` | execute | 执行 |

---

# 12. Symbolic Mode 示例

## 12.1 给 owner 添加读、写、执行权限

```bash
chmod u+rwx my_script.sh
```

含义：

```text
u    = user / owner
+    = add
rwx  = read + write + execute
```

也就是：

> 给文件所有者添加读、写、执行权限。

执行后查看：

```bash
ls -l my_script.sh
```

可能显示：

```bash
-rwx------ 1 jdoe community_group 355 2010-02-18 15:50 my_script.sh
```

这表示：

| 对象 | 权限 |
|---|---|
| User | `rwx` |
| Group | `---` |
| Others | `---` |

---

## 12.2 让脚本可以被 owner 执行

常见用法：

```bash
chmod u+x my_script.sh
```

这会给 owner 添加 execute 权限。

然后可以运行脚本：

```bash
./my_script.sh
```

注意：

```bash
./my_script.sh
```

表示运行当前目录下的 `my_script.sh`。

---

# 13. 运行脚本的两种方式

## 13.1 方式一：作为可执行脚本运行

如果脚本有 execute 权限，可以使用：

```bash
./my_script.sh
```

这种方式会在新的 shell 环境中运行脚本。

特点：

> 脚本内部定义或修改的变量，一般不会影响当前 shell 环境。

---

## 13.2 方式二：使用 `source` 运行脚本

也可以使用：

```bash
source my_script.sh
```

这种情况下，脚本不一定需要 execute 权限。

但是要注意：

> `source` 会把脚本中的命令当作是在当前 shell 中直接输入。

因此，脚本中定义或修改的变量会保留在当前 shell 环境中。

例如：

```bash
source my_script.sh
```

可能会改变当前 shell 的环境变量、工作目录或其他状态。

---

## 13.3 `./script.sh` 和 `source script.sh` 的区别

| 方式 | 是否需要 execute 权限 | 是否影响当前 shell |
|---|---|---|
| `./my_script.sh` | 需要 | 通常不会 |
| `source my_script.sh` | 不需要 | 会 |

简单理解：

```text
./script.sh  = 在子 shell 中运行
source file  = 在当前 shell 中运行
```

---

# 14. Numeric Mode 数字模式

## 14.1 基本思想

Numeric mode 使用数字表示权限。

每一类用户的权限用一个数字表示：

| 权限 | 数值 |
|---|---|
| read `r` | 4 |
| write `w` | 2 |
| execute `x` | 1 |

通过相加得到最终权限值。

---

## 14.2 常见数字含义

| 数字 | 权限 | 含义 |
|---|---|---|
| `0` | `---` | 无权限 |
| `1` | `--x` | 只能执行 |
| `2` | `-w-` | 只能写 |
| `3` | `-wx` | 写 + 执行 |
| `4` | `r--` | 只能读 |
| `5` | `r-x` | 读 + 执行 |
| `6` | `rw-` | 读 + 写 |
| `7` | `rwx` | 读 + 写 + 执行 |

记忆方式：

```text
r = 4
w = 2
x = 1
```

所以：

```text
rw- = 4 + 2 = 6
r-x = 4 + 1 = 5
rwx = 4 + 2 + 1 = 7
--- = 0
```

---

## 14.3 三位数字分别对应谁？

Numeric mode 通常使用三位数字：

```text
750
```

对应：

```text
User Group Others
 7     5     0
```

也就是：

| 位置 | 对象 |
|---|---|
| 第一位 | User / owner |
| 第二位 | Group |
| 第三位 | Others / World |

注意：有时权限可以有第四位数字，但这属于更高级内容，本节只需要掌握常见三位模式。

---

# 15. Numeric Mode 示例：`chmod 750`

假设你有一个可执行文件，希望：

- owner 可以读、写、执行
- group 可以读和执行
- others 没有任何权限

那么权限应该是：

| 对象 | 权限 | 数字 |
|---|---|---|
| User | `rwx` | `7` |
| Group | `r-x` | `5` |
| Others | `---` | `0` |

所以完整数字是：

```text
750
```

命令：

```bash
chmod 750 my_script.sh
```

执行前可能是：

```bash
rw-r--r-- 1 jdoe community_group 355 2010-02-18 15:50 my_script.sh
```

执行后：

```bash
-rwxr-x--- 1 jdoe community_group 355 2010-02-18 15:50 my_script.sh
```

拆解：

```text
-   rwx   r-x   ---
    User  Group Others
```

含义：

> owner 可以读写执行，group 可以读和执行，其他人没有权限。

---

# 16. 常见 chmod 数字权限

| 命令 | 权限 | 常见含义 |
|---|---|---|
| `chmod 600 file` | `rw-------` | 只有 owner 可读写 |
| `chmod 644 file` | `rw-r--r--` | owner 可读写，其他人只读 |
| `chmod 700 file` | `rwx------` | 只有 owner 可读写执行 |
| `chmod 750 file` | `rwxr-x---` | owner 全权限，group 可读执行，others 无权限 |
| `chmod 755 file` | `rwxr-xr-x` | owner 全权限，其他人可读执行 |
| `chmod 777 file` | `rwxrwxrwx` | 所有人都有全部权限，不推荐随便使用 |

特别注意：

```bash
chmod 777 file
```

虽然方便，但通常不安全，因为所有人都可以读、写、执行。

---

# 17. 文件权限和目录权限的区别

同样是 `r`、`w`、`x`，对文件和目录的含义略有不同。

## 17.1 对文件而言

| 权限 | 对文件的含义 |
|---|---|
| `r` | 可以读取文件内容 |
| `w` | 可以修改文件内容 |
| `x` | 可以执行文件，例如运行脚本或程序 |

---

## 17.2 对目录而言

| 权限 | 对目录的含义 |
|---|---|
| `r` | 可以列出目录内容，例如 `ls` |
| `w` | 可以在目录中创建、删除、重命名文件 |
| `x` | 可以进入目录，例如 `cd`，也可以访问目录内文件路径 |

目录的 `x` 特别重要。

如果一个目录没有 execute 权限，即使你知道里面有文件，也可能无法进入或访问。

---

# 18. 递归修改权限：`chmod -R`

如果要修改一个目录以及目录中所有内容，可以使用：

```bash
chmod -R 755 my_directory
```

或：

```bash
chmod -R u+rw my_directory
```

其中：

```text
-R = recursive
```

表示递归应用到目录下的所有文件和子目录。

注意：

> `chmod -R` 要谨慎使用，因为它会批量修改很多文件和目录的权限。

尤其不要随便对系统目录运行：

```bash
chmod -R 777 /
```

这类命令非常危险，可能破坏系统安全和正常运行。

# 19. 本节核心概念总结

| English Keyword | 中文含义 | 说明 |
|---|---|---|
| File permissions | 文件权限 | 控制谁可以读、写、执行文件 |
| Read | 读取权限 | 符号为 `r` |
| Write | 写入权限 | 符号为 `w` |
| Execute | 执行权限 | 符号为 `x` |
| User / Owner | 所有者 | 文件拥有者的权限 |
| Group | 用户组 | 文件所属组的权限 |
| Others / World | 其他人 | 系统中其他所有用户 |
| `ls -l` | 长列表显示 | 查看文件详细信息和权限 |
| `chmod` | change mode | 修改文件或目录权限 |
| Symbolic mode | 符号模式 | 用 `u/g/o/a + - = r/w/x` 修改权限 |
| Numeric mode | 数字模式 | 用八进制数字表示权限 |
| Recursive | 递归 | 应用到目录及其所有内容 |
| `source` | 在当前 shell 中执行 | 不一定需要 execute 权限，但会影响当前环境 |

查看权限：

```bash

```

```bash

```

给 owner 添加执行权限：

```bash
chmod u+x my_script.sh
```

运行当前目录下的脚本：


使用 source 在当前 shell 中运行脚本：

```bash

chmod -R 755 my_directory
```

# 21. 学习这一节时最重要的理解

文件权限可以用一句话概括：

> Linux 文件权限 = 谁 User/Group/Others + 能做什么 Read/Write/Execute。

例如：

```bash
-rwxr-x---
```

可以拆成：

```text
-   rwx   r-x   ---
    User  Group Others
```

也就是：

- `-`：普通文件
- `rwx`：owner 可以读、写、执行
- `r-x`：group 可以读、执行
- `---`：others 没有权限

掌握这个拆解方法后，大部分 Linux 权限问题都能看懂。
