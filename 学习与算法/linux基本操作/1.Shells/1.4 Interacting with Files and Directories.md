# Interacting with Files and Directories

## 1. 本节快速概览

这一节主要介绍如何在 Linux shell 中操作文件和目录，包括：

- 创建目录
- 移动文件或目录
- 重命名文件
- 复制文件或目录
- 删除文件或目录
- 创建空文件或更新时间戳

核心命令如下：

| 命令 | 英文关键词 | 作用 |
|---|---|---|
| `mkdir` | make directory | 创建新目录 |
| `mv` | move | 移动文件或目录，也可以重命名 |
| `cp` | copy | 复制文件或目录 |
| `rm` | remove | 删除文件或目录 |
| `rmdir` | remove directory | 删除空目录 |
| `touch` | touch file | 创建空文件或更新时间戳 |

---

## 2. `mkdir` 命令

`mkdir` 的全称是：

```text
make directory
````

作用是创建一个新目录。

基本格式：

```bash
mkdir <directory_name>
```

例如：

```bash
mkdir Newdir
```

这会在当前目录下创建一个名为 `Newdir` 的新目录。

---

## 3. `mkdir` 示例

假设当前目录中还没有 `Newdir`，执行：

```bash
mkdir Newdir
```

然后可以用：

```bash
ls
```

检查目录是否创建成功。

可能看到：

```text
Newdir
```

---

## 4. 创建多级目录

有时候想创建嵌套目录，例如：

```text
project/src/code
```

如果直接执行：

```bash
mkdir project/src/code
```

但 `project` 或 `src` 不存在，可能会报错。

可以使用：

```bash
mkdir -p project/src/code
```

其中：

```text
-p = parents
```

意思是如果上级目录不存在，就一并创建。

---

## 5. `mv` 命令

`mv` 的全称是：

```text
move
```

作用是移动文件、目录，或者对文件和目录进行重命名。

基本格式：

```bash
mv <source> <destination>
```

例如：

```bash
mv file1 Newdir
```

意思是把 `file1` 移动到 `Newdir` 目录中。

执行后，文件位置会从当前目录变为：

```text
Newdir/file1
```

---

## 6. 使用 `mv` 移动文件

假设当前目录中有：

```text
file1
Newdir/
```

执行：

```bash
mv file1 Newdir
```

之后可以检查：

```bash
ls
```

当前目录可能只剩：

```text
Newdir
```

再查看 `Newdir`：

```bash
ls Newdir
```

可能看到：

```text
file1
```

这说明 `file1` 已经被移动到 `Newdir` 中。

---

## 7. 使用 `mv` 重命名文件

`mv` 也可以用来重命名文件。

例如：

```bash
mv file1 file2
```

意思是把 `file1` 改名为 `file2`。

注意：Linux 中“重命名”和“移动”本质上都可以看作改变文件路径，所以都使用 `mv`。

---

## ==8. 使用 `mv` 重命名目录==

不仅文件可以重命名，目录也可以。

例如：

```bash
mv OldDir NewDir
```

意思是把目录 `OldDir` 改名为 `NewDir`。

---

## 9. 使用 wildcard 移动文件

可以使用 wildcard `*` 批量移动某类文件。

例如：

```bash
mv *.c ../CodeDir
```

意思是：

- `*.c` 匹配当前目录下所有以 `.c` 结尾的文件
    
- `../CodeDir` 表示上一级目录中的 `CodeDir` 目录
    

所以这条命令会把当前目录下所有 C source files 移动到：

```text
../CodeDir
```

---

## 10. `mv *.c ../CodeDir` 的路径解释

假设当前目录是：

```text
/home/leyan/project/src
```

那么：

```bash
mv *.c ../CodeDir
```

其中：

```text
..
```

表示上一级目录：

```text
/home/leyan/project
```

所以目标目录是：

```text
/home/leyan/project/CodeDir
```

如果当前目录下有：

```text
main.c
utils.c
test.txt
```

执行后：

```text
main.c
utils.c
```

会被移动到 `../CodeDir`。

`test.txt` 不会被移动，因为它不匹配 `*.c`。

---

## 11. 移动后如何检查

教材提醒：移动文件后，可以使用 `ls` 验证。

例如：

```bash
mv file1 Newdir
ls
ls Newdir
```

或者：

```bash
mv *.c ../CodeDir
ls ../CodeDir
```

建议养成习惯：执行 `mv` 之后用 `ls` 检查目标目录。

---

## 12. `cp` 命令

`cp` 的全称是：

```text
copy
```

作用是复制文件或目录。

基本格式：

```bash
cp <source> <destination>
```

例如：

```bash
cp file1 ~/
```

意思是把 `file1` 复制到 home directory。

这里：

```text
~/
```

表示当前用户的 home directory。

---

## 13. `cp` 和 `mv` 的区别

|命令|操作后原文件是否还在原位置|作用|
|---|---|---|
|`mv`|不在原位置，移动到新位置|移动或重命名|
|`cp`|仍然保留在原位置|复制|

例如：

```bash
mv file1 Newdir
```

执行后，原目录下没有 `file1`，它被移动到了 `Newdir`。

而：

```bash
cp file1 Newdir
```

执行后，原目录下还有 `file1`，`Newdir` 中也有一个副本。

---

## 14. 复制并改名

可以在复制时给副本一个新名字。

例如：

```bash
cp file1 file1_copy
```

意思是：

- 原文件：`file1`
    
- 复制出的新文件：`file1_copy`
    

执行后，当前目录中会有：

```text
file1
file1_copy
```

---

## 15. 复制到其他目录并改名

也可以复制到其他目录并改名。

例如：

```bash
cp file1 Newdir/file1_backup
```

意思是把 `file1` 复制到 `Newdir` 中，并命名为 `file1_backup`。

---

## 16. 复制目录：`cp -r`

如果要复制目录，需要使用 `-r` 选项。

```bash
cp -r Test1 ~/testresults
```

其中：

```text
-r = recursive
```

意思是递归复制目录及其内部内容。

---

## 17. 什么是 recursive？

**Recursive** 可以理解为“递归地处理整个目录树”。

如果目录 `Test1` 中有：

```text
Test1/
├── file1.txt
├── file2.txt
└── subdir/
    └── file3.txt
```

那么：

```bash
cp -r Test1 ~/testresults
```

会复制：

- `Test1` 本身
    
- `Test1/file1.txt`
    
- `Test1/file2.txt`
    
- `Test1/subdir`
    
- `Test1/subdir/file3.txt`
    

也就是目录中的所有内容都会被复制。

---

## 18. 教材中 `cp -r Test1 ~/testresults` 的解释

命令：

```bash
cp -r Test1 ~/testresults
```

含义是：

- `Test1` 是源目录
    
- `~/testresults` 是目标目录
    
- `-r` 表示递归复制
    

教材中提到：在这个例子中，source 和 destination 都是目录，并且目标目录应当已经存在。

也就是说，`~/testresults` 最好提前创建好：

```bash
mkdir ~/testresults
cp -r Test1 ~/testresults
```

执行后目录结构可能是：

```text
~/testresults/Test1/
```

---

## 19. `rm` 命令

`rm` 的全称是：

```text
remove
```

作用是删除文件或目录。

基本格式：

```bash
rm <file>
```

例如：

```bash
rm file1
```

意思是删除 `file1`。

---

## 20. `rm` 的重要警告

Linux 中的 `rm` 非常危险。

原因是：

> Linux 通常没有像 Windows 或 macOS 那样的 Trash Bin / Recycle Bin。

执行：

```bash
rm file1
```

之后，文件通常不会进入回收站，而是直接从文件系统中删除。

如果没有备份，恢复会非常困难，甚至不可能。

所以使用 `rm` 前一定要确认路径和文件名。

---

## 21. 删除目录：`rm -r`

如果要删除目录及其内部所有内容，可以使用：

```bash
rm -r Junk
```

其中：

```text
-r = recursive
```

意思是递归删除。

如果 `Junk` 目录中有很多文件和子目录，它们都会被删除。

---

## 22. `rm -r` 示例

假设有目录：

```text
Junk/
├── old.txt
├── temp.log
└── subdir/
    └── useless.txt
```

执行：

```bash
rm -r Junk
```

会删除整个 `Junk` 目录，包括其中所有文件和子目录。

---

## 23. 避免滥用 wildcard `*`

教材特别提醒：

> 避免在 `rm` 中随便使用 wildcard `*`。

例如：

```bash
rm -r *
```

这条命令非常危险。

它的含义是：

- 删除当前目录下所有匹配 `*` 的文件和目录
    
- `-r` 表示目录也递归删除
    

也就是说，它会删除当前目录下几乎所有非隐藏文件和子目录。

---

## 24. 为什么 `rm -r *` 危险？

假设当前目录是：

```text
/home/leyan/project
```

其中有：

```text
main.c
README.md
results/
src/
```

执行：

```bash
rm -r *
```

会删除：

```text
main.c
README.md
results/
src/
```

这可能会导致整个项目丢失。

如果你在错误目录中运行它，后果会很严重。

执行删除命令前建议先使用：

```bash
pwd
ls
```

确认自己在哪个目录、要删除什么内容。

---

## 25. 更安全的删除习惯

在使用危险删除命令前，可以先用 `ls` 测试 wildcard 会匹配什么。

例如想删除所有 `.out` 文件，不要一上来就写：

```bash
rm *.out
```

可以先写：

```bash
ls *.out
```

确认列出的文件确实都是想删除的，再执行：

```bash
rm *.out
```

对于目录也一样。

先看：

```bash
ls Junk
```

确认后再删：

```bash
rm -r Junk
```

---

## 26. 删除空目录：`rmdir`

如果只想删除一个空目录，可以使用：

```bash
rmdir <directory>
```

例如：

```bash
rmdir EmptyDir
```

`rmdir` 只能删除空目录。

如果目录中还有文件或子目录，它会失败。

这比 `rm -r` 更安全，因为它不会递归删除目录内容。

---

## 27. `rmdir` 和 `rm -r` 对比

|命令|可以删除非空目录吗|危险程度|
|---|---|---|
|`rmdir EmptyDir`|不可以|较低|
|`rm -r Dir`|可以|较高|

如果只是删除空目录，优先使用：

```bash
rmdir EmptyDir
```

如果确认要删除目录及其所有内容，再使用：

```bash
rm -r Dir
```

---

## 28. `touch` 命令

`touch` 的作用有两个：

1. 修改文件的 modification timestamp
    
2. 如果文件不存在，则创建一个空文件
    

基本格式：

```bash
touch <filename>
```

例如：

```bash
touch newfile.txt
```

如果 `newfile.txt` 不存在，它会创建一个空文件。

如果 `newfile.txt` 已经存在，它不会修改文件内容，只会更新文件的修改时间。

---

## 29. 使用 `touch` 创建空文件

例如：

```bash
touch notes.txt
```

然后用：

```bash
ls
```

可能看到：

```text
notes.txt
```

也可以使用：

```bash
ls -l notes.txt
```

查看它的大小和修改时间。

空文件大小通常是 `0` bytes。

---

## 30. 使用 `touch` 更新时间戳

假设文件 `main.c` 已经存在。

执行：

```bash
touch main.c
```

不会改变 `main.c` 的内容，但会更新它的 modification time。

这在某些构建系统中有用，因为 `make` 等工具会根据文件修改时间判断是否需要重新编译。

---

## 31. 文件操作命令总结

|命令|作用|示例|
|---|---|---|
|`mkdir Newdir`|创建目录|创建 `Newdir`|
|`mv file1 Newdir`|移动文件|把 `file1` 移到 `Newdir`|
|`mv file1 file2`|重命名文件|把 `file1` 改名为 `file2`|
|`mv *.c ../CodeDir`|批量移动文件|移动所有 `.c` 文件|
|`cp file1 ~/`|复制文件到 home directory|复制 `file1`|
|`cp file1 file1_copy`|复制并改名|创建 `file1_copy`|
|`cp -r Test1 ~/testresults`|递归复制目录|复制整个 `Test1` 目录|
|`rm file1`|删除文件|删除 `file1`|
|`rm -r Junk`|删除目录及其内容|删除 `Junk`|
|`rmdir EmptyDir`|删除空目录|删除空目录|
|`touch file.txt`|创建空文件或更新时间戳|创建或更新 `file.txt`|

## 33. 一个完整小例子

目标：创建一个项目目录，并在里面创建、复制、移动、删除文件。

```bash
mkdir linux_test
cd linux_test
touch file1
cp file1 file1_copy
mkdir Newdir
mv file1 Newdir
ls

rm file1_copy
```

执行过程解释：

1. `mkdir linux_test`：创建目录
    
2. `cd linux_test`：进入目录
    
3. `touch file1`：创建空文件
    
4. `cp file1 file1_copy`：复制文件
    
5. `mkdir Newdir`：创建新目录
    
6. `mv file1 Newdir`：移动文件
    
7. `ls`：查看当前目录
    
8. `ls Newdir`：查看 `Newdir` 目录
    
9. `rm file1_copy`：删除副本文件

## 34. HPC / Slurm 场景中的例子

在 HPC 作业目录中，经常会有源代码、脚本和输出文件。

例如：

```text

results.txt
```

可以创建不同目录分类存放：

```bash
mkdir CodeDir

```

```bash
mv *.c CodeDir
```

```bash
mv *.slurm Scripts
```

```bash
mv *.out Results
```

```bash
cp Results/results.txt Results/results_backup.txt
```

删除临时输出文件前，先查看：

```bash
ls *.out
```

确认后再删除：

```bash
rm *.out
```

## 35. 易错点

### 1. `mv` 会改变原文件位置

执行：

```bash
mv file1 Newdir
```

之后，当前目录下不再有 `file1`。

如果只是想保留原文件并创建副本，应该使用：

```bash
cp file1 Newdir
```

---

### 2. `cp` 复制目录时需要 `-r`

错误写法：

```bash
cp Test1 ~/testresults
```

如果 `Test1` 是目录，这通常会失败。

正确写法：

```bash
cp -r Test1 ~/testresults
```

---

### 3. `rm` 删除后通常不能轻易恢复

执行：

```bash
rm file1
```

不是放到回收站，而是直接删除。

所以在删除前要确认：

```bash
pwd
ls
```

---

### 4. `rm -r *` 非常危险

```bash
rm -r *
```

会递归删除当前目录下的文件和目录。

如果当前目录选错，可能造成严重损失。

---

### 5. wildcard 先用 `ls` 检查

删除或移动前，可以先查看 wildcard 匹配了什么：

```bash
ls *.c
```

确认后再执行：

```bash
mv *.c CodeDir
```

或者：

```bash
rm *.out
```

---

### 6. `rmdir` 只能删除空目录

如果目录非空：

```bash
rmdir Junk
```

会失败。

要删除非空目录需要：

```bash
rm -r Junk
```

但要非常谨慎。

## 37. 推荐安全操作习惯

在 Linux 中操作文件时，建议遵循：

```bash
pwd
ls -lha
```

先确认自己在哪、目录里有什么。

在使用 `rm` 前尤其要谨慎：

```bash
ls <target>
rm <target>
```

在使用 wildcard 前先检查：

```bash
ls *.c
mv *.c CodeDir
```

或者：

```bash
ls *.out
rm *.out
```
