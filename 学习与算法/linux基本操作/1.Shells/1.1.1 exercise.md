# Exercise: Working with the Shell

## 1. 本节目标

这一节主要练习几个基础 shell 命令，包括：

- 查看 `$PATH` 环境变量；
- 查看系统中可用的 shells；
- 查看系统当前日期和时间；
- 查看当前用户正在运行的 processes。

这些命令虽然简单，但都是 Linux 命令行入门中非常重要的基础操作。

---

## 2. Display `$PATH`

### 命令

```bash
echo $PATH
````

示例输出：

```bash
/usr/local/bin:/usr/bin:/bin
```

---

## 3. 什么是 `$PATH`？

`$PATH` 是一个 **environment variable**，即环境变量。

它保存了一组目录路径，shell 会在这些目录中寻找可执行程序。

例如，当你输入：

```bash
ls
```

shell 不会凭空知道 `ls` 在哪里。  
它会根据 `$PATH` 中列出的目录依次查找，例如：

```text
/usr/local/bin
/usr/bin
/bin
```

如果在这些目录中找到了 `ls`，shell 就会执行它。

---

## 4. `$PATH` 的作用

`$PATH` 的作用可以总结为：

```text
让用户可以直接运行某些 executable，而不需要输入完整路径。
```

例如，如果 `python` 位于：

```bash
/usr/bin/python
```

并且 `/usr/bin` 在 `$PATH` 中，那么你可以直接输入：

```bash
python
```

而不需要输入：

```bash
/usr/bin/python
```

---

## 5. “in the path” 的含义

如果某个 executable 所在的目录被包含在 `$PATH` 中，那么这个程序就可以说是：

```text
in the path
```

意思是当前 shell 可以直接找到并运行它。

例如：

```bash
/usr/bin/gcc
```

如果 `/usr/bin` 在 `$PATH` 中，那么你可以直接运行：

```bash
gcc
```

---

## 6. `$PATH` 中路径的分隔方式

在 Linux 中，`$PATH` 中多个目录之间使用冒号 `:` 分隔。

例如：

```bash
/usr/local/bin:/usr/bin:/bin
```

可以拆解为：

```text
/usr/local/bin
/usr/bin
/bin
```

shell 会按照顺序搜索这些目录。

---

## 7. 为什么自己写的程序常常要用 `./program`？

假设当前目录下有一个可执行文件：

```bash
program
```

你可能不能直接运行：

```bash
program
```

而需要运行：

```bash
./program
```

原因是：

```text
当前目录通常不在 $PATH 中。
```

`./` 表示 **current directory**，也就是当前目录。

所以：

```bash
./program
```

意思是：

```text
运行当前目录下名为 program 的可执行文件。
```

### 命令

cat /etc/shells
```

/bin/rksh
/bin/zsh
```

---

## 9. `cat` 命令

`cat` 是 **concatenate** 的缩写。

它是一个标准 Linux 工具，常用于把文件内容打印到 shell 输出中。

基本格式是：

```bash
cat <filename>
```

例如：

```bash
cat notes.txt
```

会把 `notes.txt` 的内容显示在终端中。

在本节中：

```bash
cat /etc/shells
```

表示显示 `/etc/shells` 这个文件的内容。

---

## 10. `/etc/shells` 是什么？

`/etc/shells` 是一个系统配置文件，里面列出了当前系统中可用的 shell。

其中：

```text
/etc/
```

是保存系统配置文件的目录。

```text
shells
```

是文件名。

所以：

```bash
/etc/shells
```

是该文件的完整路径。

---

## 11. `/etc/shells` 输出内容的含义

示例：

```bash
/bin/bash
/bin/zsh
/bin/csh
/bin/tcsh
```

这些是不同 shell 程序的路径。

例如：

|路径|含义|
|---|---|
|`/bin/sh`|Bourne shell 或兼容 shell|
|`/bin/bash`|Bash shell|
|`/bin/tcsh`|Tcsh shell|
|`/bin/csh`|C shell|
|`/bin/zsh`|Z shell|

不同 Linux 系统中可用的 shell 可能不同。

---

## 12. `/sbin/nologin` 是什么？

在输出中可能会看到：

```bash
/sbin/nologin
```

或：

```bash
/usr/sbin/nologin
```

它不是普通交互式 shell，而是用于禁止某些账户登录的程序。

例如，某些系统服务账户不需要用户登录。  
这些账户可能会使用 `nologin` 作为 shell。

可以理解为：

```text
nologin 表示该账户不能用于正常登录 shell。
```

## 13. 查看系统当前日期和时间

### 命令

```bash
date
```

示例输出：

```bash
Fri Nov  9 19:23:23 CST 2018
```

---

## 14. `date` 命令

`date` 命令用于显示系统当前日期和时间。

输出通常包含：

|部分|含义|
|---|---|
|`Fri`|星期五|
|`Nov`|11 月|
|`9`|日期|
|`19:23:23`|当前时间|
|`CST`|时区|
|`2018`|年份|

不同系统的输出格式可能略有不同，取决于系统语言、地区和时区设置。

---

## 15. `date` 的常见用途

`date` 不只是用来查看当前时间，也经常用于脚本中记录运行时间。

例如：

```bash
echo "Job started at:"
date
```

在 HPC 作业脚本中，可以用它记录程序开始和结束的时间：

```bash
echo "Start time:"
date

srun ./program

echo "End time:"
date
```

这样可以帮助你查看程序大概运行了多久。

## 16. 查看当前运行的 Processes

### 命令

```bash
ps
```

示例输出：

```bash
  PID TTY          TIME CMD
  916 pts/58   00:00:00 bash
 1531 pts/58   00:00:00 ps
```

---

## 17. `ps` 命令

`ps` 是 **process status** 的缩写。

它用于显示当前 shell 会话中正在运行的进程。

在 Linux 中，每个 process 都有一个唯一编号：

```text
PID = Process Identifier
```

Linux 使用 PID 来识别和管理进程。

---

## 18. `ps` 输出内容解释

示例：

```bash
  PID TTY          TIME CMD
  916 pts/58   00:00:00 bash
 1531 pts/58   00:00:00 ps
```

可以理解为：

|列名|含义|
|---|---|
|`PID`|Process Identifier，进程编号|
|`TTY`|当前进程连接的终端|
|`TIME`|该进程已经使用的 CPU 时间|
|`CMD`|启动该进程的命令|

---

## 19. 为什么 `ps` 输出中会看到 `ps` 自己？

当你运行：

```bash
ps
```

系统会启动一个 `ps` 进程来查看当前进程状态。

因此，在输出中你通常会看到：

```bash
ps
```

这是正常的，因为 `ps` 命令本身在运行时也是一个 process。

---

## 20. 为什么会看到 `bash`？

输出中通常会看到：

```bash
bash
```

这是因为你当前正在使用 bash shell。  
你的 shell 本身也是一个正在运行的 process。

也就是说，当前终端会话中至少可能有：

```text
bash process
ps process
```

其中：

- `bash` 是你的交互式 shell；
    
- `ps` 是你刚刚运行的查看进程命令。
    

---

## 21. 更常见的进程查看命令

除了：

```bash
ps
```

还常见：

```bash
ps aux
```

它会显示系统中更完整的进程列表。

常见用途：

```bash
ps aux | grep python
```

用于查找正在运行的 Python 进程。

例如：

```bash
ps aux | grep train_model.py
```

可以用来检查某个程序是否还在运行。

---

## 22. 四个命令总结

|命令|作用|关键词|
|---|---|---|
|`echo $PATH`|显示可执行程序搜索路径|environment variable|
|`cat /etc/shells`|显示系统可用 shells|cat, system config|
|`date`|显示当前日期和时间|system time|
|`ps`|显示当前运行进程|process status, PID|
