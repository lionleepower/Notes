# Exercise: Redirecting Output

## 1. 本节快速概览

这一节主要练习 Linux 中的 output redirection 和文件查看方法。

核心内容包括：

- 使用 `>` 把命令输出保存到文件
- 使用 `cat`、`more`、`less` 查看文件
- 使用 `cat > file` 从键盘输入内容并保存到文件
- 使用 `cat >> file` 向已有文件追加内容
- 创建一个简单 shell script
- 使用 `<` 把文件内容作为 stdin 输入给脚本

本节核心命令：

| 命令 | 作用 |
|---|---|
| `ls -l /etc > mylist` | 把 `/etc` 目录的详细列表保存到 `mylist` |
| `cat mylist` | 直接显示整个文件内容 |
| `more mylist` | 分页查看文件 |
| `less mylist` | 更灵活地分页查看文件 |
| `cat > lincoln.txt` | 从键盘输入内容并写入文件，覆盖原内容 |
| `cat >> lincoln.txt` | 从键盘输入内容并追加到文件末尾 |
| `source tryme.sh < lincoln.txt` | 把 `lincoln.txt` 作为 stdin 输入给脚本 |

---

## 2. 使用重定向创建文件

教材中的第一个练习是：

```bash
ls -l /etc > mylist
````

含义是：

- `ls -l /etc`：列出 `/etc` 目录下文件的详细信息
    
- `>`：把 stdout 重定向到文件
    
- `mylist`：保存输出结果的文件名
    

执行后，屏幕上通常不会显示 `ls -l /etc` 的结果，因为结果被写入了 `mylist`。

可以用：

```bash
ls
```

检查当前目录下是否出现了 `mylist`。

---

## 3. `>` 的作用复习

```bash
command > file
```

含义是：

> 把 command 的标准输出 stdout 写入 file。

需要注意：

|情况|结果|
|---|---|
|文件不存在|创建新文件|
|文件已存在|覆盖原文件内容|

所以：

```bash
ls -l /etc > mylist
```

如果 `mylist` 已经存在，原来的内容会被覆盖。

---

## 4. 查看文件的三种主要方法

在 shell 中查看文本文件，常见方法有三种：

|命令|特点|
|---|---|
|`cat`|一次性显示整个文件|
|`more`|分页显示文件，功能较基础|
|`less`|分页显示文件，功能更强，更适合大文件|

本节使用 `mylist` 文件进行练习。

---

## 5. 使用 `cat` 查看文件

命令：

```bash
cat mylist
```

`cat` 会把整个文件内容直接输出到 terminal。

特点：

- 会显示完整文件内容
    
- 如果文件很长，屏幕会快速滚动到最后
    
- 适合查看短文件
    
- 不适合查看非常长的文件
    

例如：

```bash
cat lincoln.txt
```

适合查看几行文本。

---

## 6. `cat` 的特点

`cat` 的英文来源是：

```text
concatenate
```

原本意思是连接、拼接文件内容。

但在日常 Linux 使用中，它经常被用来查看短文件。

例如：

```bash
cat file.txt
```

也可以一次显示多个文件：

```bash
cat file1.txt file2.txt
```

这会依次输出两个文件的内容。

---

## 7. 使用 `more` 查看文件

命令：

```bash
more mylist
```

`more` 会分页显示文件内容。

特点：

- 当内容填满屏幕后暂停
    
- 使用 Space 进入下一页
    
- 比 `cat` 更适合查看长文件
    
- 但它会先读取整个文件，再开始显示
    
- 对非常大的文件可能加载较慢
    

常用操作：

|按键|作用|
|---|---|
|Space|下一页|
|Enter|下一行|
|q|退出|

---

## 8. 使用 `less` 查看文件

命令：

```bash
less mylist
```

`less` 和 `more` 类似，也是分页查看文件，但功能更多。

特点：

- 不需要先读取完整文件
    
- 对大文件更快
    
- 可以上下滚动
    
- 可以跳到开头或结尾
    
- 可以搜索关键词
    
- 是实际使用中更推荐的查看工具
    

常用操作：

|按键|作用|
|---|---|
|Space|下一页|
|↑ / ↓|上下移动一行|
|q|退出|
|g|跳到文件开头|
|G|跳到文件结尾|
|`/keyword`|向下搜索 keyword|
|n|跳到下一个搜索结果|
|`?keyword`|向上搜索 keyword|

---

## 9. `cat`、`more`、`less` 对比

|命令|适合场景|优点|缺点|
|---|---|---|---|
|`cat`|短文件|简单直接|长文件会刷屏|
|`more`|中等长度文件|可以分页|功能较少，大文件较慢|
|`less`|长文件、日志文件|快速、可搜索、可滚动|需要记住一些快捷键|

实际使用中，推荐：

```bash
less filename
```

尤其是查看长日志、Slurm 输出、程序运行记录时。

---

## 10. `less` 中搜索内容

在 `less` 中，可以使用类似 Vim 的搜索方式。

例如：

```bash
less mylist
```

进入文件后，输入：

```text
/foo
```

表示搜索 `foo`。

如果找到了匹配项，`less` 会跳到第一个匹配位置。

之后：

|按键|作用|
|---|---|
|`n`|跳到下一个匹配项|
|`?foo`|向上搜索 `foo`|

这个功能在查看大型日志文件时非常有用。

例如查找错误：

```bash
less run.log
```

进入后输入：

```text
/error
```

或者：

```text
/ERROR
```

---

## 11. `head` 和 `tail`

教材建议参考 `head` 和 `tail` 的 man pages。

这两个命令常用于查看文件开头或结尾。

### `head`

查看文件前几行：

```bash
head mylist
```

默认显示前 10 行。

也可以指定行数：

```bash
head -n 20 mylist
```

表示显示前 20 行。

---

### `tail`

查看文件最后几行：

```bash
tail mylist
```

默认显示最后 10 行。

也可以指定行数：

```bash
tail -n 20 mylist
```

表示显示最后 20 行。

---

## 12. 实时查看日志：`tail -f`

`tail` 常用于查看正在增长的日志文件。

```bash
tail -f run.log
```

含义是：

- 显示 `run.log` 最后几行
    
- 文件有新内容时自动继续显示
    

这在程序运行、服务器日志、Slurm 输出中非常常见。

退出：

```text
Control-C
```

## 13. 使用 `cat > file` 创建文本文件

教材中的例子是把一句话写入 `lincoln.txt`：

```bash
cat > lincoln.txt
```

执行后，shell 会等待你输入内容。

输入：

```text
Four score and seven years ago
```

然后按：

```text
Control-D
```

结束输入。

完整过程：

```bash
cat > lincoln.txt
Four score and seven years ago
[Control-D]
```

---

## 14. `cat > lincoln.txt` 的含义

```bash
cat > lincoln.txt
```

可以理解为：

- `cat` 没有指定文件，所以它从 stdin 读取
    
- stdin 默认来自键盘
    
- `>` 把 cat 的 stdout 写入 `lincoln.txt`
    
- 所以你键盘输入的内容会被写入文件
    

数据流是：

```text
Keyboard stdin → cat → stdout → lincoln.txt
```

---

## 15. Control-D 的作用

在输入模式下，`Control-D` 表示：

```text
End of File
```

也就是告诉 shell：

> 输入结束了。

如果不按 `Control-D`，`cat` 会继续等待你输入更多内容。

注意：

- `Control-D` 不是输入字符
    
- 它是结束 stdin 的信号
    
- 在终端中常用于结束手动输入
    

---

## 16. 查看刚创建的文件

创建完成后，可以使用：

```bash
cat lincoln.txt
```

输出应为：

```text
Four score and seven years ago
```

这说明内容已经成功写入文件。

也可以使用：

```bash
less lincoln.txt
```

或：

```bash
more lincoln.txt
```

查看。

---

## 17. 使用 `cat >> file` 追加内容

如果想向已有文件末尾添加内容，可以使用：

```bash
cat >> lincoln.txt
```

然后输入：

```text
our fathers brought forth on this continent, a new nation
```

最后按：

```text
Control-D
```

完整过程：

```bash
cat >> lincoln.txt
our fathers brought forth on this continent, a new nation
[Control-D]
```

---

## 18. `>` 和 `>>` 的区别复习

|符号|作用|
|---|---|
|`>`|写入文件，覆盖原内容|
|`>>`|追加到文件末尾|

如果执行：

```bash
cat > lincoln.txt
```

会覆盖 `lincoln.txt` 原有内容。

如果执行：

```bash
cat >> lincoln.txt
```

会保留原有内容，并在末尾追加新内容。

---

## 19. 查看追加后的文件

执行：

```bash
cat lincoln.txt
```

应该看到：

```text
Four score and seven years ago
our fathers brought forth on this continent, a new nation
```

这说明第二行已经被追加到了原文件中。

## 20. 创建简单脚本文件

教材接下来创建一个简单脚本：

```bash
cat > tryme.sh
#!/bin/sh
cat
[Control-D]
```

这会创建一个名为 `tryme.sh` 的文件，内容是：

```sh
#!/bin/sh
cat
```

其中：

- 第一行 `#!/bin/sh` 指定脚本使用的 shell interpreter
    
- 第二行 `cat` 表示运行 `cat` 命令
    

---

## 21. 什么是 shell script？

**Shell script** 是由一系列 shell 命令组成的文本文件。

例如：

```sh
#!/bin/sh
cat
```

这个脚本的作用非常简单：

- 启动 `/bin/sh`
    
- 执行 `cat`
    
- 因为 `cat` 没有指定文件，所以它从 stdin 读取内容
    
- 然后把读取到的内容输出到 stdout
    

---

## 22. Shebang：`#!/bin/sh`

脚本第一行：

```sh
#!/bin/sh
```

叫做：

```text
shebang
```

它告诉系统用哪个 interpreter 来运行这个脚本。

其中：

```text
/bin/sh
```

是 shell interpreter 的路径。

在很多系统上，`/bin/sh` 是一个 symlink，指向默认 shell。

可能指向：

```text
/bin/bash
```

也可能指向其他 shell，例如：

```text
/bin/dash
```

---

## 23. 查看 `/bin/sh` 指向哪里

教材中的命令是：

```bash
ls -l /bin/sh
```

可能看到类似输出：

```text
/bin/sh -> bash
```

或者：

```text
/bin/sh -> dash
```

这说明 `/bin/sh` 不是普通文件，而是一个 symbolic link。

---

## 24. 什么是 symlink？

**Symlink** 全称是：

```text
symbolic link
```

中文一般叫符号链接。

它可以理解为一个“快捷方式”或“指针”，指向另一个文件。

例如：

```text
/bin/sh -> /bin/bash
```

表示 `/bin/sh` 实际上指向 `/bin/bash`。

---

## 25. 使用 bash 作为 interpreter

如果想明确使用 Bash，可以把脚本第一行改成：

```sh
#!/bin/bash
```

这表示使用 `/bin/bash` 来执行脚本。

一般来说：

|写法|含义|
|---|---|
|`#!/bin/sh`|使用系统默认 POSIX shell|
|`#!/bin/bash`|明确使用 Bash|

如果脚本中使用了 Bash 特有语法，应该使用：

```sh
#!/bin/bash
```

---

## 26. 使用 `source` 执行脚本

教材中使用：

```bash
source tryme.sh < lincoln.txt
```

含义是：

- `source tryme.sh`：在当前 shell 中执行 `tryme.sh`
    
- `< lincoln.txt`：把 `lincoln.txt` 作为 stdin 输入给脚本
    

因为 `tryme.sh` 中只有：

```sh
cat
```

所以它会读取 stdin，也就是 `lincoln.txt` 的内容，然后打印到屏幕。

---

## 27. 数据流解释：`source tryme.sh < lincoln.txt`

命令：

```bash
source tryme.sh < lincoln.txt
```

数据流是：

```text
lincoln.txt → stdin → tryme.sh → cat → stdout → screen
```

最终屏幕输出：

```text
Four score and seven years ago
our fathers brought forth on this continent, a new nation
```

---

## 28. 如果省略 `< lincoln.txt` 会怎样？

如果只运行：

```bash
source tryme.sh
```

脚本中的 `cat` 会从默认 stdin 读取，也就是键盘输入。

此时你输入什么，它就会立刻输出什么。

例如：

```text
hello
hello
test
test
```

因为 `cat` 会把 stdin 原样复制到 stdout。

结束输入依然需要按：

```text
Control-D
```

---

## 29. `source` 和直接执行脚本的区别

教材中使用的是：

```bash
source tryme.sh
```

实际中也常见直接执行脚本：

```bash
./tryme.sh
```

但直接执行通常需要先给脚本执行权限：

```bash
chmod +x tryme.sh
./tryme.sh < lincoln.txt
```

区别：

|方法|含义|
|---|---|
|`source tryme.sh`|在当前 shell 中执行脚本内容|
|`./tryme.sh`|启动一个新的子 shell 执行脚本|

这一节教材主要想说明 input redirection，所以使用 `source` 已经足够。

---

## 30. 为什么这个脚本会输出文件内容？

脚本内容是：

```sh
#!/bin/sh
cat
```

`cat` 没有参数时，会从 stdin 读取内容。

命令：

```bash
source tryme.sh < lincoln.txt
```

把 `lincoln.txt` 连接到 stdin。

所以 `cat` 读取的是 `lincoln.txt` 的内容，而不是键盘输入。

然后 `cat` 默认把内容输出到 stdout，也就是屏幕。

## 37. 易错点

### 1. `>` 会覆盖文件

如果执行：

```bash
cat > lincoln.txt
```

并重新输入内容，原来的 `lincoln.txt` 会被覆盖。

如果只是想添加内容，应该使用：

```bash
cat >> lincoln.txt
```

---

### 2. 忘记按 `Control-D`

执行：

```bash
cat > lincoln.txt
```

后，如果输入完内容但不按 `Control-D`，shell 会继续等待输入。

按：

```text
Control-D
```

表示输入结束。

---

### 3. `cat` 查看长文件会刷屏

```bash
cat mylist
```

如果文件很长，内容会直接滚到最后。

查看长文件更推荐：

```bash
less mylist
```

---

### 4. `more` 和 `less` 不完全一样

`more` 功能比较基础，而 `less` 更适合大文件。

一般实际使用中优先选：

```bash
less file
```

---

### 5. `cat` 没有参数时会读 stdin

```bash
cat
```

如果不指定文件，`cat` 会等待键盘输入，并把输入内容输出出来。

结束需要按：

```text
Control-D
```

---

### 6. `source tryme.sh < lincoln.txt` 中的 `<` 很关键

如果没有：

```bash
< lincoln.txt
```

脚本不会读取 `lincoln.txt`，而是从键盘读取输入。
