# Directory Navigation

## 1. 本节快速概览

在 Linux 中，文件系统是一个 **hierarchical file system**，也就是层级式文件系统。

目录导航的核心目标是：

- 知道自己现在在哪个目录
- 查看当前目录中有什么文件
- 在不同目录之间切换
- 理解 absolute path 和 relative path
- 理解特殊目录符号，如 `.`, `..`, `~`, `$HOME`

本节主要命令：

| 命令 | 英文关键词 | 作用 |
|---|---|---|
| `pwd` | print working directory | 显示当前所在目录 |
| `ls` | list | 列出目录内容 |
| `cd` | change directory | 切换目录 |

---

## 2. Linux 的目录结构

Linux 使用层级式文件系统。

最顶层目录叫做：

```bash
/
````

它被称为：

- root directory
    
- 根目录
    
- 文件系统的最高层目录
    

所有文件和目录最终都位于 `/` 下面。

例如：

```text
/
├── home
├── usr
├── bin
├── var
├── etc
└── tmp
```

可以把 `/` 理解成整个 Linux 文件系统的起点。

---

## 3. Home Directory

每个用户通常都有自己的 home directory。

一般路径是：

```bash
/home/<username>
```

例如用户叫 `leyan`，那么 home directory 可能是：

```bash
/home/leyan
```

在 HPC 系统中，路径可能不完全一样，例如：

```bash
/home1/05574/<username>
```

Home directory 通常是打开 shell 后的默认工作目录。

---

## 4. Home Directory 的常见简写

Home directory 有两个常见简写：

|写法|含义|
|---|---|
|`~`|当前用户的 home directory|
|`$HOME`|保存 home directory 路径的环境变量|

例如下面三个命令通常含义接近：

```bash
cd ~
```

```bash
cd $HOME
```

```bash
cd
```

它们一般都会回到当前用户的 home directory。

---

## 5. Absolute Path

**Absolute path** 也叫 **full path**，即绝对路径或完整路径。

它从根目录 `/` 开始，完整描述从文件系统顶层到目标文件或目录的路径。

例如：

```bash
/home/leyan/project/test.c
```

这是 absolute path，因为它从 `/` 开始。

判断一个路径是不是 absolute path 的简单方法：

> 如果路径最开头是 `/`，它就是 absolute path。

例如：

```bash
/usr/bin/python
```

```bash
/etc/shells
```

```bash
/home/leyan/notes/linux.md
```

这些都是 absolute path。

---

## 6. Relative Path

**Relative path** 是相对路径。

它不是从 `/` 开始，而是从当前所在目录出发，描述如何到达目标位置。

例如当前目录是：

```bash
/home/leyan
```

如果有一个文件在：

```bash
/home/leyan/project/test.c
```

那么可以用 relative path 表示为：

```bash
project/test.c
```

因为它是从当前目录 `/home/leyan` 出发的。

---

## 7. Absolute Path 和 Relative Path 对比

|类型|起点|示例|特点|
|---|---|---|---|
|Absolute path|`/` 根目录|`/home/leyan/project/test.c`|不受当前目录影响|
|Relative path|当前目录|`project/test.c`|取决于当前所在目录|

### 例子

假设当前目录是：

```bash
/home/leyan
```

目标文件是：

```bash
/home/leyan/project/main.c
```

可以写成：

```bash
/home/leyan/project/main.c
```

也可以写成：

```bash
project/main.c
```

第一种是 absolute path，第二种是 relative path。

---

## 8. 当前目录：`.`

`.` 表示当前目录。

例如：

```bash
ls .
```

意思是列出当前目录中的内容。

再例如：

```bash
./program
```

意思是运行当前目录下的 `program` 文件。

这里的 `./` 很常见，尤其是在运行自己编译出来的程序时。

例如：

```bash
gcc main.c -o main
./main
```

其中 `./main` 表示运行当前目录下的 `main` 可执行文件。

---

## 9. 上一级目录：`..`

`..` 表示当前目录的上一级目录，也就是 parent directory。

例如当前目录是：

```bash
/home/leyan/project/src
```

那么：

```bash
..
```

表示：

```bash
/home/leyan/project
```

可以使用：

```bash
cd ..
```

回到上一级目录。

也可以连续使用：

```bash
cd ../..
```

表示回到上两级目录。

---

## 10. root directory 中的 `..`

在普通目录中，`..` 表示上一级目录。

但是 root directory `/` 已经是最高层目录，所以它没有真正意义上的上一级目录。

如果你已经在 `/` 中，再执行：

```bash
cd ..
```

通常仍然会停留在 `/`。

---

## 11. Dot Files

以 `.` 开头的文件叫做：

- dot files
    
- hidden files
    
- 隐藏文件
    

例如：

```bash
.bashrc
```

```bash
.profile
```

```bash
.gitignore
```

这些文件默认不会被普通的 `ls` 显示出来。

如果想显示隐藏文件，需要使用：

```bash
ls -a
```

dot files 通常用于：

- 用户配置
    
- shell 配置
    
- 程序配置
    
- 系统偏好设置
    

例如 `.bashrc` 常用于配置 Bash shell 的环境变量、别名和启动行为。

---

## 12. `pwd` 命令

`pwd` 的全称是：

```text
print working directory
```

作用是显示当前所在目录的完整路径。

使用方式：

```bash
pwd
```

示例输出：

```bash
/home1/05574/<username>
```

这说明你当前所在目录是：

```bash
/home1/05574/<username>
```

在 Linux 中，如果你不知道自己现在在哪个目录，第一反应可以使用：

```bash
pwd
```

---

## 13. `ls` 命令

`ls` 的作用是列出目录内容。

基本用法：

```bash
ls
```

示例输出：

```bash
test1.txt  test2.txt  test3.txt
```

表示当前目录中有三个文件：

```text
test1.txt
test2.txt
test3.txt
```

---

## 14. 查看指定目录内容

`ls` 不仅可以查看当前目录，也可以查看指定目录。

例如：

```bash
ls /home
```

表示列出 `/home` 目录下的内容。

再例如：

```bash
ls ~/project
```

表示列出 home directory 下 `project` 目录的内容。

---

## 15. 使用 wildcard `*`

`*` 是 wildcard，表示匹配任意内容。

例如：

```bash
ls *.c
```

表示列出当前目录下所有以 `.c` 结尾的文件，也就是所有 C source code 文件。

例如当前目录有：

```text
main.c
stencil.c
test.txt
README.md
```

执行：

```bash
ls *.c
```

输出可能是：

```text
main.c  stencil.c
```

---

## 16. 常见 wildcard 例子

|命令|含义|
|---|---|
|`ls *.c`|显示所有 `.c` 文件|
|`ls *.txt`|显示所有 `.txt` 文件|
|`ls test*`|显示所有以 `test` 开头的文件|
|`ls *test*`|显示所有文件名中包含 `test` 的文件|
|`ls *.slurm`|显示所有 `.slurm` 作业脚本|

在 HPC 课程或实验中，经常会看到：

```bash
ls *.c
ls *.h
ls *.slurm
ls *.out
```

---

## 17. `ls -a`

默认情况下，`ls` 不显示 dot files。

如果要显示所有文件，包括隐藏文件，使用：

```bash
ls -a
```

其中：

```text
-a = all
```

例如可能看到：

```text
.  ..  .bashrc  .profile  project
```

这里：

|文件或目录|含义|
|---|---|
|`.`|当前目录|
|`..`|上一级目录|
|`.bashrc`|Bash 配置文件|
|`.profile`|用户登录配置文件|
|`project`|普通目录|

---

## 18. `ls -l`

`ls -l` 使用 long format 显示详细信息。

```bash
ls -l
```

示例：

```text
-rw-r--r-- 1 leyan users 1024 May 14 10:00 test.txt
```

它会显示：

- 文件权限
    
- 链接数
    
- 所有者
    
- 用户组
    
- 文件大小
    
- 修改时间
    
- 文件名
    

虽然本节原文重点不在权限，但 `ls -l` 是非常常用的查看文件信息方式。

---

## 19. `ls -h`

`-h` 表示 human-readable。

通常和 `-l` 一起使用：

```bash
ls -lh
```

它会把文件大小显示成人类更容易读的格式。

例如：

```text
4.0K
12M
1.5G
```

而不是纯字节数。

---

## 20. `ls -lha`

这是一个非常常用的组合：

```bash
ls -lha
```

含义是：

|选项|含义|
|---|---|
|`-l`|long format，显示详细信息|
|`-h`|human-readable，文件大小更易读|
|`-a`|all，显示隐藏文件|

例如：

```bash
ls -lha $SCRATCH
```

表示以详细、人类可读的格式，列出 `$SCRATCH` 目录下的所有文件，包括隐藏文件。

---

## 21. 示例：`ls -lha $SCRATCH`

假设 `$SCRATCH` 是一个目录路径：

```bash
ls -lha $SCRATCH
```

可能输出：

```text
total 12K
drwx------ 3 <username> G-819251 4.0K Jul  9 14:57 .
drwxr-xr-x 5 root       root     4.0K May 14 16:13 ..
drwx------ 2 <username> G-819251 4.0K Jul  9 14:57 .slurm
```

其中：

- `.` 表示 `$SCRATCH` 当前目录本身
    
- `..` 表示 `$SCRATCH` 的上一级目录
    
- `.slurm` 是隐藏目录，通常和 Slurm 作业系统相关
    

---

## 22. `$SCRATCH`

`$SCRATCH` 通常是一个环境变量，指向临时或高性能存储目录。

在 HPC 系统中，常见目录包括：

|目录类型|常见用途|
|---|---|
|`$HOME`|存放配置文件、小型代码、长期保留的文件|
|`$SCRATCH`|存放实验输出、大型中间文件、临时计算数据|
|`$WORK`|有些系统中用于项目工作目录|

在 HPC 上运行程序时，大型输出文件通常不建议直接放在 `$HOME`，而应放在 `$SCRATCH` 或项目指定存储区域。

## 23. `cd` 命令

`cd` 的全称是：

```text
change directory
```

作用是切换当前所在目录。

基本格式：

```bash
cd <path>
```

例如：

```bash
cd ~/project
```

表示进入 home directory 下的 `project` 目录。

---

## 24. 回到 home directory

下面命令可以回到 home directory：

```bash
cd ~
```

也可以写成：

```bash
cd $HOME
```

更简单地，也可以直接：

```bash
cd
```

不带参数的 `cd` 通常会回到 home directory。

---

## 25. 进入上一级目录

```bash
cd ..
```

表示进入当前目录的上一级目录。

例如当前目录是：

```bash
/home/leyan/project/src
```

执行：

```bash
cd ..
```

之后会到：

```bash
/home/leyan/project
```

---

## 26. 使用 relative path 切换目录

假设当前目录是：

```bash
/home/leyan/project/src
```

执行：

```bash
cd ../data
```

含义是：

1. `..` 先回到上一级目录 `/home/leyan/project`
    
2. `data` 再进入 `/home/leyan/project/data`
    

最终目录是：

```bash
/home/leyan/project/data
```

---

## 27. 教材例子：`cd ../var`

教材中给出的例子是：

```bash
cd ../var
```

它的含义是：

1. `..` 先进入当前目录的上一级目录
    
2. 再进入那个上一级目录下的 `var` 目录
    

如果当前目录是：

```bash
/usr
```

那么：

```bash
cd ../var
```

过程是：

```text
/usr
→ ..
→ /
→ var
→ /var
```

最终进入：

```bash
/var
```

## 34. 实用工作流

进入一个 Linux 项目目录时，可以按这个顺序操作：

```bash
pwd
ls -lha
cd <target_directory>
pwd
ls
```

含义是：

1. `pwd`：确认自己现在在哪里
    
2. `ls -lha`：查看当前目录下有什么
    
3. `cd <target_directory>`：进入目标目录
    
4. `pwd`：确认是否进入成功
    
5. `ls`：查看目标目录内容
    

这个习惯可以避免在错误目录中运行命令。

## 35. HPC 场景中的使用例子

在 HPC 系统中，常见操作可能是：

```bash
pwd
ls -lha
cd $SCRATCH
ls -lha
```

含义：

1. 查看当前目录
    
2. 查看当前目录内容
    
3. 进入 `$SCRATCH`
    
4. 查看 scratch 空间内容
    

如果要进入某个作业目录：

```bash
cd $SCRATCH/my_job
ls -lha
```

如果要找 Slurm 输出文件：

```bash
ls *.out
```

如果要找 Slurm 脚本：

```bash
ls *.slurm
```

## 36. 易错点

### 1. `/` 在开头和中间含义不同

路径开头的 `/` 表示 root directory。

例如：

```bash
/home/leyan
```

这是 absolute path。

但路径中间的 `/` 只是目录分隔符。

例如：

```bash
project/src/main.c
```

这里的 `/` 用于分隔 `project`、`src` 和 `main.c`。

---

### 2. `cd /` 和 `cd ~` 完全不同

```bash
cd /
```

进入 root directory。

```bash
cd ~
```

进入当前用户的 home directory。

二者不是一个地方。

---

### 3. `.` 和 `..` 容易混淆

|符号|含义|
|---|---|
|`.`|当前目录|
|`..`|上一级目录|

例如：

```bash
ls .
```

查看当前目录。

```bash
ls ..
```

查看上一级目录。

---

### 4. 隐藏文件不是不存在

如果你执行：

```bash
ls
```

看不到 `.bashrc` 或 `.gitignore`，不代表它们不存在。

需要使用：

```bash
ls -a
```

才能显示隐藏文件。

---

### 5. relative path 会受当前目录影响

例如：

```bash
cd project
```

只有当当前目录下存在 `project` 目录时才有效。

如果当前目录不同，这个命令可能失败。

所以在不确定时，先使用：

```bash
pwd
```

确认当前位置。

## 38. 本节重点总结

目录导航的核心是理解自己在文件系统中的位置。

最重要的三个命令是：

```bash
pwd
ls
cd
```

最重要的路径概念是：

|概念|含义|
|---|---|
|absolute path|从 `/` 开始的完整路径|
|relative path|从当前目录出发的路径|
|root directory `/`|文件系统最高层目录|
|home directory `~` 或 `$HOME`|当前用户的主目录|
|current directory `.`|当前目录|
|parent directory `..`|上一级目录|
