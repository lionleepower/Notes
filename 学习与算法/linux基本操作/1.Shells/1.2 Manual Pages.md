# Linux Manual Pages

## 1. 本节快速概览

在 Linux / Unix shell 中，如果想查询某个命令或程序的详细用法，最常用的方法是使用：

```bash
man <program_or_command>
````

例如：

```bash
man rm
```

`man` 是 **manual** 的缩写，表示系统手册。Linux 通常为大多数命令都内置了 manual page，因此 `man` 应该是学习和查找命令用法时的首选参考。

---
## 2. 什么是 Manual Page？

**Manual Page**，简称 **man page**，是 Linux 系统中用于说明命令、系统调用、库函数、文件格式等内容的官方文档。

它通常包含：

- 命令的作用
    
- 命令的基本语法
    
- 可用选项
    
- 使用示例
    
- 相关命令
    

例如：

```bash
man rm
```

会打开 `rm` 命令的手册页面。

`rm` 的作用是删除文件或目录。

---

## 3. man 命令的基本用法

### 基本格式

```bash
man <command>
```

例如：

```bash
man ls
man cd
man rm
man mkdir
```

含义是查看对应命令的 manual page。

---

## 4. man page 的章节分类

Linux 的 manual 被分成多个 section，不同 section 对应不同类型的内容。

|Section|内容|
|---|---|
|1|Executable programs and shell commands，可执行程序和 shell 命令|
|2|System calls，系统调用，即 kernel 提供的函数|
|3|Library calls，库函数|
|4|Special files，特殊文件|
|5|File formats and conventions，文件格式和约定|
|6|Games，游戏|
|7|Miscellaneous，杂项内容，例如宏包、约定等|
|8|System administration commands，系统管理命令，通常需要 root 权限|

---

## 5. 指定 section 查看 man page

有些名称可能在多个 section 中都有对应的 manual page。

例如 `mkdir` 既可以是一个 shell 命令，也可以是一个系统调用。

默认情况下：

```bash
man mkdir
```

通常会显示 Section 1，也就是 shell 命令版本。

如果想查看 Section 2 中的系统调用版本，可以使用：

```bash
man 2 mkdir
```

其中：

- `2` 表示 manual 的 Section 2
    
- `mkdir` 是要查询的对象
    

---

## 6. man page 的常见结构

不同命令的 manual page 内容可能不同，但一般会遵循类似结构。

### NAME

简短说明命令的作用。

例如：

```text
rm - remove files or directories
```

意思是：`rm` 用于删除文件或目录。

---

### SYNOPSIS

说明命令的基本语法。

例如：

```text
rm [OPTION]... FILE...
```

这里的含义是：

- `rm` 是命令本身
    
- `[OPTION]...` 表示可以有零个或多个选项
    
- `FILE...` 表示可以指定一个或多个文件
    

常见写法含义：

|写法|含义|
|---|---|
|`[OPTION]`|可选参数|
|`...`|可以重复多个|
|`FILE`|文件名或路径|
|`<command>`|需要替换成实际命令名|

---

### DESCRIPTION

详细描述命令的功能。

例如 `rm` 的说明中会提到：

- `rm` 会删除指定文件
    
- 默认情况下不会删除目录
    
- 如果使用递归选项，则可以删除目录及其内容
    

---

### OPTIONS

列出命令支持的选项。

例如 `rm` 的常见选项：

|选项|含义|
|---|---|
|`-f`, `--force`|强制删除，不提示；忽略不存在的文件|
|`-i`|每次删除前都询问确认|
|`-r`, `-R`, `--recursive`|递归删除目录及其内容|
|`-v`, `--verbose`|显示正在执行的操作|

---

### EXAMPLES

给出命令的使用示例。

有些命令的 man page 会包含这一部分，但不是所有命令都有。

---

### SEE ALSO

列出相关命令或相关 manual page。

例如在某个命令页面的 `SEE ALSO` 中，可能会看到与它功能相近或相关的命令。

---

## 7. 在 man page 中导航

进入 man page 后，可以使用键盘进行浏览。

|按键|作用|
|---|---|
|↑|向上滚动|
|↓|向下滚动|
|Enter|向下移动一行|
|Space|向下翻一页|
|q|退出 man page|

如果终端不支持鼠标滚动，就主要使用这些按键。

---

## 8. 示例：查看 rm 命令

输入：

```bash
man rm
```

可能看到：

```text
NAME
    rm - remove files or directories

SYNOPSIS
    rm [OPTION]... FILE...

DESCRIPTION
    rm removes each specified file.
    By default, it does not remove directories.
```

其中：

- `NAME` 告诉我们 `rm` 是删除文件或目录的命令
    
- `SYNOPSIS` 告诉我们 `rm` 的命令格式
    
- `DESCRIPTION` 说明它默认删除文件，但默认不删除目录
    

---

## 9. rm 常见选项解释

### `-f` 或 `--force`

```bash
rm -f file.txt
```

含义：

强制删除文件，即使文件不存在也不会报错或提示。

适合脚本中使用，但手动操作时要谨慎。

---

### `-i`

```bash
rm -i file.txt
```

含义：

删除前询问用户确认。

例如系统可能提示：

```text
rm: remove regular file 'file.txt'?
```

适合防止误删。

---

### `-r`, `-R`, `--recursive`

```bash
rm -r my_folder
```

含义：

递归删除目录及其内部所有内容。

这个选项很危险，尤其是与 `-f` 组合使用时。

例如：

```bash
rm -rf my_folder
```

表示强制递归删除 `my_folder`。

---

### `-v` 或 `--verbose`

```bash
rm -v file.txt
```

含义：

显示命令正在做什么。

例如：

```text
removed 'file.txt'
```

---

## 10. 查看 man 命令自己的说明

`man` 本身也是一个命令，所以也有自己的 manual page。

```bash
man man
```

这可以查看 `man` 命令本身的用法。

---

## 11. 使用 `man -k` 搜索相关命令

如果不确定某个命令是否存在，或者只知道关键词，可以使用：

```bash
man -k <keyword>
```

例如：

```bash
man -k who
```

它会搜索 manual page 中与 `who` 相关的条目，并显示简短描述。

如果结果中有 `who` 命令，就可以继续使用：

```bash
man who
```

查看详细说明。

---

## 12. `man -k` 的作用

`man -k` 类似于在 manual page 中搜索关键词。

它的作用是：

- 查找某个关键词相关的命令
    
- 查看是否存在对应的 man page
    
- 帮助发现自己不知道的命令
    

例如：

```bash
man -k copy
```

可能会找到和复制相关的命令，例如 `cp`。

---

## 13. 本节重点命令总结

|命令|作用|
|---|---|
|`man <command>`|查看某个命令的 manual page|
|`man <section> <command>`|查看指定 section 中的 manual page|
|`man man`|查看 `man` 命令自己的 manual page|
|`man -k <keyword>`|按关键词搜索 manual page|

---

## 14. 重要概念总结

|概念|解释|
|---|---|
|`man`|manual 的缩写，用于查看系统手册|
|`man page`|Linux / Unix 系统中的命令或功能说明文档|
|`section`|manual page 的分类编号|
|`SYNOPSIS`|命令语法摘要|
|`OPTIONS`|命令可用选项|
|`SEE ALSO`|相关命令或文档|

##  易错点

### 1. `man mkdir` 和 `man 2 mkdir` 不完全一样

```bash
man mkdir
```

通常查看的是 shell command。

```bash
man 2 mkdir
```

查看的是 system call。

它们名字一样，但面向对象不同。

---

### 2. `rm -r` 很危险

```bash
rm -r folder
```

会删除整个目录及其内容。

尤其要小心：

```bash
rm -rf folder
```

这个命令不会提示确认，可能造成严重误删。

---

### 3. 不是所有 man page 都有 EXAMPLES

有些 manual page 很详细，有些比较简略。没有 `EXAMPLES` 并不代表命令不能用，只是该页面没有提供示例部分。
