# Linux Shell：Piping, Sorting, and Counting 管道、排序与计数

本节主要介绍 Linux Shell 中三个非常实用的操作：

- `|`：pipe，管道，把一个命令的输出传给另一个命令
- `sort`：排序
- `wc`：word count，用于统计行数、单词数和字节数

这一节的核心思想是：

```text
把多个简单命令组合起来，完成更复杂的任务。
````

---

## 1. Pipe 管道 `|`

管道符号是：

```bash
|
```

它的作用是：

```text
把左边命令的输出 output，作为右边命令的输入 input。
```

基本格式：

```bash
command1 | command2
```

含义是：

```text
command1 的输出结果，不直接显示在屏幕上，而是交给 command2 继续处理。
```

---

## 2. `|` 和 `>` 的区别

之前学过的重定向符号 `>` 是把输出写入文件。

例如：

```bash
ls > mylist
```

意思是：

```text
把 ls 的输出保存到 mylist 文件中。
```

而管道 `|` 是把输出传给另一个命令。

例如：

```bash
ls | sort
```

意思是：

```text
把 ls 的输出传给 sort，让 sort 对文件列表排序。
```

两者区别如下：

|符号|英文|作用|
|---|---|---|
|`>`|redirection|把输出写入文件|
|`|`|pipe|

简单记忆：

```text
> 是输出到文件
| 是输出到命令
```

---

## 3. stdin 和 stdout

理解管道前，需要知道两个概念：

|名称|英文|含义|
|---|---|---|
|标准输入|`stdin`|命令接收输入的地方|
|标准输出|`stdout`|命令输出结果的地方|

很多 Linux 命令都可以：

```text
从 stdin 读取输入
向 stdout 输出结果
```

所以管道可以把多个命令连接起来：

```bash
command1 | command2 | command3
```

这表示：

```text
command1 的输出传给 command2，
command2 的输出再传给 command3。
```

# Part 1：Piping 管道

## 4. `history` 命令

`history` 命令用于显示当前用户曾经在 Shell 中输入过的命令。

```bash
history
```

它会把历史命令输出到屏幕，也就是输出到 `stdout`。

例如可能会看到：

```bash
1901  cat /etc/shells
1929  cat mylist
1937  history | grep cat
```

左边的数字是 bash history 中的行号，通常表示命令在历史记录中的位置。

---

## 5. 使用 `history | grep` 搜索历史命令

如果历史命令很多，直接看 `history` 会很麻烦。

这时可以使用：

```bash
history | grep cat
```

意思是：

```text
先用 history 输出所有历史命令，
再把这些输出传给 grep，
最后由 grep 找出包含 cat 的行。
```

执行逻辑可以拆成两步理解：

```bash
history
```

输出历史命令。

然后：

```bash
grep cat
```

筛选出包含 `cat` 的行。

用管道合起来就是：

```bash
history | grep cat
```

---

## 6. 示例：搜索历史中的 `cat` 命令

```bash
$ history | grep cat
 1901  cat /etc/shells
 1927  cat programinput | mycommand
 1929  cat mylist
 1930  cat > lincoln.txt
 1931  cat lincoln.txt
 1932  cat >> lincoln.txt
 1933  cat > tryme.sh
 1936  cat foo
 1937  history | grep cat
```

这个结果表示：

```text
历史命令中所有包含 cat 的命令都被筛选出来了。
```

注意最后一行：

```bash
1937  history | grep cat
```

它自己也包含 `cat`，所以也会被显示出来。

---

## 7. 把管道结果保存到文件

管道可以和重定向一起使用。

例如：

```bash
history | grep cat > cat_history.txt
```

含义是：

```text
先从历史命令中找出包含 cat 的行，
再把结果保存到 cat_history.txt 文件中。
```

这里的执行流程是：

```text
history → grep cat → cat_history.txt
```

也就是：

```text
命令输出 → 筛选 → 保存到文件
```

---

## 8. 再次搜索保存后的文件

假设已经生成了文件：

```bash
cat_history.txt
```

可以继续搜索其中包含 `lincoln` 的行：

```bash
cat cat_history.txt | grep lincoln
```

输出：

```bash
1930  cat > lincoln.txt
1931  cat lincoln.txt
1932  cat >> lincoln.txt
```

意思是：

```text
在 cat_history.txt 文件中查找包含 lincoln 的行。
```

不过这个命令也可以更简洁地写成：

```bash
grep lincoln cat_history.txt
```

因为 `grep` 本身可以直接读取文件。

---

## 9. `cat file | grep pattern` 和 `grep pattern file`

下面两个命令效果类似：

```bash
cat cat_history.txt | grep lincoln
```

和：

```bash
grep lincoln cat_history.txt
```

更推荐写成：

```bash
grep lincoln cat_history.txt
```

原因是：

```text
grep 本身就可以读取文件，不需要先用 cat 输出文件内容。
```

但是在学习管道时，`cat file | grep pattern` 更容易帮助理解：

```text
一个命令的输出可以传给另一个命令。
```

---

## 10. 使用 `ps -ef | grep bash` 查找进程

另一个常见例子是查找正在运行的进程。

```bash
ps -ef | grep bash
```

其中：

|命令|含义|
|---|---|
|`ps`|process status，查看进程|
|`-ef`|显示系统中的所有进程，并使用 full format|
|`grep bash`|筛选出包含 `bash` 的行|

所以：

```bash
ps -ef | grep bash
```

意思是：

```text
列出所有进程，然后找出其中和 bash 相关的进程。
```

---

## 11. 管道的核心作用

管道的本质是：

```text
把一个命令的结果交给另一个命令继续处理。
```

常见组合包括：

```bash
history | grep cat
```

搜索历史命令。

```bash
ls | sort
```

对文件列表排序。

```bash
ls | wc -l
```

统计当前目录下有多少个文件或目录。

```bash
ps -ef | grep bash
```

查找 bash 相关进程。

# Part 2：Sorting 排序

## 12. `sort` 命令

`sort` 命令用于排序。

它可以对：

- 文件内容排序
    
- stdin 输入内容排序
    
- 其他命令的输出排序
    

基本格式：

```bash
sort filename
```

或者：

```bash
command | sort
```

---

## 13. 示例：对文本文件排序

假设 `temp.txt` 内容如下：

```bash
$ cat temp.txt
cherry
apple
x-ray
clock
orange
bananna
```

执行：

```bash
sort temp.txt
```

输出：

```bash
apple
bananna
cherry
clock
orange
x-ray
```

解释：

```text
sort 默认按照字母顺序 alphabetically 排序。
```

---

## 14. 反向排序：`sort -r`

`-r` 表示 reverse，反向排序。

```bash
sort -r temp.txt
```

如果原本排序结果是：

```text
apple
bananna
cherry
clock
orange
x-ray
```

那么反向排序就是：

```text
x-ray
orange
clock
cherry
bananna
apple
```

---

## 15. 数字排序：`sort -n`

默认情况下，`sort` 按文本顺序排序，而不是按真正的数字大小排序。

数字排序要用：

```bash
sort -n filename
```

其中 `-n` 表示 numeric sort。

---

## 16. 数字反向排序：`sort -nr`

假设文件 `temp2.txt` 内容如下：

```bash
$ cat temp2.txt
7
48
1
56
8
32
```

执行：

```bash
sort -nr temp2.txt
```

输出：

```bash
56
48
32
8
7
1
```

解释：

|选项|含义|
|---|---|
|`-n`|按数字大小排序|
|`-r`|反向排序|
|`-nr`|按数字从大到小排序|

---

## 17. 选项可以组合

下面几种写法在这里通常等价：

```bash
sort -nr temp2.txt
```

```bash
sort -rn temp2.txt
```

```bash
sort -n -r temp2.txt
```

含义都是：

```text
按数字排序，并反向输出。
```

也就是从大到小排序。

---

## 18. 使用 `ls | sort -r`

如果想把当前目录中的文件名反向排序，可以使用：

```bash
ls | sort -r
```

含义是：

```text
先用 ls 列出当前目录内容，
再用 sort -r 进行反向排序。
```

执行流程：

```text
ls → sort -r
```

# Part 3：Counting 计数

## 19. `wc` 命令

`wc` 是 word count 的缩写。

它可以统计：

- 行数 lines
    
- 单词数 words
    
- 字节数 bytes
    

基本格式：

```bash
wc filename
```

---

## 20. 示例：统计文件内容

假设 `temp.txt` 内容如下：

```text
cherry
apple
x-ray
clock
orange
bananna
```

执行：

```bash
wc temp.txt
```

输出：

```bash
6  6  40 temp.txt
```

这三个数字分别表示：

|数字|含义|
|---|---|
|`6`|行数 lines|
|`6`|单词数 words|
|`40`|字节数 bytes|

所以：

```bash
wc temp.txt
```

输出格式是：

```text
行数 单词数 字节数 文件名
```

---

## 21. 只统计行数：`wc -l`

`-l` 表示 line count，只统计行数。

```bash
wc -l temp.txt
```

输出：

```bash
6 temp.txt
```

表示文件 `temp.txt` 有 6 行。

---

## 22. 只统计单词数：`wc -w`

`-w` 表示 word count，只统计单词数。

```bash
wc -w temp.txt
```

输出：

```bash
6 temp.txt
```

表示文件 `temp.txt` 有 6 个单词。

---

## 23. 只统计字节数：`wc -c`

`-c` 表示 byte count，只统计字节数。

```bash
wc -c temp.txt
```

输出：

```bash
40 temp.txt
```

表示文件 `temp.txt` 有 40 个 bytes。

注意：

```text
在英文纯文本中，byte 经常可以近似理解为字符数量。
但在中文、特殊符号或 UTF-8 编码中，一个字符可能占多个 bytes。
```

所以更准确地说，`wc -c` 统计的是字节数，不一定等于字符数。

---

## 24. 使用 `ls | wc -l` 统计目录项目数量

可以把 `ls` 和 `wc -l` 组合起来：

```bash
ls | wc -l
```

含义是：

```text
先用 ls 列出当前目录下的内容，
再用 wc -l 统计输出有多少行。
```

因为 `ls` 默认每个文件或目录占一行，所以：

```bash
ls | wc -l
```

可以用来估算当前目录下有多少个文件或目录。

执行流程：

```text
ls → wc -l
```

# 本节重点理解

这一节最重要的是掌握命令之间的组合方式。

Linux Shell 的强大之处不只是单个命令，而是可以把很多小工具连接起来。

例如：

```bash
history | grep cat > cat_history.txt
```

这个命令同时包含了三个操作：

```text
history：输出历史命令
grep cat：筛选包含 cat 的行
> cat_history.txt：把结果保存到文件
```

也就是：

```text
生成数据 → 筛选数据 → 保存数据
```

# 易混点总结

## 1. `|` 和 `>` 不一样

```bash
command1 | command2
```

表示：

```text
把 command1 的输出交给 command2。
```

```bash
command > file
```

表示：

```text
把 command 的输出写入 file。
```

---

## 2. `sort` 默认不是数字排序

例如有这些数字：

```text
1
7
8
32
48
56
```

如果不用 `-n`，`sort` 会按字符串方式排序，可能不是你想要的数字大小顺序。

数字排序应该用：

```bash
sort -n
```

数字倒序应该用：

```bash
sort -nr
```

---

## 3. `wc -c` 是字节数，不一定是字符数

尤其是中文文本中：

```text
一个中文字符通常不只占 1 byte。
```

所以 `wc -c` 更准确地说是统计 bytes，而不是普通意义上的字符数。

# 小结

本节学习了三个核心内容：

1. `|` 管道可以把一个命令的输出传给另一个命令。
    
2. `sort` 可以对文本或数字内容排序。
    
3. `wc` 可以统计行数、单词数和字节数。
    

最重要的思维方式是：

```text
Linux Shell 中的命令可以像积木一样组合。
```

例如：

```bash
history | grep cat | wc -l
```

这个命令的含义是：

```text
查看历史命令 → 找出包含 cat 的命令 → 统计有多少条
```

这就是 Linux 命令行中非常典型的处理方式。
