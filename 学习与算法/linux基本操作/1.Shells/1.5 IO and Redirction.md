# I/O and Redirection

## 1. 本节快速概览

在 Linux shell 中，很多命令或程序都会有输入和输出。

**I/O** 是：

```text
Input / Output
````

也就是输入与输出。

Linux 中，程序的输入和输出通常被看作 **streams**，即数据流。

最核心的三个 stream 是：

|Stream|英文全称|中文含义|默认来源或去向|
|---|---|---|---|
|`stdin`|standard input|标准输入|键盘|
|`stdout`|standard output|标准输出|屏幕|
|`stderr`|standard error|标准错误|屏幕|

本节主要讲如何改变这些 stream 的来源或去向。

常见操作包括：

|操作|符号或命令|作用|
|---|---|---|
|输出重定向|`>`|把 stdout 写入文件，覆盖原文件|
|追加输出|`>>`|把 stdout 追加到文件末尾|
|输入重定向|`<`|从文件读取 stdin|
|管道|`|`|
|错误重定向|`2>`|把 stderr 写入文件|
|合并 stderr 和 stdout|`2>&1`|把 stderr 合并到 stdout|
|同时显示和保存|`tee`|输出到屏幕，同时写入文件|

---

## 2. 什么是 Stream？

在 Linux 中，程序的输入和输出被称为：

```text
streams
```

可以理解为程序和外部环境之间的数据通道。

程序可能：

- 从键盘读取输入
    
- 把正常结果输出到屏幕
    
- 把错误信息输出到屏幕
    
- 从文件读取输入
    
- 把输出写入文件
    

Linux 将这些不同的数据流分成不同类别，最重要的是：

```text
stdin
stdout
stderr
```

---

## 3. stdin

`stdin` 的全称是：

```text
standard input
```

中文是标准输入。

它是进入程序的数据流。

默认情况下，`stdin` 来自键盘。

例如你运行一个程序：

```bash
./myprogram
```

程序要求你输入一个数字：

```text
Please enter a number:
```

你在键盘上输入：

```text
10
```

这个 `10` 就是通过 `stdin` 传入程序的。

---

## 4. stdout

`stdout` 的全称是：

```text
standard output
```

中文是标准输出。

它是程序正常输出结果的数据流。

默认情况下，`stdout` 会显示在屏幕上。

例如：

```bash
ls
```

如果当前目录中有：

```text
main.c  README.md  results.txt
```

这些内容就是 `ls` 的标准输出 `stdout`。

---

## 5. stderr

`stderr` 的全称是：

```text
standard error
```

中文是标准错误。

它是程序输出错误信息的数据流。

默认情况下，`stderr` 也会显示在屏幕上。

例如：

```bash
ls not_exist
```

如果文件不存在，可能输出：

```text
ls: cannot access 'not_exist': No such file or directory
```

这条错误信息通常来自 `stderr`，而不是 `stdout`。

---

## 6. 为什么要区分 stdout 和 stderr？

因为正常输出和错误输出的用途不同。

例如：

```bash
./myprogram
```

正常结果可能是：

```text
Simulation finished successfully.
Final energy = 123.45
```

错误信息可能是：

```text
Error: input file not found.
```

如果把它们分开，就可以：

- 正常结果保存到一个文件
    
- 错误信息保存到另一个文件
    
- 调试时更容易找到问题
    
- 在 HPC 作业中更方便查看运行日志
    

---

## 7. 三个标准流的编号

Linux shell 中，这三个 stream 有对应编号：

|编号|Stream|含义|
|---|---|---|
|`0`|`stdin`|标准输入|
|`1`|`stdout`|标准输出|
|`2`|`stderr`|标准错误|

所以：

```bash
2> error.txt
```

中的 `2` 指的是 `stderr`。

通常：

```bash
> output.txt
```

等价于：

```bash
1> output.txt
```

因为默认重定向的是 `stdout`。

---

## 8. Output Redirection：输出重定向

输出重定向用于把程序的标准输出 `stdout` 保存到文件中，而不是显示在屏幕上。

使用符号：

```bash
>
```

基本格式：

```bash
command > output_file
```

例如：

```bash
example_program > my_output.txt
```

意思是：

- 运行 `example_program`
    
- 把正常输出 `stdout` 写入 `my_output.txt`
    
- 屏幕上不再显示这些正常输出
    

---

## 9. 示例：保存 `ls` 的输出

如果一个目录中文件太多，直接运行：

```bash
ls
```

可能会刷满屏幕，看不清楚。

可以把输出保存到文件：

```bash
ls > output_file.txt
```

这会创建一个文件：

```text
output_file.txt
```

里面保存 `ls` 的输出结果。

之后可以用文本编辑器打开，也可以用：

```bash
cat output_file.txt
```

查看内容。

---

## ==10. `>` 会覆盖原文件==

使用：

```bash
command > file
```

时，如果 `file` 不存在，会创建它。

如果 `file` 已经存在，会覆盖原来的内容。

例如：

```bash
echo "first line" > output.txt
echo "second line" > output.txt
```

最后 `output.txt` 中只会有：

```text
second line
```

因为第二次 `>` 覆盖了第一次的内容。

---

## 11. Append Redirection：追加输出

==如果不想覆盖原文件，而是把新输出追加到文件末尾，可以使用：==

```bash
>>
```

基本格式：

```bash
command >> output_file
```

例如：

```bash
ls >> output_file
```

意思是把 `ls` 的输出追加到 `output_file` 的末尾。

如果文件不存在，`>>` 也会创建文件。

---

## 12. `>` 和 `>>` 对比

|符号|英文关键词|文件不存在|文件已存在|
|---|---|---|---|
|`>`|overwrite redirection|创建文件|覆盖原内容|
|`>>`|append redirection|创建文件|追加到末尾|

示例：

```bash
echo "Run 1" > log.txt
echo "Run 2" >> log.txt
echo "Run 3" >> log.txt
```

最终 `log.txt` 内容是：

```text
Run 1
Run 2
Run 3
```

---

## 13. Input Redirection：输入重定向

输入重定向用于让程序从文件读取输入，而不是从键盘读取。

使用符号：

```bash
<
```

基本格式：

```bash
command < input_file
```

例如：

```bash
mycommand < programinput
```

意思是：

- `mycommand` 需要输入
    
- 输入内容不从键盘手动输入
    
- 而是从文件 `programinput` 中读取
    

---

## 14. 输入重定向的例子

假设有一个程序：

```bash
./sum_program
```

它要求用户输入两个数字。

正常交互可能是：

```text
3
5
```

如果把输入写在文件 `input.txt` 中：

```text
3
5
```

就可以运行：

```bash
./sum_program < input.txt
```

程序会从 `input.txt` 读取输入，而不是等你用键盘输入。

这在自动测试程序时非常有用。

---

## 15. Pipe：管道

管道符号是：

```bash
|
```

它的作用是：

> 把左边命令的 `stdout` 作为右边命令的 `stdin`。

基本格式：

```bash
command1 | command2
```

也就是说：

```text
command1 的输出 → command2 的输入
```

---

## 16. 教材例子：`cat programinput | mycommand`

教材中给出的例子是：

```bash
cat programinput | mycommand
```

含义是：

1. `cat programinput` 读取并输出 `programinput` 文件内容
    
2. 这个输出本来会进入屏幕，也就是 `stdout`
    
3. 管道 `|` 把这个 `stdout` 传给 `mycommand`
    
4. `mycommand` 把它当作 `stdin` 使用
    

所以它和下面命令有类似效果：

```bash
mycommand < programinput
```

---

## 17. `<` 和 `|` 的区别

|写法|含义|特点|
|---|---|---|
|`mycommand < input.txt`|直接让命令从文件读取输入|更直接|
|`cat input.txt|mycommand`|先用 `cat` 输出文件，再通过 pipe 传给命令|

对于简单文件输入，更推荐：

```bash
mycommand < input.txt
```

但在命令组合中，管道更灵活。

---

## 18. 常见 pipe 例子

### 例子 1：查看长输出的一部分

```bash
ls -lha | less
```

含义：

- `ls -lha` 产生很多输出
    
- `less` 分页查看输出
    

---

### 例子 2：筛选包含关键词的行

```bash
ls -lha | grep ".c"
```

含义：

- `ls -lha` 列出详细文件信息
    
- `grep ".c"` 只筛选包含 `.c` 的行
    

---

### 例子 3：统计文件行数

```bash
cat output.txt | wc -l
```

含义：

- `cat output.txt` 输出文件内容
    
- `wc -l` 统计行数
    

更直接的写法是：

```bash
wc -l < output.txt
```

或者：

```bash
wc -l output.txt
```

---

## 19. Error Redirection：错误重定向

普通的输出重定向：

```bash
command > output.txt
```

只会重定向 `stdout`。

它不会重定向 `stderr`，因为 `stderr` 是独立的数据流。

如果想把错误信息保存到文件中，需要使用：

```bash
2>
```

基本格式：

```bash
command 2> error_file
```

例如：

```bash
command 2> my_error_file
```

意思是：

- 正常输出仍然显示在屏幕上
    
- 错误输出 `stderr` 写入 `my_error_file`
    

---

## 20. 为什么是 `2>`？

因为在 Linux 中：

|编号|Stream|
|---|---|
|`0`|stdin|
|`1`|stdout|
|`2`|stderr|

所以：

```bash
2> error.txt
```

意思是把 stream 2，也就是 `stderr`，重定向到 `error.txt`。

同理：

```bash
1> output.txt
```

表示把 stream 1，也就是 `stdout`，重定向到 `output.txt`。

不过 `1>` 通常可以简写成：

```bash
> output.txt
```

---

## 21. 示例：只保存错误信息

执行：

```bash
ls existing_file missing_file > output.txt 2> error.txt
```

含义：

- `existing_file` 的正常输出进入 `output.txt`
    
- `missing_file` 的错误信息进入 `error.txt`
    

这样可以把正常结果和错误信息分开保存。

---

## 22. 合并 stdout 和 stderr

如果想把正常输出和错误输出都保存到同一个文件，可以使用：

```bash
2>&1
```

教材例子：

```bash
command > combined_output_file 2>&1
```

含义是：

1. `> combined_output_file` 先把 `stdout` 重定向到 `combined_output_file`
    
2. `2>&1` 再把 `stderr` 重定向到当前 `stdout` 所指向的位置
    
3. 最终 stdout 和 stderr 都进入 `combined_output_file`
    

---

## 23. `2>&1` 的含义

`2>&1` 可以拆开理解：

```text
2     stderr
>     redirect
&1    stdout 当前指向的位置
```

所以：

```bash
2>&1
```

意思是：

> 把 stderr 重定向到 stdout 当前的位置。

注意这里的 `&1` 不是文件名 `1`，而是指 file descriptor 1，也就是 stdout。

---

## 24. 顺序很重要

这条命令是正确的：

```bash
command > combined_output_file 2>&1
```

它表示：

1. stdout 先到文件
    
2. stderr 再跟随 stdout 到同一个文件
    

但是如果写成：

```bash
command 2>&1 > combined_output_file
```

含义会不同：

1. stderr 先被重定向到当前 stdout，也就是屏幕
    
2. stdout 后来才被重定向到文件
    

所以结果可能是：

- stdout 进入文件
    
- stderr 仍然显示在屏幕
    

因此常用写法是：

```bash
command > combined_output_file 2>&1
```

---

## 25. Bash 中更简洁的写法

在 Bash 中，也常见：

```bash
command &> combined_output_file
```

含义是把 stdout 和 stderr 都写入同一个文件。

但是教材中的写法：

```bash
command > combined_output_file 2>&1
```

更通用，也更适合理解底层逻辑。

---

## 26. Redirect and Save Output：同时显示和保存

如果使用：

```bash
command > outputfile
```

输出会进入文件，但屏幕上不会显示正常输出。

对于长时间运行的程序，这可能不方便，因为你看不到程序运行进度。

这时可以使用：

```bash
tee
```

基本格式：

```bash
command | tee outputfile
```

它的作用是：

- 在屏幕显示输出
    
- 同时把输出保存到文件
    

---

## 27. `tee` 的例子

```bash
ls -lha | tee outputfile
```

含义：

- `ls -lha` 生成输出
    
- `tee outputfile` 把输出显示在屏幕上
    
- 同时保存到 `outputfile`
    

这适合需要一边观察结果、一边保留日志的情况。

---

## 28. `tee` 默认会覆盖文件

和 `>` 类似，`tee outputfile` 默认会覆盖原文件。

例如：

```bash
echo "Run 1" | tee log.txt
echo "Run 2" | tee log.txt
```

最后 `log.txt` 中通常只保留：

```text
Run 2
```

如果想追加，需要使用：

```bash
tee -a
```

例如：

```bash
echo "Run 1" | tee log.txt
echo "Run 2" | tee -a log.txt
```

最终 `log.txt` 中是：

```text
Run 1
Run 2
```

---

## 29. 捕获 stderr 并使用 tee

教材给出的命令是：

```bash
command 2>&1 | tee outputfile
```

含义是：

1. `command` 运行
    
2. `2>&1` 把 stderr 合并到 stdout
    
3. `| tee outputfile` 把合并后的输出同时显示在屏幕和保存到文件
    

这样正常输出和错误信息都会：

- 显示在屏幕上
    
- 保存到 `outputfile`
    

---

## 30. 为什么 `command 2>&1 | tee outputfile` 很有用？

如果你运行的是长时间程序，例如：

```bash
./simulation 2>&1 | tee run.log
```

你可以：

- 在屏幕上实时看到程序输出
    
- 同时把完整日志保存到 `run.log`
    
- 程序出错时，可以之后查看日志定位问题
    

这在 HPC 作业、实验记录和调试程序时非常常用。

---

## 31. 常见重定向写法总结

|命令|作用|
|---|---|
|`command > file`|把 stdout 写入文件，覆盖|
|`command >> file`|把 stdout 追加到文件|
|`command < file`|从文件读取 stdin|
|`command 2> error.txt`|把 stderr 写入文件，覆盖|
|`command 2>> error.txt`|把 stderr 追加到文件|
|`command > out.txt 2> err.txt`|stdout 和 stderr 分别保存|
|`command > all.txt 2>&1`|stdout 和 stderr 合并保存|
|`command 2>&1|tee log.txt`|
|`command|tee output.txt`|
|`command|tee -a output.txt`|

## 32. 一个完整例子：正常输出和错误输出分开保存

假设运行：

```bash
ls existing_file missing_file
```

其中：

- `existing_file` 存在
    
- `missing_file` 不存在
    

可以写：

```bash
ls existing_file missing_file > output.txt 2> error.txt
```

结果：

- `output.txt` 保存正常输出
    
- `error.txt` 保存错误信息
    

可以查看：

```bash
cat output.txt
cat error.txt
```

---

## 33. 一个完整例子：正常输出和错误输出合并保存

```bash
ls existing_file missing_file > combined.txt 2>&1
```

结果：

- 正常输出进入 `combined.txt`
    
- 错误信息也进入 `combined.txt`
    

查看：

```bash
cat combined.txt
```

---

## 34. 一个完整例子：合并输出并实时显示

```bash
ls existing_file missing_file 2>&1 | tee combined.txt
```

结果：

- 正常输出显示在屏幕
    
- 错误输出显示在屏幕
    
- 两者同时保存到 `combined.txt`

## 35. HPC / Slurm 场景中的应用

在 HPC 上运行程序时，I/O redirection 非常常用。

例如运行一个程序：

```bash
./stencil 1024 1000 > result.txt 2> error.txt
```

含义：

- 正常结果写入 `result.txt`
    
- 错误信息写入 `error.txt`
    

如果想把所有输出合并：

```bash
./stencil 1024 1000 > run.log 2>&1
```

如果想一边看输出，一边保存：

```bash
./stencil 1024 1000 2>&1 | tee run.log
```

---

## 36. Slurm 脚本中的例子

在 Slurm 作业脚本中，也经常设置输出文件：

```bash
#SBATCH --output=job_output.out
#SBATCH --error=job_error.err
```

这类似于把：

- stdout 保存到 `job_output.out`
    
- stderr 保存到 `job_error.err`
    

如果希望手动运行时保存日志，也可以写：

```bash
srun ./program > output.txt 2> error.txt
```

或者：

```bash
srun ./program > combined.log 2>&1
```

---

## 37. 程序调试时的推荐写法

如果正在调试程序，推荐使用：

```bash
./program 2>&1 | tee debug.log
```

优点：

- 屏幕能实时看到输出
    
- 日志文件中保留完整记录
    
- 错误信息不会丢失
    
- 之后可以把 `debug.log` 发给别人分析
    

如果是多次运行实验，可以使用追加模式：

```bash
./program 2>&1 | tee -a debug.log
```

## 38. 常见易错点

### 1. `>` 会覆盖文件

```bash
command > output.txt
```

如果 `output.txt` 已经存在，原内容会被清空。

如果想保留旧内容，应使用：

```bash
command >> output.txt
```

---

### 2. `>` 不会重定向错误信息

```bash
command > output.txt
```

只会保存 stdout。

如果程序报错，错误信息可能仍然显示在屏幕上。

要保存错误信息，需要：

```bash
command 2> error.txt
```

或者合并保存：

```bash
command > output.txt 2>&1
```

---

### 3. `2>&1` 的顺序很重要

推荐写法：

```bash
command > combined.txt 2>&1
```

不要随意写成：

```bash
command 2>&1 > combined.txt
```

二者效果不一定一样。

---

### 4. `tee` 默认覆盖文件

```bash
command | tee output.txt
```

会覆盖 `output.txt`。

如果想追加，应使用：

```bash
command | tee -a output.txt
```

---

### 5. Pipe 只传 stdout，不自动传 stderr

```bash
command | tee log.txt
```

默认只把 stdout 传给 `tee`。

如果想连 stderr 一起传，需要：

```bash
command 2>&1 | tee log.txt
```

## 40. 本节重点总结

这一节的核心是理解 Linux 中的三个标准数据流：

|Stream|编号|默认行为|
|---|---|---|
|`stdin`|`0`|从键盘读取|
|`stdout`|`1`|输出到屏幕|
|`stderr`|`2`|错误输出到屏幕|

重定向符号总结：

|符号|作用|
|---|---|
|`>`|stdout 写入文件，覆盖|
|`>>`|stdout 追加到文件|
|`<`|stdin 从文件读取|
|`|`|
|`2>`|stderr 写入文件，覆盖|
|`2>&1`|stderr 合并到 stdout|

## 41. 本节一句话总结

Linux I/O redirection 的核心是控制程序的输入和输出位置：`>` 和 `>>` 控制 stdout，`<` 控制 stdin，`2>` 控制 stderr，`|` 用于连接命令，而 `tee` 可以让输出既显示在屏幕上又保存到文件。

````

这一节对你后面做实验特别关键。以后运行程序或 Slurm 作业时，最实用的习惯是：

```bash
./program 2>&1 | tee run.log
````

它能让你一边看到程序输出，一边保存完整日志，调试和写实验报告时都很方便。
