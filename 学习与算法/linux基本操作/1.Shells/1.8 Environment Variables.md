# Linux Shell：Environment Variables 环境变量

本节主要介绍 Linux Shell 中的 **environment variables 环境变量**。

环境变量可以理解为：

```text
Shell 和程序运行时可以读取的一些“配置值”。
````

例如：

- `PATH`：告诉 Shell 去哪些目录里找命令
    
- `HOME`：当前用户的 home 目录
    
- `SHELL`：当前使用的 shell 类型
    

本节重点是理解：

```text
变量可以保存信息，而环境变量可以被当前 Shell 以及它启动的子 Shell 或程序使用。
```

---

## 1. 什么是 Environment Variable？

`Environment Variable` 是 Shell 环境中的变量。

它通常用于保存一些系统或用户相关的信息，例如：

|环境变量|含义|
|---|---|
|`PATH`|命令搜索路径|
|`HOME`|当前用户的 home 目录|
|`SHELL`|当前用户使用的 shell|
|`USER`|当前用户名，部分系统中可用|

可以把环境变量理解成：

```text
Shell 的全局配置信息。
```

很多命令和程序会根据这些变量来决定如何运行。

---

# Part 1：PATH 环境变量

## 2. `PATH` 是什么？

`PATH` 是一个非常重要的环境变量。

它保存了一组目录路径。

当你在 Shell 中输入一个命令时，Shell 会按照 `PATH` 中列出的目录顺序去寻找这个命令。

例如你输入：

```bash
ls
```

Shell 会去 `PATH` 中的目录里查找有没有叫 `ls` 的可执行文件。

---

## 3. 查看当前 `PATH`

可以使用：

```bash
echo $PATH
```

示例输出可能是：

```bash
/usr/local/bin:/usr/bin:/bin
```

这里表示 Shell 会依次在这些目录中查找命令：

```text
/usr/local/bin
/usr/bin
/bin
```

---

## 4. `$PATH` 中的 `$` 是什么？

当使用变量的值时，需要在变量名前加 `$`。

```bash
echo $PATH
```

意思是：

```text
显示 PATH 变量中保存的值。
```

注意区别：

|写法|含义|
|---|---|
|`PATH`|变量名本身|
|`$PATH`|变量中保存的值|

所以：

```bash
echo PATH
```

会输出文本 `PATH`。

而：

```bash
echo $PATH
```

会输出 `PATH` 变量真正保存的目录列表。

---

## 5. `PATH` 中的冒号 `:` 是什么？

在 `PATH` 中，不同目录之间用冒号 `:` 分隔。

例如：

```bash
/usr/local/bin:/usr/bin:/bin
```

可以理解为：

```text
/usr/local/bin
然后 /usr/bin
然后 /bin
```

也就是说：

```text
: 用于连接多个目录路径。
```

---

## 6. Shell 如何根据 `PATH` 找命令？

当输入一个命令，例如：

```bash
python
```

Shell 会按照 `PATH` 的顺序查找：

```text
先查 /usr/local/bin
再查 /usr/bin
再查 /bin
```

如果找到了可执行文件，就运行它。

如果找不到，就会出现类似：

```bash
command not found
```

这说明：

```text
该命令不在 PATH 指定的目录中，Shell 无法直接找到它。
```

---

## 7. 如果命令不在 `PATH` 中怎么办？

如果一个程序不在 `PATH` 中，你仍然可以用完整路径执行它。

例如程序位于：

```bash
/home/jolo/mytools/mycommand
```

可以直接运行：

```bash
/home/jolo/mytools/mycommand
```

或者如果你当前就在它所在目录下，可以使用：

```bash
./mycommand
```

其中：

```text
./ 表示当前目录。
```

---

# Part 2：修改 PATH

## 8. 向 `PATH` 添加新目录

如果你希望 Shell 能在新目录中查找命令，可以把这个目录加入 `PATH`。

基本格式：

```bash
export PATH=$PATH:/path/to/new/command
```

例如：

```bash
export PATH=$PATH:/home/jolo/mytools
```

含义是：

```text
保留原来的 PATH，
然后在末尾追加 /home/jolo/mytools。
```

---

## 9. 解释 `export PATH=$PATH:/path/to/new/command`

这个命令可以拆开理解：

```bash
export PATH=$PATH:/path/to/new/command
```

|部分|含义|
|---|---|
|`export`|把变量导出为环境变量|
|`PATH=`|给 PATH 重新赋值|
|`$PATH`|原来 PATH 的值|
|`:`|连接不同目录|
|`/path/to/new/command`|要添加的新目录|

所以它不是直接覆盖，而是：

```text
旧 PATH + 新目录
```

---

## 10. 为什么右边要写 `$PATH`？

例如：

```bash
export PATH=$PATH:/home/jolo/mytools
```

右边的 `$PATH` 表示：

```text
保留当前已有的 PATH 内容。
```

如果不写 `$PATH`，就会直接替换掉原来的路径列表。

---

## 11. 把新目录加到 PATH 末尾

```bash
export PATH=$PATH:/home/jolo/mytools
```

这表示：

```text
先搜索原来的目录，
最后再搜索 /home/jolo/mytools。
```

搜索顺序是：

```text
原来的 PATH → /home/jolo/mytools
```

---

## 12. 把新目录加到 PATH 开头

如果想让新目录最先被搜索，可以写成：

```bash
export PATH=/home/jolo/mytools:$PATH
```

这表示：

```text
先搜索 /home/jolo/mytools，
再搜索原来的 PATH。
```

搜索顺序是：

```text
/home/jolo/mytools → 原来的 PATH
```

这在你想优先使用自己安装或自己编译的程序时很有用。

---

## 13."="两边不能有空格

设置变量时，等号两边不能加空格。

正确：

```bash
export PATH=$PATH:/home/jolo/mytools
```

错误：

```bash
export PATH = $PATH:/home/jolo/mytools
```

错误原因是：

```text
Shell 会把 PATH、=、路径分别当成不同的东西，而不是变量赋值。
```

---

# Part 3：完全替换 PATH

## 14. Redefine path 重新定义 PATH

如果想完全替换原来的 `PATH`，可以写：

```bash
export PATH=/path/to/replacement/directory
```

例如：

```bash
export PATH=/home/jolo/mytools
```

这表示：

```text
PATH 中只剩下 /home/jolo/mytools。
```

---

## 15. 完全替换 PATH 的风险

下面这个命令要非常小心：

```bash
export PATH=/path/to/replacement/directory
```

它会：

```text
完全擦掉原来的 PATH。
```

如果你把系统常用目录删除了，例如：

```text
/usr/bin
/bin
```

那么很多普通命令可能会找不到，例如：

```bash
ls
cat
grep
python
```

可能会出现：

```bash
command not found
```

所以通常更推荐使用追加方式：

```bash
export PATH=$PATH:/new/path
```

而不是完全替换：

```bash
export PATH=/new/path
```

---

## 16. 如果 PATH 被改坏了怎么办？

如果不小心把 `PATH` 改坏了，可以尝试临时恢复：

```bash
export PATH=/usr/local/bin:/usr/bin:/bin
```

这会恢复一些常见系统路径。

如果命令已经无法直接运行，也可以使用完整路径，例如：

```bash
/bin/ls
/usr/bin/grep
```

# Part 4：查看所有环境变量

## 17. 使用 `env` 查看环境变量

可以使用：

```bash
env
```

列出当前 Shell 环境中的所有环境变量。

示例中可能会看到：

```text
SHELL=/bin/bash
HOME=/home/jolo
PATH=/usr/local/bin:/usr/bin:/bin
```

这些都是常见的内置环境变量。

---

## 18. 常见环境变量

|环境变量|示例|含义|
|---|---|---|
|`SHELL`|`/bin/bash`|当前默认 Shell|
|`HOME`|`/home/jolo`|用户主目录|
|`PATH`|`/usr/local/bin:/usr/bin:/bin`|命令搜索路径|

查看某个变量可以使用：

```bash
echo $HOME
```

或者：

```bash
echo $SHELL
```

---

# Part 5：Shell Variable 和 Environment Variable

## 19. 普通 Shell 变量

可以在 Shell 中定义一个变量：

```bash
MYVAR=something
```

这表示：

```text
创建一个名为 MYVAR 的变量，它的值是 something。
```

注意：

```text
设置变量时，变量名前不加 $。
```

正确：

```bash
MYVAR=something
```

错误：

```bash
$MYVAR=something
```

---

## 20. 使用变量的值

使用变量时，需要加 `$`：

```bash
echo $MYVAR
```

输出：

```text
something
```

所以规则是：

```text
设置变量时不用 $
使用变量时要加 $
```

---

## 21. Shell 变量和环境变量的区别

|类型|作用范围|
|---|---|
|Shell variable|只在当前 Shell 中有效|
|Environment variable|当前 Shell 以及它启动的子 Shell 或程序都可以使用|

也就是说：

```text
普通变量只属于当前 Shell。
export 后的变量可以传递给子进程。
```

---

## 22. 什么是 subshell？

`subshell` 是当前 Shell 启动出来的子 Shell。

例如你在当前终端里输入：

```bash
bash
```

就进入了一个新的子 Shell。

如果变量没有 `export`，子 Shell 通常看不到它。

如果变量被 `export`，子 Shell 可以继承它。

---

# Part 6：export 命令

## 23. 使用 `export` 导出变量

如果已经有一个普通变量：

```bash
MYVAR=something
```

可以用：

```bash
export MYVAR
```

把它变成环境变量。

完整流程：

```bash
MYVAR=something
export MYVAR
```

---

## 24. 创建并导出变量

也可以一行完成创建和导出：

```bash
export MYVAR2=something_else
```

含义是：

```text
创建 MYVAR2，并把它导出为环境变量。
```

---

## 25. 为什么需要 export？

`export` 的作用是：

```text
让变量进入当前 Shell 的 environment，
并传递给之后从该 Shell 启动的子 Shell 或程序。
```

例如：

```bash
MYVAR=something
```

只在当前 Shell 中有效。

而：

```bash
export MYVAR=something
```

可以被当前 Shell 启动的子进程继承。

---

## 26. 变量名通常使用大写

按照习惯，Shell 变量和环境变量通常使用大写字母：

```bash
MYVAR=something
DATAPATH=/home/jolo/Project
```

这不是强制规定，但非常常见。

---

# Part 7：在命令中使用变量

## 27. 使用 `$MYVAR`

变量可以插入到命令中。

例如：

```bash
DATAPATH=/home/jolo/Project
```

然后：

```bash
ls $DATAPATH
```

等价于：

```bash
ls /home/jolo/Project
```

---

## 28. 示例：使用变量保存路径

```bash
DATAPATH=/home/jolo/Project
ls $DATAPATH
```

含义是：

```text
先把 /home/jolo/Project 保存到 DATAPATH，
然后用 ls 查看这个目录。
```

这样做的好处是：

```text
路径很长时，不需要重复输入完整路径。
```

---

## 29. 使用 `${MYVAR}` 更安全

如果变量和其他字符连在一起，建议使用：

```bash
${MYVAR}
```

而不是：

```bash
$MYVAR
```

因为 `${MYVAR}` 可以明确告诉 Shell：

```text
变量名从哪里开始，到哪里结束。
```

---

## 30. 示例：复制文件到变量指定目录

课本示例：

```bash
DATAPATH=/home/jolo/Project
ls $DATAPATH
cp newdatafile.txt ${DATAPATH}/todaysdatafile.txt
ls $DATAPATH
```

解释：

第一行：

```bash
DATAPATH=/home/jolo/Project
```

定义变量 `DATAPATH`，值为 `/home/jolo/Project`。

第二行：

```bash
ls $DATAPATH
```

查看该目录内容。

第三行：

```bash
cp newdatafile.txt ${DATAPATH}/todaysdatafile.txt
```

把 `newdatafile.txt` 复制到 `/home/jolo/Project` 目录下，并重命名为 `todaysdatafile.txt`。

第四行：

```bash
ls $DATAPATH
```

再次查看该目录，应该可以看到：

```bash
todaysdatafile.txt
```

---

## 31. `${DATAPATH}/todaysdatafile.txt` 的作用

```bash
${DATAPATH}/todaysdatafile.txt
```

等价于：

```bash
/home/jolo/Project/todaysdatafile.txt
```

这里用 `${DATAPATH}` 是为了避免 Shell 误解变量名。

如果写成：

```bash
$DATAPATH/todaysdatafile.txt
```

在这个例子中通常也可以工作。

但如果变量后面紧跟字母或数字，使用 `{}` 会更安全。

# 易混点总结

## 1. 设置变量不用 `$`，使用变量要用 `$`

设置变量：

```bash
MYVAR=something
```

使用变量：

```bash
echo $MYVAR
```

不要写成：

```bash
$MYVAR=something
```

---

## 2. `PATH=$PATH:/new/path` 不是重复废话

这里右边的 `$PATH` 表示旧值。

```bash
export PATH=$PATH:/new/path
```

意思是：

```text
在旧 PATH 后面追加一个新目录。
```

如果写成：

```bash
export PATH=/new/path
```

意思就变成：

```text
完全替换旧 PATH。
```

---

## 3. `:` 是 PATH 中的分隔符

```bash
/usr/local/bin:/usr/bin:/bin
```

表示三个目录：

```text
/usr/local/bin
/usr/bin
/bin
```

不是一个完整的单一路径。

---

## 4. `export` 的作用不是简单赋值

```bash
MYVAR=something
```

只是创建普通 Shell 变量。

```bash
export MYVAR=something
```

创建变量并让它成为环境变量，可以传递给子 Shell 或程序。

---

## 5. `PATH` 顺序会影响命令执行

如果两个目录里都有同名命令，Shell 会执行 `PATH` 中更靠前目录里的那个。

例如：

```bash
export PATH=/home/jolo/mytools:$PATH
```

会优先搜索：

```bash
/home/jolo/mytools
```

所以这可能改变实际执行的程序版本。


```bash
backup.sh
```

而不用写：

```bash
$HOME/scripts/backup.sh
```

前提是该脚本有执行权限。

在原有 PATH 前面追加新目录。

```bash
MYVAR=something
```

设置变量。

```bash
echo $MYVAR
```

使用变量。

```bash
export MYVAR=something
```
