# Shells

## 1. 什么是 Shell？

**Shell** 是用户与 Linux / Unix 操作系统交互的重要接口。

它的主要作用是：

```text
Interpret commands for the Operating System to execute.
````

也就是说，用户在 shell 中输入命令，shell 负责解释这些命令，然后把任务交给操作系统执行。

可以简单理解为：

```text
User → Shell → Operating System / Kernel → Hardware
```

Shell 不只是一个命令输入窗口，它也是一个：

- **command interpreter**：命令解释器；
    
- **scripting language**：脚本语言；
    
- **interactive environment**：交互式环境；
    
- **customizable environment**：可自定义环境。
    

---

## 2. 常见 Shell 类型

Linux 和 Unix 中有很多不同的 shell，每种 shell 都有自己的优点和缺点。

常见 shell 包括：

|Shell|说明|
|---|---|
|`sh`|Bourne shell，早期 Unix shell|
|`bash`|Bourne Again Shell，`sh` 的增强版本|
|`csh`|C shell，语法风格接近 C 语言|
|`tcsh`|`csh` 的增强版本|
|`zsh`|功能强大的现代 shell，常见于 macOS 和开发者环境|

在大多数 Linux distributions 中，默认 shell 通常是：

```text
bash
```

也就是 **Bourne Again Shell**。

---

## 3. Bash

**Bash** 是 Linux 中最常用的 shell 之一。

它是 `sh` 的更新和增强版本，提供了更多交互功能和脚本功能。

Bash 的特点包括：

- 支持命令执行；
    
- 支持 shell scripting；
    
- 支持变量；
    
- 支持条件判断；
    
- 支持循环；
    
- 支持 tab completion；
    
- 支持环境变量；
    
- 支持命令历史记录；
    
- 支持用户自定义配置。
    

本教程中的所有例子都使用：

```text
bash shell
```

所以学习时可以优先熟悉 bash。

---

## 4. Shell 的两个主要用途

Shell 主要有两个用途：

## 4.1 交互式使用

用户可以直接输入命令并立即得到结果。

例如：

```bash
ls
pwd
cd Documents
echo hello
```

这种方式叫做：

```text
interactive shell
```

也就是交互式 shell。

---

## 4.2 编写脚本

由于 bash 和其他 shell 本身也是 scripting languages，因此可以用 shell 编写脚本。

例如，可以创建一个文件：

```bash
run.sh
```

内容如下：

```bash
#!/bin/bash

echo "Start running program"
python analyse.py
echo "Finished"
```

然后执行：

```bash
bash run.sh
```

在 HPC 中，shell script 非常重要，例如常见的 Slurm 作业脚本：

```bash
#!/bin/bash

#SBATCH --job-name=test
#SBATCH --time=00:10:00
#SBATCH --nodes=1

module load gcc
srun ./program
```

这说明 shell 不只是用来输入命令，也可以用来组织和自动化计算任务。

# Introduction to Shells

## 5. Shell Prompt

**Shell prompt** 是 shell 等待用户输入命令的位置。

在技术文档中，prompt 通常用 `$` 表示，例如：

```bash
$ echo hello
```

这里要注意：

```text
不要输入 $ 符号。
```

==`$` 只是提示符，表示后面的内容是你应该输入的命令==。

真正需要输入的是：

```bash
echo hello
```

不是：

```bash
$ echo hello
```

---

## 6. Shell 命令的基本格式

Shell 中运行命令通常遵循以下格式：

```bash
<command> <option(s)> <argument(s)>
```

也就是：

```text
命令 + 选项 + 参数
```

例如：

```bash
ls -l /home/leyan
```

可以拆解为：

|部分|内容|作用|
|---|---|---|
|command|`ls`|要运行的程序|
|option|`-l`|改变命令行为，显示详细信息|
|argument|`/home/leyan`|命令作用的目标路径|

---

## 7. Command

**Command** 是要执行的程序、软件包或可执行文件。

例如：

```bash
ls
echo
cat
python
gcc
```

这些都是 command。

如果命令来自系统本身或通过 package manager 安装，一般不需要写完整路径。  
例如：

```bash
ls
python
gcc
```

这是因为这些程序通常位于：

```text
/bin
/usr/bin
```

并且已经被加入环境变量：

```text
$PATH
```

---

## 8. 运行自己的程序时需要路径

如果你运行的是自己编译或自己写的程序，通常需要提供：

- **full path**：绝对路径；
    
- **relative path**：相对路径。
    

例如，你有一个当前目录下的可执行文件：

```bash
program
```

通常不能直接运行：

```bash
program
```

而需要写成：

```bash
./program
```

这里的 `./` 表示：

```text
current directory
```

也就是当前目录。

再例如，使用完整路径运行：

```bash
/home/leyan/project/program
```

或者使用相对路径：

```bash
../project/program
```

---

## 9. $PATH 环境变量

`$PATH` 是一个非常重要的 **environment variable**。

它告诉 shell：

```text
当用户输入一个命令时，应该去哪些目录中寻找可执行程序。
```

可以使用下面命令查看 `$PATH`：

```bash
echo $PATH
```

输出可能类似：

```text
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
```

不同目录之间用冒号 `:` 分隔。

当你输入：

```bash
python
```

shell 会在 `$PATH` 指定的目录中查找名为 `python` 的可执行文件。

如果找到了，就执行它；如果找不到，就会出现类似错误：

```text
command not found
```

---

## 10. Option / Flag

**Option** 也常被称为 **flag**。

它用于改变 command 的行为。

Option 通常以：

```text
-
```

或者：

```text
--
```

开头。

例如：

```bash
ls -l
```

这里 `-l` 表示使用 long listing format，也就是显示详细信息。

再如：

```bash
ls -a
```

这里 `-a` 表示显示隐藏文件。

也可以组合：

```bash
ls -la
```

等价于：

```bash
ls -l -a
```

---

## 11. 短选项和长选项

Linux 命令中常见两种选项格式：

|类型|示例|说明|
|---|---|---|
|short option|`-h`|单个字母，前面一个 `-`|
|long option|`--help`|完整单词，前面两个 `--`|

例如：

```bash
ls --help
```

或者：

```bash
python --version
```

一般来说，`--help` 常用于查看命令帮助信息。

---

## 12. 有些 Option 需要 Argument

有些 option 后面必须跟一个参数。

例如：

```bash
gcc -o main main.c
```

这里：

|部分|说明|
|---|---|
|`gcc`|command|
|`-o`|option|
|`main`|`-o` 的 argument，表示输出文件名|
|`main.c`|要编译的源文件|

如果只写：

```bash
gcc -o main.c
```

通常是不完整或含义错误的，因为 `-o` 后面的参数被解释为输出文件名。

再例如：

```bash
python script.py
```

这里 `script.py` 是 `python` 命令的 argument。

---

## 13. Argument

**Argument** 是传递给 command 的输入对象或参数。

Argument 的含义取决于具体命令。

例如：

```bash
cat notes.txt
```

这里 `notes.txt` 是 argument，表示要显示的文件。

```bash
mkdir test_folder
```

这里 `test_folder` 是 argument，表示要创建的目录名。

```bash
cp a.txt b.txt
```

这里有两个 arguments：

|Argument|作用|
|---|---|
|`a.txt`|源文件|
|`b.txt`|目标文件|

---

## 14. 文件名参数需要路径

如果文件不在当前目录中，就需要给出路径。

例如，当前目录下有：

```text
notes.txt
```

可以直接运行：

```bash
cat notes.txt
```

但是如果文件在 `Documents` 目录中，就需要：

```bash
cat Documents/notes.txt
```

或者使用绝对路径：

```bash
cat /home/leyan/Documents/notes.txt
```

---

## 15. 查看当前使用的 Shell

可以使用下面命令查看当前默认 shell：

```bash
echo $SHELL
```

示例输出：

```bash
/bin/bash
```

这里：

|部分|含义|
|---|---|
|`echo`|command，用来打印内容|
|`$SHELL`|argument，是一个环境变量|
|`/bin/bash`|输出结果，表示 bash 的安装位置|

`$SHELL` 是一个 environment variable，保存当前用户的默认 shell 路径。

---

## 16. echo 命令

`echo` 用于把后面的内容输出到屏幕。

例如：

```bash
echo hello
```

输出：

```text
hello
```

再如：

```bash
echo $SHELL
```

输出当前 shell 路径。

还可以查看用户名：

```bash
echo $USER
```

查看 home directory：

```bash
echo $HOME
```

## 17. Tab Completion

**Tab completion** 是 bash 中非常有用的功能。

它允许你输入命令、文件名或目录名的一部分，然后按 `Tab` 键自动补全。

例如，你想输入：

```bash
python
```

可以先输入：

```bash
py
```

然后按 `Tab`。

如果没有歧义，bash 会自动补全命令。  
如果有多个可能选项，再按一次 `Tab`，shell 会列出所有候选项。

例如输入：

```bash
py
```

然后按两次 `Tab`，可能看到：

```text
pydoc
python
python3
python3.11
pygmentize
```

---

## 18. Tab Completion 可以补全什么？

Tab completion 不仅可以补全 command，也可以补全：

- commands；
    
- file names；
    
- directory names；
    
- paths；
    
- 有些情况下也可以补全 package 或参数。
    

例如，当前目录下有文件：

```text
experiment_result_2026.csv
```

你可以输入：

```bash
cat exp
```

然后按 `Tab`，它可能自动补全为：

```bash
cat experiment_result_2026.csv
```

如果当前目录下有多个以 `exp` 开头的文件：

```text
experiment_result_2026.csv
experiment_log.txt
```

按一次 `Tab` 可能不会补全，因为存在歧义。  
再按一次 `Tab`，shell 会显示可能选项。

---

## 19. Tab Completion 的实际价值

Tab completion 可以显著提高命令行效率。

它的优点包括：

- 减少打字量；
    
- 避免拼写错误；
    
- 快速查看可能的命令或文件；
    
- 更方便地输入长路径；
    
- 在 HPC 和服务器环境中尤其有用。
    

例如，在 HPC 项目中，文件名可能很长：

```text
strong_scaling_result_nodes_4_tasks_128.csv
```

使用 tab completion 可以避免手动输入完整文件名。

## 20. Manual Pages

如果想查看某个命令有哪些 options，可以阅读：

```text
Manual Pages
```

通常使用 `man` 命令。

例如：

```bash
man ls
```

会打开 `ls` 命令的手册页。

也可以用：

```bash
ls --help
```

查看简短帮助。

一般来说：

|方法|适合场景|
|---|---|
|`command --help`|快速查看常用选项|
|`man command`|查看更完整的官方说明|

例如：

```bash
man echo
man cp
man grep
man tar
```

## 21. 常见命令格式例子

### 例子一：列出文件

```bash
ls -l
```

|部分|含义|
|---|---|
|`ls`|command|
|`-l`|option，显示详细列表|
|无|没有显式 argument，默认作用于当前目录|

---

### 例子二：显示文件内容

```bash
cat notes.txt
```

|部分|含义|
|---|---|
|`cat`|command|
|`notes.txt`|argument，目标文件|

---

### 例子三：复制文件

```bash
cp input.txt output.txt
```

|部分|含义|
|---|---|
|`cp`|command|
|`input.txt`|argument，源文件|
|`output.txt`|argument，目标文件|

---

### 例子四：编译 C 程序

```bash
gcc -O3 -o main main.c
```

|部分|含义|
|---|---|
|`gcc`|command，C 编译器|
|`-O3`|option，优化级别|
|`-o`|option，指定输出文件|
|`main`|`-o` 的 argument，输出文件名|
|`main.c`|argument，输入源文件|

---

### 例子五：运行当前目录下的程序

```bash
./main
```

|部分|含义|
|---|---|
|`./main`|command，当前目录下的可执行文件|

这里 `./` 很重要，因为当前目录通常不在 `$PATH` 中。

### 练习 3

为什么运行当前目录下的程序时常写成：

```bash
./program
```

答案：

```text
因为当前目录通常不在 $PATH 中，shell 不会自动在当前目录查找可执行文件。
./ 表示 current directory，因此 ./program 明确告诉 shell 去当前目录运行 program。
```
