# Linux Shell：Job Control 作业控制

本节主要介绍 Linux Shell 中的 **job control 作业控制**。

在 Shell 中，我们不仅可以启动命令，还可以对正在运行的程序进行基本控制，例如：

- 查看进程
- 中断进程
- 暂停进程
- 将进程放到后台运行
- 将后台任务恢复到前台
- 杀死进程

这一节的核心是理解：

```text
Shell 不只是执行命令，也可以控制正在运行的任务。
````

---

## 1. 什么是 Job Control？

`Job Control` 指的是 Shell 对当前会话中运行任务的控制能力。

它允许用户在 Shell 中对任务进行：

|操作|英文|含义|
|---|---|---|
|停止|stop / terminate|结束正在运行的进程|
|暂停|suspend|临时挂起进程|
|恢复|resume|让暂停的进程继续运行|
|后台运行|background|让进程在后台继续执行|
|前台运行|foreground|把后台任务带回当前终端|

---

## 2. 为什么需要 Job Control？

有些程序可能运行很久，例如：

```bash
./longscript.sh
```

或者：

```bash
./mylong-runningcode.o
```

如果它一直占用当前 Shell，那么你就不能继续在同一个终端里输入其他命令。

这时就可以使用 job control：

```text
让长时间运行的程序去后台运行，
然后你继续在同一个 Shell 中输入其他命令。
```

它也适用于：

- 程序运行时间太长
    
- 程序卡住
    
- 程序因为 bug 无法结束
    
- 想暂停程序后稍后恢复
    
- 想查看和管理当前 Shell 中的任务

# Part 1：查看进程

## 3. 使用 `ps` 查看当前进程

`ps` 是 process status 的缩写，用于查看当前进程。

```bash
ps
```

示例输出：

```bash
$ ps
  PID TTY          TIME CMD
 4621 pts/7    00:00:00 bash
32273 pts/7    00:00:00 ps
```

---

## 4. `ps` 输出解释

|字段|含义|
|---|---|
|`PID`|Process ID，进程编号|
|`TTY`|当前进程关联的终端|
|`TIME`|该进程占用的 CPU 时间|
|`CMD`|启动该进程的命令|

在上面的例子中：

```bash
4621 pts/7 00:00:00 bash
```

表示当前 Shell 本身是一个 `bash` 进程，它的 PID 是 `4621`。

```bash
32273 pts/7 00:00:00 ps
```

表示刚刚运行的 `ps` 命令也是一个进程，它的 PID 是 `32273`。

---

## 5. PID 是什么？

`PID` 的全称是：

```text
Process ID
```

也就是进程编号。

Linux 系统会给每个正在运行的进程分配一个唯一的 PID。

之后如果想杀死某个进程，就可以使用它的 PID：

```bash
kill PID
```

例如：

```bash
kill 32273
```

表示尝试终止 PID 为 `32273` 的进程。

---

# Part 2：中断和暂停进程

## 6. 使用 `Ctrl C` 中断进程

如果一个程序正在前台运行，可以使用：

```text
Ctrl C
```

向它发送 interrupt signal，中断信号。

通常情况下，程序收到这个信号后会终止运行。

例如你运行了：

```bash
./longscript.sh
```

如果不想让它继续运行，可以按：

```text
Ctrl C
```

它通常会停止执行。

---

## 7. `Ctrl C` 的作用

`Ctrl C` 的含义不是普通的复制操作。

在终端中，它表示：

```text
给当前前台进程发送 interrupt signal。
```

大多数命令收到 interrupt signal 后会退出。

但注意：

```text
有些程序可能不会立刻停止。
```

原因可能包括：

- 程序正在进行 I/O 操作
    
- 程序暂时无法响应信号
    
- 程序自己捕获并处理了该信号
    

所以 `Ctrl C` 通常能终止进程，但不是绝对立即生效。

---

## 8. 使用 `Ctrl Z` 暂停进程

如果你不想终止程序，只是想暂时把它挂起，可以使用：

```text
Ctrl Z
```

`Ctrl Z` 会 suspend 当前前台进程。

也就是：

```text
暂停它，但不杀死它。
```

例如：

```bash
./longscript.sh
```

程序正在运行时按：

```text
Ctrl Z
```

它会被暂停。

之后可以选择：

- 用 `bg` 放到后台继续运行
    
- 用 `fg` 恢复到前台运行
    
- 用 `kill` 终止它
    

---

## 9. `Ctrl C` 和 `Ctrl Z` 的区别

|操作|含义|结果|
|---|---|---|
|`Ctrl C`|interrupt|通常终止进程|
|`Ctrl Z`|suspend|暂停进程，不结束|

简单理解：

```text
Ctrl C：不要跑了，结束吧。
Ctrl Z：先停一下，之后还能继续。
```

---

# Part 3：后台运行任务

## 10. 使用 `bg` 恢复到后台运行

如果一个任务被 `Ctrl Z` 暂停了，可以使用：

```bash
bg
```

让它在后台继续运行。

`bg` 是 background 的缩写。

执行逻辑是：

```text
先 Ctrl Z 暂停任务，
再 bg 让它在后台继续运行。
```

---

## 11. 使用 `&` 直接在后台启动命令

如果你一开始就希望程序在后台运行，可以在命令结尾加上：

```bash
&
```

例如：

```bash
mylong-runningcode.o &
```

含义是：

```text
启动 mylong-runningcode.o，并立即把它放到后台运行。
```

这样当前 Shell 不会被它一直占用，你可以继续输入其他命令。

---

## 12. `&` 的作用

下面两个命令区别很大：

```bash
./longscript.sh
```

表示在前台运行。

运行期间，你通常不能继续输入其他命令，除非程序结束。

而：

```bash
./longscript.sh &
```

表示在后台运行。

程序继续执行，但 Shell 会立刻把控制权还给你。

---

## 13. 前台 foreground 和后台 background

|状态|英文|含义|
|---|---|---|
|前台|foreground|当前占用终端、接收键盘输入的任务|
|后台|background|在背后运行，不直接占用终端输入|

简单理解：

```text
前台任务：你正在和它交互。
后台任务：它自己在跑，你可以继续用 Shell。
```

---

# Part 4：把后台任务带回前台

## 14. 使用 `fg` 恢复到前台

`fg` 是 foreground 的缩写。

它可以把后台任务带回前台。

例如：

```bash
$ ./longscript.sh &
$ fg
./longscript.sh
```

解释：

第一行：

```bash
./longscript.sh &
```

把 `longscript.sh` 放到后台运行。

第二行：

```bash
fg
```

把它恢复到前台。

恢复到前台后，终端又会被这个程序占用，直到它结束、暂停或被中断。

---

## 15. `fg` 的典型使用场景

假设你运行了一个程序：

```bash
./longscript.sh
```

然后发现它会运行很久。

你可以：

```text
Ctrl Z
```

先暂停它。

然后：

```bash
bg
```

让它在后台继续运行。

如果之后想把它拿回前台查看或控制：

```bash
fg
```

---

# Part 5：查看当前 Shell 中的 jobs

## 16. 使用 `jobs` 查看后台任务

`jobs` 用于查看当前 Shell 管理的任务。

例如：

```bash
$ ./longscript.sh &
$ jobs
[1]+  Running                 ./longscript.sh &
```

输出表示：

```text
当前 Shell 中有一个后台任务正在运行。
```

---

## 17. `jobs` 输出解释

示例：

```bash
[1]+  Running                 ./longscript.sh &
```

|部分|含义|
|---|---|
|`[1]`|job number，任务编号|
|`+`|当前默认任务|
|`Running`|任务状态是正在运行|
|`./longscript.sh &`|原始命令|

这里的 `[1]` 不是 PID，而是 job number。

---

## 18. Job number 和 PID 的区别

|名称|示例|含义|
|---|---|---|
|PID|`4621`|系统级进程编号|
|Job number|`[1]`|当前 Shell 中的任务编号|

区别是：

```text
PID 是 Linux 系统给进程的编号。
Job number 是当前 Shell 给后台任务的编号。
```

在 job control 中，常见的是：

```bash
fg %1
```

表示把 job number 为 `1` 的任务带回前台。

也可以：

```bash
bg %1
```

表示让 job number 为 `1` 的任务在后台继续运行。

---

## 19. 多个后台任务时如何指定

如果只有一个后台任务，直接使用：

```bash
fg
```

通常就可以。

如果有多个后台任务，则需要指定具体任务。

例如：

```bash
jobs
```

输出：

```bash
[1]-  Running                 ./script1.sh &
[2]+  Running                 ./script2.sh &
```

可以使用：

```bash
fg %1
```

把第一个任务带回前台。

或者：

```bash
fg %2
```

把第二个任务带回前台。

---

# Part 6：top、htop 和 kill

## 20. 使用 `top` 查看系统进程

`top` 可以动态查看系统中正在运行的进程。

```bash
top
```

它会显示类似以下信息：

- 当前系统负载
    
- CPU 使用情况
    
- 内存使用情况
    
- 正在运行的进程
    
- 每个进程的 PID
    
- 每个进程的 CPU 和内存占用
    

`top` 适合用来观察系统当前是否被某些进程占用过多资源。

---

## 21. 使用 `htop` 交互式查看进程

`htop` 是 `top` 的一个更交互式版本。

```bash
htop
```

它通常比 `top` 更容易阅读和操作。

但是注意：

```text
htop 通常不是 Linux 系统默认安装的命令。
```

如果系统中没有安装，执行时可能会提示：

```bash
command not found
```

---

## 22. 使用 `kill` 停止进程

`kill` 命令用于向进程发送信号，常用于终止进程。

基本格式：

```bash
kill PID
```

例如：

```bash
kill 4621
```

表示向 PID 为 `4621` 的进程发送终止信号。

---

## 23. `kill` 的使用步骤

一般可以这样使用：

第一步，查找进程：

```bash
ps
```

或者：

```bash
ps -ef | grep bash
```

第二步，找到对应的 PID。

第三步，执行：

```bash
kill PID
```

例如：

```bash
kill 12345
```

---

## 24. `kill` 不一定表示“强制杀死”

虽然命令叫 `kill`，但它本质上是：

```text
向进程发送 signal。
```

默认情况下，`kill PID` 发送的是终止信号，通常会请求进程正常退出。

如果进程没有响应，才可能需要更强制的方式，例如：

```bash
kill -9 PID
```

==但 `kill -9` 会强制终止进程，一般不作为首选==。

推荐顺序是：

```text
先 Ctrl C
再 kill PID
最后才考虑 kill -9 PID
```

# 典型操作流程

## 场景 1：程序运行太久，想暂停后放到后台

假设你运行：

```bash
./longscript.sh
```

程序一直不结束。

可以按：

```text
Ctrl Z
```

暂停它。

然后输入：

```bash
bg
```

让它在后台继续运行。

之后可以输入：

```bash
jobs
```

查看它是否还在运行。

---

## 场景 2：一开始就想让程序后台运行

```bash
./longscript.sh &
```

然后你可以继续在当前 Shell 输入其他命令。

---

## 场景 3：后台任务需要重新拿回前台

```bash
jobs
```

查看任务：

```bash
[1]+  Running                 ./longscript.sh &
```

然后：

```bash
fg %1
```

把它带回前台。

---

## 场景 4：程序卡死，想结束它

先查看进程：

```bash
ps
```

或：

```bash
ps -ef | grep longscript
```

找到 PID 后：

```bash
kill PID
```

如果还是无法结束，再考虑：

```bash
kill -9 PID
```

# 易混点总结

## 1. `Ctrl C` 和 `Ctrl Z`

```text
Ctrl C：终止当前前台进程。
Ctrl Z：暂停当前前台进程。
```

---

## 2. `bg` 和 `fg`

```text
bg：background，让任务在后台运行。
fg：foreground，把任务带回前台。
```

---

## 3. `ps` 和 `jobs`

|命令|作用范围|
|---|---|
|`ps`|查看进程，关注 PID|
|`jobs`|查看当前 Shell 管理的任务，关注 job number|

简单理解：

```text
ps 更偏向系统进程视角。
jobs 更偏向当前 Shell 任务视角。
```

---

## 4. `&` 和 `bg`

|命令|使用时机|
|---|---|
|`command &`|启动时直接放到后台|
|`bg`|对已经暂停的任务，让它后台继续运行|

例如：

```bash
./script.sh &
```

是一开始就后台运行。

而：

```text
Ctrl Z
bg
```

是先暂停，再恢复到后台运行。

---

# 本节重点理解

这一节重点不是背很多命令，而是理解 Shell 对任务的控制方式。

最重要的几个命令和快捷键是：

```text
Ctrl C：中断前台进程
Ctrl Z：暂停前台进程
bg：后台继续运行
fg：恢复到前台
jobs：查看当前 Shell 的任务
ps：查看进程
kill：终止指定进程
```

可以记成一个流程：

```text
运行程序 → 程序太久 → Ctrl Z 暂停 → bg 后台运行 → jobs 查看 → fg 拿回前台 → kill 结束
```

# 小结

Job control 让用户可以在 Shell 中灵活管理任务。

当程序正在运行时：

- 想结束它，用 `Ctrl C`
    
- 想暂停它，用 `Ctrl Z`
    
- 想后台继续运行，用 `bg`
    
- 想一开始就后台运行，在命令后加 `&`
    
- 想查看当前 Shell 的任务，用 `jobs`
    
- 想拿回前台，用 `fg`
    
- 想通过 PID 终止进程，用 `kill`
    

这一节在日常使用 Linux、远程服务器、HPC 集群、运行长时间脚本时都非常重要。
