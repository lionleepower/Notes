# Files and Processes

## 1. Linux 中的基本思想

在 Linux 中，一个非常重要的思想是：

```text
Everything is either a file or a process.
````

也就是说，Linux 系统中的很多东西都可以被理解为两类：

1. **File**：文件
    
2. **Process**：进程
    

这句话是理解 Linux 的基础。  
用户创建、编辑、读取的是文件；程序运行起来之后，就变成进程。

---

## 2. Process

**Process** 指的是正在执行的程序。

一个程序本身只是存储在磁盘上的文件，例如：

```bash
python
gcc
vim
a.out
```

但是当程序被运行之后，它就会成为一个 **process**。

例如：

```bash
python script.py
```

这条命令会启动一个 Python 解释器进程，并让它执行 `script.py`。

---

## 3. PID

每一个 process 都有一个唯一的编号，称为：

```text
PID = Process Identifier
```

也就是进程标识符。

Linux 使用 PID 来识别和管理不同进程。

例如，可以使用：

```bash
ps
```

查看当前 shell 中的进程。

也可以使用：

```bash
ps aux
```

查看系统中更完整的进程列表。

示例输出可能类似：

```text
USER       PID  %CPU  %MEM  COMMAND
leyan     1234   0.1   0.3  bash
leyan     1250   5.2   1.1  python script.py
```

其中：

- `PID` 是进程编号；
    
- `COMMAND` 是正在运行的命令或程序；
    
- `%CPU` 表示 CPU 使用率；
    
- `%MEM` 表示内存使用率。
    

---

## 4. Process 的生命周期

进程的运行时间可以很短，也可以很长。

### 短时间运行的进程

例如：

```bash
ls
```

这个命令会列出目录内容，执行完成后进程就结束了。

再如：

```bash
cat file.txt
```

它把文件内容打印到屏幕上，完成后也会退出。

### 长时间运行的进程

有些进程会持续运行很久，例如：

- 系统监控程序；
    
- 后台服务；
    
- 服务器程序；
    
- 长时间运行的科学计算任务；
    
- HPC 中提交到 compute node 上的大型并行程序。
    

例如：

```bash
python train_model.py
```

如果这是一个深度学习训练程序，它可能运行几个小时甚至几天。

---

## 5. File

**File** 是一组数据的集合。

在 Linux 中，file 不一定只是普通的文本文件。  
它可以是：

- 文本文档；
    
- 程序源代码；
    
- 可执行程序；
    
- 图片；
    
- PDF；
    
- 二进制数据；
    
- 配置文件；
    
- 设备接口；
    
- 系统内部信息。
    

每个 file 在文件系统中都有自己的位置，这个位置称为：

```text
path
```

---

## 6. Path

**Path** 指文件或目录在文件系统中的位置。

Linux 中的路径通常由多个目录名组成，并使用 `/` 分隔。

例如：

```bash
/home/leyan/Documents/notes/linux.md
```

可以拆成：

```text
/
home
leyan
Documents
notes
linux.md
```

其中：

- `/` 表示根目录，即 root directory；
    
- `home` 是根目录下的一个目录；
    
- `leyan` 是用户目录；
    
- `Documents` 是文档目录；
    
- `notes` 是笔记目录；
    
- `linux.md` 是文件名。
    

---

## 7. Directory

**Directory** 是一种特殊类型的 file。

它的作用是保存关于其他文件的信息。  
可以把 directory 理解成 Windows 或 macOS 中的 **folder**。

也就是说：

```text
Directory = special file that contains other files or directories
```

例如：

```bash
/home/leyan/Documents
```

这是一个 directory，它里面可能包含：

```text
linux.md
report.pdf
code/
images/
```

其中 `code/` 和 `images/` 也可以是 directories。

---

## 8. File、Directory 和 Path 的关系

可以用下面的结构理解：

```text
/
└── home
    └── leyan
        └── Documents
            ├── linux.md
            ├── report.pdf
            └── code
                └── main.c
```

对应路径为：

```bash
/home/leyan/Documents/linux.md
/home/leyan/Documents/report.pdf
/home/leyan/Documents/code/main.c
```

在 Linux 中，目录也是文件的一种，只不过它专门用于组织其他文件。

---

## 9. File Name

每个文件都必须有一个名字，因为操作系统通过文件名来识别文件。

Linux 中的文件名可以包含很多字符，但实际使用时应该避免一些特殊字符，例如：

- 空格；
    
- 引号；
    
- 括号；
    
- 星号；
    
- 问号；
    
- 反斜杠；
    
- 分号。
    

原因是这些字符在 shell 中可能有特殊含义，容易导致命令解析错误。

不推荐：

```text
my file (final).txt
```

更推荐：

```text
my_file_final.txt
```

或者：

```text
my-file-final.txt
```

---

## 10. 文件名长度

在大多数常见 Linux 系统中，文件名最长可以达到：

```text
255 characters
```

因此可以使用较有描述性的文件名。

例如，相比：

```text
a.txt
```

更推荐：

```text
experiment_result_2026_05_14.txt
```

在科研计算或 HPC 项目中，描述性文件名尤其重要，因为实验结果、参数配置和输出文件通常很多。

例如：

```text
mappo_qmix_predatorprey_seed1_return.csv
```

这个文件名能直接说明：

- 使用的算法是 MAPPO-QMIX；
    
- 环境是 PredatorPrey；
    
- 随机种子是 seed1；
    
- 存储的是 return 结果；
    
- 格式是 CSV。
    

---

## 11. File 的内容

Linux 文件本质上可以保存任意字节序列：

```text
A file can hold any sequence of bytes.
```

这意味着 Linux 本身不强制规定文件内容必须是什么格式。  
文件内容如何解释，取决于用户使用什么应用程序打开它。

例如：

|文件类型|内容解释方式|
|---|---|
|`.txt`|人类可读文本|
|`.c`|C 语言源代码|
|`.py`|Python 源代码|
|`.pdf`|PDF 阅读器解释|
|`.jpg`|图片查看器解释|
|`.exe` / 可执行文件|操作系统作为程序执行|
|`.bin`|二进制数据|
|`.h5`|HDF5 工具或库解释|

---

## 12. Human-readable Text Files

有些文件是人类可读的文本文件，通常按行组织。

例如：

```text
This is a Linux note.
Each line contains text.
```

常见例子包括：

```text
.txt
.md
.c
.py
.sh
.csv
```

这些文件可以用文本编辑器查看，例如：

```bash
nano file.txt
vim file.txt
cat file.txt
less file.txt
```

在 Obsidian 中使用的 Markdown 文件 `.md` 就是一种文本文件。

---

## 13. Structured Files

有些文件具有特定结构，需要专门程序才能正确解释。

例如：

- PDF 文件；
    
- PostScript 文件；
    
- Word 文档；
    
- 图片文件；
    
- 压缩包；
    
- 数据库文件。
    

这些文件虽然在 Linux 中仍然是 file，但不能简单地当作普通文本读取。

例如：

```bash
cat file.pdf
```

通常不会得到可读内容，因为 PDF 是一种结构化格式，需要 PDF reader 来解释。

---

## 14. Binary Files

在科学计算中，经常使用 **binary files**。

**Binary file** 是机器更容易读取和写入的字节序列，不一定适合人类直接阅读。

相比文本文件，binary files 的优点包括：

- 存储效率更高；
    
- 读写速度更快；
    
- 更适合大规模数值数据；
    
- 可以保留更精确的数据表示；
    
- 适合 HPC 和 scientific computing。
    

例如，一个大规模矩阵或三维模拟数据，如果保存为文本，文件会非常大，读写也很慢。  
保存为二进制格式则通常更高效。

---

## 15. Scientific Data Formats

科学计算中还有一些专门的数据格式，例如：

```text
NetCDF
HDF
HDF5
```

这些格式通常有固定的数据结构，并提供对应的：

```text
API = Application Programming Interface
```

也就是说，程序可以通过 API 来读取和写入这些文件。

例如，HDF5 常用于保存：

- 大规模数组；
    
- 多维科学数据；
    
- 模拟结果；
    
- 图像数据；
    
- 时间序列数据；
    
- HPC 程序输出。
    

一个 HDF5 文件可能包含多个 dataset，例如：

```text
simulation_output.h5
├── velocity
├── pressure
├── temperature
└── metadata
```

这类格式适合科学计算，因为它们可以高效地组织复杂数据。

## 16. Kernel 对 Process 和 File 的管理

Linux kernel 负责组织和管理 processes，并与 files 交互。

Kernel 的主要工作包括：

- 为每个 process 分配 CPU 时间；
    
- 为每个 process 分配 memory；
    
- 管理 file system；
    
- 处理 I/O；
    
- 管理进程之间的通信；
    
- 响应 system calls。
    

---

## 17. System Call

**System call** 是用户程序请求 kernel 提供服务的方式。

普通程序不能直接操作硬件或内核资源。  
如果程序想读取文件、写入文件、创建进程或申请内存，就需要通过 system call 请求 kernel。

例如，当程序执行：

```c
fopen("data.txt", "r");
```

底层最终会请求 kernel 打开文件。

当程序执行：

```bash
ls
```

kernel 会通过文件系统相关的 system calls 获取目录内容，然后 shell 把结果显示出来。

可以理解为：

```text
User program
↓
System call
↓
Kernel
↓
Hardware / File system
```

## 18. Linux 中“一切皆文件”的含义

Linux 使用 files 来表示系统中的很多东西，包括：

- 普通文件；
    
- 目录；
    
- 设备；
    
- kernel 内部信息；
    
- 配置；
    
- 进程信息；
    
- 输入输出接口。
    

例如：

```text
/dev
/proc
/sys
/etc
```

这些目录体现了 Linux “everything is a file” 的思想。

### `/dev`

`/dev` 中包含设备文件，例如：

```text
/dev/sda
/dev/null
/dev/tty
```

这些并不是普通文件，而是代表硬盘、终端、特殊设备等。

### `/proc`

`/proc` 是一个虚拟文件系统，用来表示进程和 kernel 信息。

例如：

```bash
/proc/cpuinfo
/proc/meminfo
```

可以查看 CPU 和内存信息。

### `/etc`

`/etc` 通常保存系统配置文件。

例如：

```text
/etc/hosts
/etc/passwd
```

这些文件用于配置系统行为。

## 19. 一个具体例子：运行程序时发生了什么？

假设你输入：

```bash
python analyse.py
```

大致过程是：

```text
1. Shell 读取命令 python analyse.py
2. Shell 请求 kernel 启动 python 程序
3. Kernel 创建一个新的 process
4. Kernel 为该 process 分配 PID
5. Kernel 分配内存和 CPU 时间
6. Python process 读取 analyse.py 这个 file
7. 程序运行过程中可能读取 input file
8. 程序可能写入 output file
9. 程序结束后，process 消失
10. 输出文件仍然保存在 file system 中
```

这个例子体现了 Linux 中 file 和 process 的关系：

```text
程序文件被执行后成为 process；
process 运行时会读取和写入 files。
```

## 20. File 和 Process 对比总结

|概念|英文关键词|中文理解|示例|
|---|---|---|---|
|File|file|数据集合，保存在文件系统中|`notes.md`, `data.csv`, `program.c`|
|Process|process|正在执行的程序|`python script.py`|
|PID|process identifier|进程编号|`1234`|
|Path|path|文件或目录的位置|`/home/leyan/file.txt`|
|Directory|directory|特殊文件，用来包含其他文件|`/home/leyan/Documents`|
|System call|system call|程序请求 kernel 服务的接口|open, read, write|

---

## 21. 核心理解

这一节最重要的是理解：

```text
Linux 中的对象大多可以理解为 file 或 process。
```

更具体地说：

```text
Process 是正在运行的程序，由 PID 标识；
File 是数据集合，通过 path 定位；
Directory 是特殊类型的 file，用来组织其他 files；
Kernel 管理 processes，并通过 system calls 处理文件系统和资源请求。
```
