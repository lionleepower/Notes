## 1. Unix 的起源

Linux 的历史需要从 **Unix** 开始讲起。

**Unix** 操作系统最早诞生于 **1969 年**，地点是 **Bell Laboratories**，也就是贝尔实验室。  
最初的 Unix 是用 **assembly language**，即汇编语言写成的。

在当时，操作系统和系统级程序通常使用汇编语言开发，因为汇编语言可以更直接地控制硬件，从而获得更高的性能。

---

## 2. Unix 与 C 语言

到了 **1973 年**，**Ken Thompson** 和 **Dennis Ritchie** 成功使用他们新开发的 **C language** 重写了 Unix。

这是一个非常大胆的决定，因为当时大多数人认为：

```text
系统编程应该使用汇编语言，因为这样才能最大限度发挥硬件性能。
````

但是使用 C 语言重写 Unix 带来了一个重要优势：

```text
Unix 变得更加 portable，也就是更容易移植到不同的硬件平台上。
```

---

## 3. Portable Operating System

**Portable operating system** 指的是可以比较容易地从一种硬件平台迁移到另一种硬件平台的操作系统。

在 Unix 之前，操作系统通常和特定硬件绑定得很紧。  
如果换了一种计算机架构，系统可能就需要大规模重写。

C 语言版本的 Unix 使操作系统的核心代码更接近高级语言，而不是完全依赖底层硬件指令。  
因此，只需要修改较少的硬件相关部分，就可以让 Unix 在不同机器上运行。

这对计算机行业非常重要，因为它推动了操作系统的可移植性发展。

---

## 4. Unix 的授权问题

虽然 Unix 的可移植性非常重要，但后来也出现了一个问题：

```text
不同版本的 Unix 存在 licensing 问题。
```

也就是说，Unix 的使用、修改和分发受到授权限制。  
这使得开发者和用户不能完全自由地使用和修改系统。

因此，一些开发者开始意识到，计算机社区需要更加自由、开放的操作系统。

---

## 5. GNU Project

**Richard Stallman** 是美国的软件自由运动倡导者和程序员。  
他认为软件应该允许用户自由使用、研究、修改和分发。

因此，他在 **1983 年** 发起了 **GNU Project**。  
后来，他还创立了 **Free Software Foundation**，即自由软件基金会。

GNU Project 的目标是：

```text
创建一个完全 free and open source 的操作系统，并且与 Unix 兼容。
```

这里的 **free** 不只是指免费，更强调自由，即用户拥有使用、修改和分发软件的自由。

GNU 的目标系统可以理解为：

- **Unix-compatible**
    
- **Unix-like**
    
- **free software**
    
- **open source**
    

但是 GNU Project 在早期缺少一个完整可用的 kernel。  
这也是后来 Linux kernel 变得非常关键的原因。

---

## 6. MINIX

在 **1987 年**，**Andrew Tanenbaum** 发布了一个简化版的类 Unix 操作系统，叫做 **MINIX**。

MINIX 的特点是：

- Unix-like operating system；
    
- 设计较为简洁；
    
- 源代码被发布出来；
    
- 主要用于教学和学术目的；
    
- 适合学习操作系统原理。
    

Andrew Tanenbaum 是 Vrije Universiteit 的教授。  
MINIX 对后来的 Linux 发展产生了重要影响。

---

## 7. Linux Kernel 的诞生

**Linus Torvalds** 在赫尔辛基大学读书时，开始基于 MINIX 的思想开发一个新的操作系统内核。

在 **1991 年 9 月**，Torvalds 发布了 Linux kernel 的第一个版本：

```text
Linux kernel version 0.1
```

这里要注意：

```text
Linux 最初严格来说指的是 kernel，而不是完整操作系统。
```

Kernel 是操作系统的核心部分，负责进程调度、内存管理、文件系统、设备管理等底层任务。

---

## 8. Linux 与 GNU 的结合

Linux kernel 发布后，Torvalds 将其置于 **GNU General Public License**，简称 **GPL** 之下。

这意味着：

- 每个人都可以访问 Linux kernel 的源代码；
    
- 每个人都可以修改它；
    
- 每个人都可以重新发布修改后的版本；
    
- 但需要遵守 GPL 的开源协议要求。
    

这极大促进了 Linux 和开源社区的发展。

随后，GNU Project 中的许多组件被整合进 Linux kernel 周围，例如：

```text
GNU Core Utilities
```

这些 GNU 工具和 Linux kernel 结合后，形成了第一个完整可用的自由开源操作系统之一。

所以更严格地说，很多人把完整系统称为：

```text
GNU/Linux
```

因为完整系统不仅包括 Linux kernel，也包括大量 GNU 工具。

---

## 9. Linux Distribution

随着 Linux 的发展，它被适配到了各种不同规模和用途的计算机系统中。

为了满足不同用户和应用场景的需求，出现了许多不同版本的 Linux。  
这些不同版本被称为：

```text
Linux distributions
```

简称：

```text
distros
```

Linux distribution 通常包含：

- Linux kernel；
    
- GNU 工具；
    
- 软件包管理器；
    
- 系统库；
    
- 桌面环境；
    
- 默认应用程序；
    
- 配置工具。
    

常见的 Linux distributions 包括：

- Ubuntu
    
- Debian
    
- Fedora
    
- Arch Linux
    
- Red Hat Enterprise Linux
    
- CentOS / Rocky Linux / AlmaLinux
    
- openSUSE
    

不同 distribution 的目标用户不同。  
例如：

|Distribution|常见用途|
|---|---|
|Ubuntu|桌面用户、服务器、初学者|
|Debian|稳定服务器、开源社区|
|Fedora|新技术测试、开发者|
|Arch Linux|高度自定义、进阶用户|
|Red Hat Enterprise Linux|企业级服务器|
|Rocky Linux / AlmaLinux|企业服务器、RHEL 替代方案|

---

## 10. Android 与 Linux Kernel

现在世界上最流行的操作系统之一其实是 **Android**。

Android 并不是传统意义上的桌面 Linux distribution，但它是建立在：

```text
Linux kernel
```

之上的。

也就是说，Android 的底层核心仍然使用 Linux kernel。  
这说明 Linux kernel 不仅用于服务器和 HPC，也被广泛应用在手机、嵌入式设备和消费电子产品中。

---

## 11. 时间线总结

|时间|事件|
|---|---|
|1969|Unix 在 Bell Laboratories 诞生，最初使用 assembly language 编写|
|1973|Ken Thompson 和 Dennis Ritchie 用 C language 重写 Unix|
|1983|Richard Stallman 发起 GNU Project|
|1987|Andrew Tanenbaum 发布 MINIX 源代码，用于教学和研究|
|1991|Linus Torvalds 发布 Linux kernel 0.1|
|之后|GNU 工具与 Linux kernel 结合，形成完整的自由开源操作系统|
|现在|Linux 被广泛用于 HPC、服务器、桌面、移动设备和嵌入式系统|

---

## 12. 核心理解

Linux 的发展可以概括为一条历史线：

```text
Unix → GNU Project → MINIX → Linux Kernel → GNU/Linux Distributions
```

其中：

- Unix 提供了设计思想；
    
- GNU Project 提供了自由开源的软件生态；
    
- MINIX 提供了教学和启发；
    
- Linux kernel 提供了关键的操作系统核心；
    
- Linux distributions 把 kernel、工具和软件整合成完整系统。
