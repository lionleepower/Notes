# Introduction to Linux

## 1. Linux 简介

**Linux** 是一种非常灵活的类 Unix 操作系统，英文可以称为 **Unix-like operating system**。  
它在 **高性能计算** 和 **科学计算** 领域中占据非常重要的地位，尤其是在 HPC 系统、服务器、超级计算机和科研平台中被广泛使用。

Linux 的几个核心特点：

- **Multi-user**：多用户操作系统，允许多个用户同时使用同一台机器。
- **Preemptive multitasking**：抢占式多任务系统，可以同时管理多个程序的运行。
- **Hardware resource management**：负责管理 CPU、内存、磁盘、输入输出设备等硬件资源。
- **File system management**：管理目录、文件系统和文件访问。
- **Program execution**：负责加载和执行程序。
- **Open source ecosystem**：拥有大量免费、开源的工具、库和软件。

简单来说，Linux 不只是一个操作系统本身，它还拥有一个庞大的软件生态系统，尤其适合开发、科研、服务器运维和 HPC 任务。

---

## 2. Linux 的两个主要组成部分

Linux 主要由两个重要部分组成：

1. **Kernel**
2. **Shell**

它们之间的关系可以简单理解为：

```text
User → Shell → Kernel → Hardware
```

用户通常不会直接操作 kernel，而是通过 shell 或图形界面向系统发出指令。

---

## 3. Kernel

**Kernel** 是 Linux 操作系统的核心部分，也就是系统真正管理硬件和资源的地方。

Kernel 的主要职责包括：

- 管理 **processes**，即进程；
- 调度程序运行，称为 **process scheduling**；
- 管理系统和用户的输入输出，即 **I/O management**；
- 管理硬件设备，即 **device management**；
- 管理文件和目录，即 **file management**；
- 管理内存，即 **memory management**；
- 与硬件直接交互。

可以把 kernel 理解为 Linux 的“大脑”或“核心控制层”。  
它决定哪些程序可以运行、什么时候运行、使用多少资源，以及如何访问硬件。

例如，当你运行：

```
python script.py
```

shell 会接收这条命令，然后把任务交给 kernel。  
kernel 负责创建进程、分配内存、访问文件，并让 CPU 执行程序。

---

## 4. Shell

**Shell** 是用户和 kernel 之间的文本接口，也可以称为 **command-line interface**

用户通过 shell 输入命令，shell 再把这些命令传递给 kernel 执行。

Shell 通常会不断重复四个步骤：

1. 显示提示符，即 **prompt**
2. 读取用户输入的命令，即 **read command**
3. 处理命令，即 **process command**
4. 执行命令，即 **execute command**

例如，常见的 shell 提示符可能长这样：

```
user@machine:~$
```

当你输入：

```
ls
```

shell 会读取 `ls`，识别这是一个列出目录内容的命令，然后请求 kernel 执行对应操作。

---

## 5. Kernel 和 Shell 的区别

|组件|英文关键词|作用|
|---|---|---|
|Kernel|operating system core|系统核心，直接管理硬件、内存、进程、文件和设备|
|Shell|command-line interface|用户与系统交互的文本接口，用来输入命令|
|User|user|一般不直接接触 kernel，而是通过 shell 或桌面环境操作系统|
|Hardware|hardware resources|CPU、内存、磁盘、网络、输入输出设备等|

可以用一个简单例子理解：

```
用户输入命令：ls↓Shell 接收并解析命令↓Kernel 执行文件系统操作↓Shell 显示执行结果
```

所以：

- **Kernel 负责真正执行和管理系统资源**
- **Shell 负责接收用户命令并与 kernel 沟通**

---

## 6. 用户如何与 Linux 交互

Linux 用户通常不会直接和 kernel 交互，而是通过以下方式操作系统：

### 方式一：Shell

例如：

```
cd Documentslsmkdir testrm file.txt
```

这是 HPC、服务器、远程计算环境中最常见的方式。

### 方式二：Desktop Environment

也就是图形桌面环境，例如 Ubuntu Desktop、GNOME、KDE 等。

用户可以通过鼠标点击文件夹、打开程序、移动窗口等方式使用系统。

但是在 HPC 和服务器环境中，通常更多使用 **shell**，因为：

- 远程连接更方便；
- 自动化脚本更容易编写；
- 系统资源占用更少；
- 适合批量任务和命令行工具。

---

## 7. 小结

Linux 是一个功能强大的类 Unix 操作系统，在 HPC 和科学计算中非常重要。  
它的核心是 **kernel**，负责管理硬件资源、进程、文件和内存。  
用户通常不会直接操作 kernel，而是通过 **shell** 或图形界面与系统交互。

最关键的理解是：

```
Shell 是用户输入命令的地方；Kernel 是真正执行任务和管理系统资源的核心。
```
